WebMCP Auto-UI fournit 30+ widgets prêts à l'emploi : charts, tables, cartes, galeries, timelines, et bien d'autres. Ce tutorial montre comment les utiliser dans une app Svelte.

## Widgets disponibles

### Widgets simples (données directes)

| Widget | Usage | Exemple |
|--------|-------|---------|
| `stat` | KPI simple | `{label: "Total", value: "42", trend: "up"}` |
| `alert` | Notification | `{title: "Attention", level: "warn", message: "..."}` |
| `code` | Bloc de code | `{lang: "python", content: "print('hello')"}` |
| `text` | Paragraphe | `{content: "Texte libre..."}` |
| `list` | Énumération | `{items: ["A", "B", "C"]}` |
| `kv` | Paires clé-valeur | `{rows: [["Nom", "Dupont"], ["Age", "42"]]}` |
| `tags` | Badges | `{tags: [{text: "JS", active: true}]}` |
| `actions` | Boutons | `{buttons: [{label: "OK", primary: true}]}` |
| `chart` | Barres simples | `{bars: [["Jan", 10], ["Fev", 20]]}` |

### Widgets riches (spécifications complexes)

| Widget | Usage | Cas d'usage |
|--------|-------|-----------|
| `stat-card` | KPI enrichi | Dashboards avec delta, tendance, couleur |
| `data-table` | Tableau interactif | Données tabulaires avec tri |
| `timeline` | Chronologie | Événements datés avec statuts |
| `profile` | Fiche personne | Bio, champs, statistiques |
| `trombinoscope` | Grille de portraits | Équipes, assemblées |
| `cards` | Grille de cartes | Résultats, dossiers, galerie |
| `json-viewer` | Arbre JSON | Inspection de structures complexes |
| `chart-rich` | Multi-séries | Bar, line, area, pie, donut |
| `sankey` | Diagramme de flux | Votes, parcours, transferts |
| `map` | Carte Leaflet | Marqueurs géographiques |
| `hemicycle` | Hémicycle SVG | Composition parlementaire |
| `grid-data` | Grille avec heatmap | Comparaison, mise en évidence |
| `d3` | Visualisations D3 | Treemap, hex-heatmap, force graph |
| `gallery` | Galerie d'images | Lightbox et navigation |
| `carousel` | Diaporama | Slides avec auto-play |
| `log` | Flux de logs | Événements système |
| `js-sandbox` | Code JavaScript | Prototypes interactifs |

## Exemple 1 : Dashboard avec widgets simples

```svelte
<!-- routes/dashboard/+page.svelte -->
<script lang="ts">
  import { BlockRenderer } from '@webmcp-auto-ui/ui';

  let blocks = [
    {
      id: 'stat-1',
      type: 'stat',
      data: {
        label: 'Utilisateurs actifs',
        value: '2,847',
        trend: 'up',
        trendDir: 'up',
      },
    },
    {
      id: 'alert-1',
      type: 'alert',
      data: {
        title: 'Maintenance programmée',
        message: 'Vendredi 22h-23h UTC',
        level: 'info',
      },
    },
    {
      id: 'list-1',
      type: 'list',
      data: {
        title: 'Tâches prioritaires',
        items: [
          'Migrer BD PostgreSQL',
          'Activer 2FA pour tous',
          'Auditer permissions API',
        ],
      },
    },
  ];
</script>

<div class="grid grid-cols-1 md:grid-cols-3 gap-4 p-6">
  {#each blocks as block (block.id)}
    <BlockRenderer type={block.type} data={block.data} />
  {/each}
</div>
```

## Exemple 2 : Table de données avec tri

```svelte
<!-- routes/users/+page.svelte -->
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';

  const tableData = {
    title: 'Utilisateurs',
    columns: [
      { key: 'name', label: 'Nom', align: 'left' },
      { key: 'email', label: 'Email', align: 'left' },
      { key: 'role', label: 'Rôle', align: 'center' },
      { key: 'joined', label: 'Inscription', align: 'right' },
    ],
    rows: [
      { name: 'Alice Dupont', email: 'alice@example.com', role: 'Admin', joined: '2023-01-15' },
      { name: 'Bob Martin', email: 'bob@example.com', role: 'User', joined: '2023-06-20' },
      { name: 'Claire Lemaître', email: 'claire@example.com', role: 'Editor', joined: '2024-02-10' },
    ],
  };

  function handleRowClick(row: Record<string, unknown>) {
    console.log('Ligne cliquée:', row);
    // Naviguer vers le profil de l'utilisateur
  }
</script>

<WidgetRenderer
  type="data-table"
  data={tableData}
  oninteract={(type, action, payload) => {
    if (action === 'rowclick') {
      handleRowClick(payload as Record<string, unknown>);
    }
  }}
/>
```

## Exemple 5 : Galerie d'images avec validation d'URL

WebMCP Auto-UI valide automatiquement les URLs d'images. Les URLs halluccinées sont filtrées :

```svelte
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';

  const gallery = {
    title: 'Photos de la conférence',
    images: [
      {
        src: 'https://images.example.com/conf-01.jpg', // ✅ URL valide (https)
        alt: 'Keynote speaker',
        caption: 'Discours principal',
      },
      {
        src: 'https://images.example.com/conf-02.jpg',
        alt: 'Networking',
        caption: 'Pause café',
      },
      // src: 'hallucinatedimage.jpg' — filtré automatiquement ❌
    ],
    columns: 3,
  };
</script>

<WidgetRenderer type="gallery" data={gallery} />
```

## Intégration avec l'agent loop

Utiliser `widget_display()` depuis l'agent pour ajouter des widgets dynamiquement :

```typescript
// Dans une recette MCP ou un outil WebMCP
const result = await webmcpServer.executeTool('widget_display', {
  name: 'data-table',
  params: {
    title: 'Résultats de recherche',
    columns: [
      { key: 'title', label: 'Titre' },
      { key: 'relevance', label: 'Pertinence' },
    ],
    rows: searchResults.map((r, i) => ({
      title: r.title,
      relevance: `${Math.round(r.score * 100)}%`,
    })),
  },
});
```

## Callbacks d'interaction

Les widgets riches supportent des callbacks pour capturer les interactions utilisateur :

```svelte
<WidgetRenderer
  type="cards"
  data={cardsData}
  oninteract={(type, action, payload) => {
    if (action === 'cardclick') {
      // L'utilisateur a cliqué sur une carte
      const card = payload as Record<string, unknown>;
      console.log('Carte cliquée:', card);
    }
  }}
/>
```

**Actions disponibles par widget :**

| Widget | Actions |
|--------|---------|
| `data-table` | `rowclick` |
| `timeline` | `eventclick` |
| `trombinoscope` | `personclick` |
| `hemicycle` | `groupclick` |
| `cards` | `cardclick` |
| `gallery` | `imageclick` |
| `carousel` | `slidechange` |
| `list` | `itemclick` |

## Points clés

✅ **SafeImage filtre les URLs halluccinées** — pas besoin de valider manuellement

✅ **BlockRenderer pour les widgets simples** — plus léger quand pas d'interaction

✅ **WidgetRenderer quand il faut des callbacks** — pour traiter les clics utilisateur

✅ **Types TypeScript complets** — l'IDE suggère les champs disponibles

❌ **Ne pas passer des URLs d'images invalides** — elles seront filtrées

❌ **Ne pas oublier les `required` du schéma** — les widgets doivent valider l'entrée
