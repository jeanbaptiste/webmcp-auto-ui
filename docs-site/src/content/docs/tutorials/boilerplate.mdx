Ce tutoriel guide pas a pas l'installation du boilerplate webmcp-auto-ui, la connexion a un serveur MCP, et la creation d'un premier widget custom.

## Vue d'ensemble

Le boilerplate fournit une app SvelteKit 5 pre-cablee avec :

- **shadcn-svelte** pour le design system
- **@webmcp-auto-ui/core** pour le client MCP et le serveur WebMCP
- **@webmcp-auto-ui/agent** pour la boucle agent et le provider LLM
- **@webmcp-auto-ui/sdk** pour le canvas store et l'encodage HyperSkill
- **@webmcp-auto-ui/ui** pour les composants (LLMSelector, McpStatus, WidgetRenderer, etc.)

L'app inclut 3 widgets custom (fiches deputes, scrutins, amendements) comme exemples d'integration.

---

## Etape 1 : Scaffolding

Clonez le boilerplate avec degit :

```bash
npx degit jeanbaptiste/webmcp-auto-ui/apps/boilerplate my-app
cd my-app
npm install
```

---

## Etape 2 : Configurer le proxy API

Creez le fichier `src/routes/api/chat/+server.ts` (deja present dans le boilerplate) :

```ts
import { env } from '$env/dynamic/private';
import type { RequestHandler } from '@sveltejs/kit';
import { anthropicProxy } from '@webmcp-auto-ui/agent/server';

export const POST: RequestHandler = async ({ request }) => {
  const body = await request.json() as Record<string, unknown>;
  const apiKey = (body.__apiKey as string | undefined) || env.ANTHROPIC_API_KEY || '';
  delete body.__apiKey;
  return anthropicProxy(body, apiKey, request.headers.get('X-Model'));
};
```

Ajoutez votre cle API dans `.env` :

```
ANTHROPIC_API_KEY=sk-ant-...
```

Le proxy relaie les requetes vers l'API Claude. Le modele est selectionne dynamiquement via le header `X-Model` (alias : `'haiku'`, `'sonnet'`, `'opus'`).

## Etape 3 : Connecter un serveur MCP

Le boilerplate utilise `McpMultiClient` pour gerer plusieurs serveurs MCP :

```ts
import { McpMultiClient } from '@webmcp-auto-ui/core';

const multiClient = new McpMultiClient();
await multiClient.addServer('https://mcp.code4code.eu/mcp');
```

Dans le composant principal, la connexion est deja cablee avec un champ URL et un bouton "Connecter". Au chargement de la page, l'app se connecte automatiquement a l'URL par defaut.

Pour ajouter un serveur en dur :

```ts
await multiClient.addServer('https://votre-serveur-mcp.example/mcp');
```

---

## Etape 4 : Comprendre le provider LLM

Le boilerplate utilise `RemoteLLMProvider` pour communiquer avec Claude via le proxy :

```ts
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({ proxyUrl: '/api/chat' });
```

Le modele est change dynamiquement via le composant `<LLMSelector>` qui expose les alias `'haiku'`, `'sonnet'` et `'opus'`.

## Etape 5 : Lancer la boucle agent

La boucle agent orchestre le LLM, les outils MCP et les widgets WebMCP :

```ts
import { runAgentLoop, buildSystemPrompt, fromMcpTools, autoui } from '@webmcp-auto-ui/agent';
import type { ToolLayer, McpLayer } from '@webmcp-auto-ui/agent';

// Construire les layers (MCP distant + WebMCP local)
const layers: ToolLayer[] = [];

// Layer MCP (donnees distantes)
for (const server of multiClient.listServers()) {
  const mcpLayer: McpLayer = {
    protocol: 'mcp',
    serverName: server.name,
    tools: fromMcpTools(server.tools),
  };
  layers.push(mcpLayer);
}

// Layer WebMCP (widgets natifs)
layers.push(autoui.layer());

// System prompt dynamique
const systemPrompt = buildSystemPrompt(layers);

// Lancer la boucle
const result = await runAgentLoop(userMessage, {
  provider,
  systemPrompt,
  layers,
  maxIterations: 10,
  maxTokens: 4096,
  callbacks: {
    onWidget: (type, data) => {
      // Ajouter le widget au canvas
      const id = 'b_' + Date.now().toString(36);
      blocks = [...blocks, { id, type, data }];
      return { id };
    },
    onText: (text) => { ephemeralText = text; },
    onToolCall: (call) => { console.log('Tool:', call.name); },
  },
});
```

## Etape 6 : Afficher les widgets

Le rendu des widgets se fait via `<WidgetRenderer>` :

```svelte
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';
</script>

<div class="grid grid-cols-1 md:grid-cols-2 gap-4">
  {#each blocks as block (block.id)}
    <WidgetRenderer
      id={block.id}
      type={block.type}
      data={block.data}
      {servers}
    />
  {/each}
</div>
```

Le `WidgetRenderer` resout automatiquement le composant Svelte correspondant au `type` du widget, en cherchant dans :
1. Les serveurs WebMCP custom (passes via `servers`)
2. Les widgets natifs autoui (stat, chart, table, map, etc.)

## Etape 7 : Ajouter les composants UI

Le boilerplate integre les composants UI pre-faits :

```svelte
<script lang="ts">
  import {
    LLMSelector,     // Selecteur de modele (haiku/sonnet/opus)
    McpStatus,       // Indicateur de connexion MCP
    AgentProgress,   // Progression de la boucle agent
    WidgetRenderer,  // Rendu des widgets
    getTheme,        // Theme clair/sombre
  } from '@webmcp-auto-ui/ui';
</script>

<!-- Barre superieure -->
<header class="flex items-center gap-3 px-4 border-b">
  <McpStatus
    connecting={canvas.mcpConnecting}
    connected={canvas.mcpConnected}
    name={mcpName}
    servers={multiClient.listServers().map(s => ({
      url: s.url, name: s.name, toolCount: s.tools.length
    }))}
  />
  <LLMSelector />
</header>

<!-- Progression agent -->
<AgentProgress
  active={canvas.generating}
  elapsed={chatTimer}
  toolCalls={chatToolCount}
  lastTool={chatLastTool}
/>
```

## Etape 8 : Ajouter un widget custom

Pour enregistrer vos propres widgets, creez un serveur WebMCP local :

```ts
import { createWebMcpServer } from '@webmcp-auto-ui/core';
import MonWidget from './widgets/MonWidget.svelte';
import monWidgetRecipe from './recipes/mon-widget.md?raw';

const monServeur = createWebMcpServer('mon-app', {
  description: 'Widgets custom de mon application',
});

monServeur.registerWidget(monWidgetRecipe, MonWidget);
```

Puis ajoutez le layer a la boucle agent :

```ts
layers.push(monServeur.layer());
```

Et passez le serveur au WidgetRenderer :

```svelte
<WidgetRenderer
  id={block.id}
  type={block.type}
  data={block.data}
  servers={[monServeur]}
/>
```

Pour les details sur la creation de widgets custom, voir le tutoriel [Creer un widget custom](./create-custom-widget).

## Lancer le serveur de developpement

```bash
npm run dev
```

L'app est accessible sur `http://localhost:5173`. Tapez une question dans le chat pour declencher la boucle agent.

```
  vite.config.ts
```
