Ce tutoriel montre comment créer et enregistrer un widget custom dans le système de composants WebMCP Auto-UI, puis l'utiliser via `BlockRenderer`.

## Vue d'ensemble

Un widget custom est composé de :
1. **Un composant Svelte** qui affiche les données
2. **Un enregistrement** dans le registry du canvas
3. **Un type TypeScript** pour la validation

Le système supporte deux modes de rendu :
- **Svelte** : via `<BlockRenderer>` (recommandé pour les apps web)
- **Vanilla** : via `mountWidget()` pour intégration sans framework

## Étape 1 : Créer le composant

Créez un composant Svelte simple, par exemple `CustomProgressBar.svelte` :

```svelte
<script lang="ts">
  interface Props {
    data: {
      label: string;
      current: number;
      max: number;
      color?: string;
    };
  }

  let { data }: Props = $props();
  const percentage = $derived((data.current / data.max) * 100);
</script>

<div class="card">
  <h3>{data.label}</h3>
  <div class="progress-container">
    <div
      class="progress-bar"
      style="width: {percentage}%; background-color: {data.color ?? '#3b82f6'}"
    />
  </div>
  <p class="text-sm">{data.current} / {data.max}</p>
</div>

<style>
  .card {
    padding: 1rem;
    border: 1px solid #e5e7eb;
    border-radius: 0.5rem;
  }

  .progress-container {
    height: 8px;
    background: #f3f4f6;
    border-radius: 4px;
    overflow: hidden;
    margin: 0.5rem 0;
  }

  .progress-bar {
    height: 100%;
    transition: width 0.3s ease;
  }
</style>
```

## Étape 2 : Enregistrer le widget dans le registry

Modifiez le fichier de registry de votre app (ex: `src/lib/widgets.ts`) :

```typescript
import CustomProgressBar from './widgets/CustomProgressBar.svelte';

export const CUSTOM_WIDGETS: Record<string, Component> = {
  'progress-bar': CustomProgressBar,
};
```

## Étape 3 : Utiliser BlockRenderer

Dans votre composant parent, importez `BlockRenderer` et passez le widget :

```svelte
<script lang="ts">
  import { BlockRenderer } from '@webmcp-auto-ui/ui';
  import { CUSTOM_WIDGETS } from './lib/widgets';
  import { canvas } from '@webmcp-auto-ui/sdk/canvas';

  // Ajouter un widget au canvas
  function addProgress() {
    canvas.addWidget('progress-bar', {
      label: 'Téléchargement',
      current: 45,
      max: 100,
      color: '#10b981',
    });
  }
</script>

<button onclick={addProgress}>Ajouter barre de progression</button>

<div>
  {#each canvas.blocks as block (block.id)}
    <BlockRenderer 
      id={block.id}
      type={block.type}
      data={block.data}
      customWidgets={CUSTOM_WIDGETS}
    />
  {/each}
</div>
```

## Étape 4 : Typer le widget

Ajoutez le type dans la définition d'union de `WidgetType` :

```typescript
// Dans votre types.ts ou types.d.ts
declare global {
  namespace WebMCP {
    type WidgetType = 
      | 'stat' | 'chart' | 'kv' | 'list'
      | 'progress-bar';  // ← votre widget custom
  }
}
```

Ou dans `canvas.ts` du SDK si vous modifiez le type `WidgetType` :

```typescript
export type WidgetType =
  | 'stat' | 'kv' | 'list' | 'chart' | 'alert' | 'code' | 'text' | 'actions' | 'tags'
  | 'stat-card' | 'data-table' | 'timeline' | 'profile' | 'trombinoscope' | 'json-viewer'
  | 'hemicycle' | 'chart-rich' | 'cards' | 'grid-data' | 'sankey' | 'map' | 'log'
  | 'gallery' | 'carousel' | 'd3' | 'js-sandbox'
  | 'progress-bar'  // ← ajout du type custom
  | (string & {});  // fallback pour les widgets de plugins
```

## Étape 5 : Ajouter des interactions

Si votre widget doit communiquer avec l'agent via le FONC bus, utilisez les callbacks :

```svelte
<script lang="ts">
  import { bus } from '@webmcp-auto-ui/ui';

  interface Props {
    id?: string;
    data: {
      label: string;
      current: number;
      max: number;
      onUpdate?: (value: number) => void;
    };
  }

  let { id, data }: Props = $props();

  function handleClick() {
    // Emettre un événement bus vers le canvas
    if (id) {
      bus.broadcast(id, 'progress-complete', { label: data.label });
    }
    data.onUpdate?.(data.max);
  }
</script>

<div class="progress-bar">
  <progress value={data.current} max={data.max}></progress>
  <button onclick={handleClick}>Terminé</button>
</div>
```

## Étape 6 : Intégrer avec le WebMCP bus (optionnel)

Pour permettre à l'agent de mettre à jour le widget :

```svelte
<script lang="ts">
  import { onMount } from 'svelte';

  onMount(() => {
    const mc = (navigator as any).modelContext;
    if (!mc) return;

    mc.registerTool({
      name: `progress_${id}_update`,
      description: `Update progress bar widget`,
      inputSchema: {
        type: 'object',
        properties: {
          current: { type: 'number' },
        },
        required: ['current'],
      },
      execute: (args: Record<string, any>) => {
        data.current = args.current;
        return {
          content: [{ type: 'text', text: `Progress updated to ${args.current}` }],
        };
      },
    });
  });
</script>
```

## Exemple complet : widget avec état persistant

```svelte
<script lang="ts">
  import { canvas } from '@webmcp-auto-ui/sdk/canvas';

  interface Props {
    id: string;
    data: {
      items: string[];
      completed: number;
    };
    oninteract?: (type: string, action: string, payload: unknown) => void;
  }

  let { id, data, oninteract }: Props = $props();
  let items = $state(data.items);
  let completed = $state(data.completed);

  function toggleItem(index: number) {
    completed = completed === index ? -1 : index;
    // Mettre à jour le canvas
    canvas.updateBlock(id, { completed });
    // Émettre une interaction
    oninteract?.('checklist', 'toggle', { index, completed });
  }
</script>

<div class="checklist">
  <h3>Checklist ({completed + 1}/{items.length})</h3>
  {#each items as item, i (i)}
    <label>
      <input
        type="checkbox"
        checked={completed === i}
        onchange={() => toggleItem(i)}
      />
      <span class:done={completed === i}>{item}</span>
    </label>
  {/each}
</div>

<style>
  .checklist { padding: 1rem; }
  label { display: flex; gap: 0.5rem; margin: 0.5rem 0; }
  .done { text-decoration: line-through; color: #6b7280; }
</style>
```

## Intégration avec le système WebMCP

Si vous voulez enregistrer votre widget dans un **serveur WebMCP custom** (par exemple une app qui expose ses propres widgets via MCP) :

```typescript
import { createWebMcpServer } from '@webmcp-auto-ui/core';

const myServer = createWebMcpServer('my-widgets', {
  description: 'Custom widgets for my app',
});

// Enregistrer le widget avec sa recette
myServer.registerWidget(`---
widget: progress-bar
description: Barre de progression interactive
schema:
  type: object
  required:
    - label
    - current
    - max
  properties:
    label:
      type: string
    current:
      type: number
    max:
      type: number
    color:
      type: string
---

## Utilisation

Appeler widget_display('progress-bar', {label: "Téléchargement", current: 45, max: 100})
`, undefined); // renderer = undefined → sera résolu par BlockRenderer

export { myServer };
```
