Le **tool calling** est le cœur de WebMCP Auto-UI. L'agent LLM demande des outils, et le système dispatcher les achemine vers le bon serveur (MCP ou WebMCP).

## Anatomie d'un appel d'outil

### 1. L'agent génère un tool_use block

```typescript
// LLM response
{
  type: 'tool_use',
  id: 'toolu_1234',
  name: 'weather_mcp_get_forecast',
  input: { location: 'Paris', days: 3 }
}
```

### 2. Le dispatcher route l'appel

```typescript
// Dans runAgentLoop, dispatcher.ts traite chaque block
for (const block of toolBlocks) {
  // Résoudre alias
  const resolvedName = aliasMap.get(block.name) ?? block.name;
  
  // Parser le préfixe : {serverName}_{protocol}_{toolName}
  const match = resolvedName.match(/^(.+?)_(mcp|webmcp)_(.+)$/);
  if (!match) {
    // Erreur : format invalide
    result = `Error: unknown tool format`;
  } else {
    const [, serverName, protocol, realToolName] = match;
    
    // Routing
    if (protocol === 'mcp') {
      // → MCP client.callTool()
    } else if (protocol === 'webmcp') {
      // → WebMCP server.executeTool()
    }
  }
}
```

### 3. Interceptor (pseudo-tools)

Avant dispatch normal, deux **pseudo-tools** sont interceptés localement :

```typescript
// list_tools, search_tools — pas d'activation serveur
if (toolMatch && (toolMatch[3] === 'list_tools' || toolMatch[3] === 'search_tools')) {
  const layer = layers.find(l => 
    sanitizeServerName(l.serverName) === serverName && 
    l.protocol === protocol
  );
  
  // Retourner la liste sans touching le serveur
  if (pseudoTool === 'list_tools') {
    result = JSON.stringify(layer.tools);
  } else {
    // search_tools
    const query = String(block.input.query ?? '').toLowerCase();
    const matches = layer.tools.filter(t =>
      t.name.toLowerCase().includes(query) ||
      t.description.toLowerCase().includes(query)
    );
    result = JSON.stringify(matches);
  }
}
```

### 4. Activation lazy du serveur

Lors du **premier appel réel** à un outil du serveur :

```typescript
const activateMatch = block.name.match(/^(.+?)_(mcp|webmcp)_(.+)$/);
if (activateMatch) {
  const [, serverName, protocol] = activateMatch;
  const serverKey = `${serverName}_${protocol}`;
  
  if (!activatedServers.has(serverKey)) {
    activatedServers.add(serverKey);
    const layer = layers.find(l => 
      sanitizeServerName(l.serverName) === serverName && 
      l.protocol === protocol
    );
    if (layer) {
      // Ajouter TOUS les outils du serveur à activeTools
      activeTools = activateServerTools(activeTools, layer);
    }
  }
}
```

## Dispatch MCP

<img src="/webmcp-auto-ui/diagrams/tool-calling-1.svg" alt="MCP dispatch flow from tool_use block to agent messages" />

### Code MCP

```typescript
if (protocol === 'mcp') {
  if (!client) {
    result = `Error: no MCP client available for tool ${name}`;
  } else {
    const mcpResult = await client.callTool(realToolName, block.input);
    const textContent = mcpResult.content?.find(
      (c) => c.type === 'text'
    ) as { text?: string } | undefined;
    const rawResult = textContent?.text ?? JSON.stringify(mcpResult);
    
    // Tronquer si résultat > 10,000 chars (default MAX_RESULT_LEN)
    result = truncateResults 
      ? truncateResult(rawResult, maxResultLength) 
      : rawResult;
  }
}
```

**Réseau** : Le client MCP établit une connexion SSE (Server-Sent Events) vers le serveur distant et appelle `tools/call`.

## Dispatch WebMCP

<img src="/webmcp-auto-ui/diagrams/tool-calling-2.svg" alt="WebMCP dispatch flow with recall, widget_display, canvas and executeTool branches" />

### Code WebMCP

```typescript
if (protocol === 'webmcp') {
  // ── Spécial : recall (relire un résultat compressé) ──
  if (realToolName === 'recall' && resultBuffer.size > 0) {
    const recallId = (block.input as { id: string }).id;
    result = resultBuffer.get(recallId) 
      ?? `Aucun résultat trouvé pour l'id '${recallId}'.`;
  } else {
    // ── Dispatch normal vers WebMCP server ──
    const webmcpServer = webmcpServers.get(serverName);
    if (!webmcpServer) {
      result = `Error: no WebMCP server "${serverName}" found.`;
    } else {
      const toolResult = await webmcpServer.executeTool(
        realToolName, 
        block.input
      );
      result = typeof toolResult === 'string' 
        ? toolResult 
        : JSON.stringify(toolResult);
      
      // ── Spécial : widget_display ──
      if (realToolName === 'widget_display') {
        const wr = toolResult as Record<string, unknown>;
        if (wr.widget && wr.data && !wr.error) {
          const widgetResult = callbacks.onWidget?.(
            wr.widget as string, 
            wr.data as Record<string, unknown>
          );
          if (widgetResult?.id) {
            result = JSON.stringify({ ...wr, id: widgetResult.id });
          }
        }
      }
      
      // ── Spécial : canvas (actions UI) ──
      if (realToolName === 'canvas') {
        const action = block.input.action as string;
        const id = block.input.id as string;
        const actionParams = block.input.params as Record<string, unknown>;
        
        switch (action) {
          case 'clear': callbacks.onClear?.(); break;
          case 'update': callbacks.onUpdate?.(id, actionParams ?? {}); break;
          case 'move': callbacks.onMove?.(
            id, 
            (actionParams?.x ?? block.input.x) as number,
            (actionParams?.y ?? block.input.y) as number
          ); break;
          case 'resize': callbacks.onResize?.(
            id,
            (actionParams?.width ?? block.input.width) as number,
            (actionParams?.height ?? block.input.height) as number
          ); break;
          case 'style': callbacks.onStyle?.(
            id,
            (actionParams?.styles ?? block.input.styles) as Record<string, string>
          ); break;
        }
      }
    }
  }
}
```

## Widget Display (cas spécial)

`widget_display()` est l'outil principal pour le **rendu UI** :

<img src="/webmcp-auto-ui/diagrams/tool-calling-3.svg" alt="widget_display validation, sanitization and rendering pipeline" />

### Exemple complet

```typescript
// Agent appelle
{
  type: 'tool_use',
  id: 'toolu_5678',
  name: 'autoui_webmcp_widget_display',
  input: {
    name: 'stat-card',
    params: {
      label: 'Conversion',
      value: '3.2%',
      unit: '%',
      trend: 'up',
      variant: 'success'
    }
  }
}

// Dispatcher :
// 1. Récupère le schema du widget 'stat-card'
// 2. Valide params vs schema
// 3. Sanitize image URLs (aucune à supprimer ici)
// 4. Génère ID 'w_1a2b3c'
// 5. Appelle onWidget('stat-card', {...params})
// 6. Canvas ajoute le widget : { id: 'w_1a2b3c', type: 'stat-card', data: {...} }
// 7. WidgetRenderer monte le composant Svelte StatCard avec les props
// 8. UI affiche la carte
```

## Canvas Tools (manipulation d'UI)

L'outil `canvas` permet à l'agent de **réarranger les widgets** :

```typescript
// Agent appelle
{
  type: 'tool_use',
  name: 'autoui_webmcp_canvas',
  input: {
    action: 'move',
    id: 'w_1a2b3c',
    params: { x: 100, y: 50 }
  }
}

// Callbacks dans l'agent
callbacks.onMove?.('w_1a2b3c', 100, 50);
// → Mise à jour CSS de la position du widget
```

**Actions** :
- `clear()` — vider le canvas
- `update(id, data)` — modifier les données d'un widget
- `move(id, x, y)` — repositionner (CSS transform)
- `resize(id, width, height)` — redimensionner
- `style(id, styles)` — appliquer styles CSS

## Recall (relire les résultats compressés)

Après ~2 itérations, les **anciens tool results** sont compressés pour économiser le contexte :

```typescript
// Avant compression
{
  type: 'tool_result',
  tool_use_id: 'toolu_1234',
  content: '{"data": [item1, item2, ... item1000], "total": 1000}'
}

// Après compression (mutated in-place)
{
  type: 'tool_result',
  tool_use_id: 'toolu_1234',
  content: '{"data": [item1, item2, ... [recall(\'toolu_1234\') pour le résultat complet, 5000 chars]'
}

// Stored in resultBuffer
resultBuffer.set('toolu_1234', '{"data": [item1, item2, ... item1000], "total": 1000}');
```

Si l'agent appelle `recall('toolu_1234')` :

```typescript
// Dispatcher intercepte recall
if (realToolName === 'recall' && resultBuffer.size > 0) {
  const recallId = (block.input as { id: string }).id;
  result = resultBuffer.get(recallId) ?? `Aucun résultat trouvé...`;
}
```

**Avantage** : Agent peut naviguer plusieurs couches de données sans bloquer le contexte.

## Gestion des erreurs

### Erreur de validation schema

```typescript
// Widget params invalides
{
  type: 'tool_use',
  name: 'autoui_webmcp_widget_display',
  input: {
    name: 'stat',
    params: { label: 'Sales' }  // ✗ manque value
  }
}

// Réponse
{
  type: 'tool_result',
  content: JSON.stringify({
    error: 'Validation failed',
    details: [{ path: '/value', message: 'required' }],
    expected_schema: { ... }
  })
}

// Agent reçoit erreur + schema → retente avec les bons champs
```

### Tool not found

```typescript
// MCP
if (!client) {
  result = `Error: no MCP client available for tool ${name}`;
}

// WebMCP
if (!webmcpServer) {
  result = `Error: no WebMCP server "${serverName}" found.`;
}
```

### Tool execution failure

```typescript
try {
  const toolResult = await webmcpServer.executeTool(realToolName, block.input);
  result = ...;
} catch (e) {
  call.error = e instanceof Error ? e.message : String(e);
  toolResults.push({ 
    type: 'tool_result', 
    tool_use_id: block.id, 
    content: `Error: ${call.error}` 
  });
}
```

## Résumé du flow

1. **LLM génère** `tool_use` block
2. **Dispatcher détecte** format + résout alias
3. **Pseudo-tools** (list_tools, search_tools) → réponse locale
4. **Activation lazy** : si 1er appel, ajouter tous les outils du serveur
5. **Routing** : MCP → réseau, WebMCP → local
6. **Post-processing** : compression, sanitize, callbacks
7. **tool_result** retourne à l'agent
8. **Itération suivante** de la boucle
