## Overview

Quand l'agent appelle un outil, le système :

1. **Résout l'alias** du nom d'outil (recette → implémentation réelle)
2. **Parse le préfixe** pour extraire `{serverName}_{protocol}_{toolName}`
3. **Route vers le bon backend** (MCP client ou WebMCP server)
4. **Capture le résultat** et le stocke pour possible `recall()`

## Dispatch par protocole

### MCP — Appels à distance

```typescript
// Outil: "recipes_mcp_search_recipes"
const [, serverName, protocol, realToolName] = 
  "recipes_mcp_search_recipes".match(/^(.+?)_(mcp|webmcp)_(.+)$/);

// serverName = "recipes"
// protocol = "mcp"
// realToolName = "search_recipes"

// Route vers MCP client
const mcpResult = await client.callTool(realToolName, block.input);
const textContent = mcpResult.content?.find(c => c.type === 'text');
const result = textContent?.text ?? JSON.stringify(mcpResult);
```

Les résultats MCP peuvent être :
- **text** — Chaîne de caractères
- **image** — Base64 encodé
- **resource** — URI externe
- **error** — Message d'erreur

### WebMCP — Exécution locale

```typescript
// Outil: "autoui_webmcp_widget_display"
const webmcpServer = webmcpServers.get("autoui");
const toolResult = await webmcpServer.executeTool("widget_display", params);

// Résultat JSON structuré
const result = typeof toolResult === 'string' ? toolResult : JSON.stringify(toolResult);
```

WebMCP est **synchrone** et directement local.

## Outils spéciaux

### `widget_display()` — Afficher un widget

```typescript
// Appel agent
widget_display('chart', {
  type: 'bar',
  labels: ['Q1', 'Q2', 'Q3', 'Q4'],
  data: [{ label: 'Ventes', values: [100, 150, 200, 250] }]
})

// Résultat
{
  "widget": "chart",
  "data": { /* params validés et nettoyés */ },
  "id": "w_abc123"
}

// Callback UI
callbacks.onWidget?.('chart', data);
// → Render <Chart spec={data} /> dans le DOM
```

### `recall()` — Retrouver un résultat comprimé

Quand les résultats sont comprimés (> 300 chars), l'agent peut les rappeler :

```typescript
// Ancien résultat tronqué
"SELECT * FROM users;... [recall('toolu_abc') pour résultat complet, 125000 chars]"

// Appel de rappel
recall('toolu_abc')

// Retour : résultat complet depuis resultBuffer
"SELECT * FROM users; [5000 rows détaillées]"
```

**Important** : `recall()` est interceptée **avant** le serveur WebMCP :

```typescript
if (realToolName === 'recall' && resultBuffer.size > 0) {
  const recallId = block.input.id;
  result = resultBuffer.get(recallId) ?? `Aucun résultat trouvé...`;
  // ← Pas d'appel au serveur
}
```

### `canvas()` — Manipuler les widgets

```typescript
// Actions disponibles
canvas('clear');                              // Vider le canvas
canvas('update', { id: 'w_1', data: {...} }); // Mettre à jour un widget
canvas('move', { id: 'w_1', x: 100, y: 200 });
canvas('resize', { id: 'w_1', width: 400, height: 300 });
canvas('style', { id: 'w_1', styles: { color: 'red' } });
```

Chaque action déclenche un callback correspondant :

```typescript
// Dans AgentCallbacks
callbacks.onClear?.();
callbacks.onUpdate?.(id, data);
callbacks.onMove?.(id, x, y);
callbacks.onResize?.(id, width, height);
callbacks.onStyle?.(id, styles);
```

### `list_tools()` / `search_tools()` — Pseudo-outils de découverte

Ces outils ne requièrent **pas d'activation serveur** — c'est de la métadonnée locale :

```typescript
// Appel
search_tools('query=chart')

// Interception AVANT dispatch normal
if (toolMatch && (toolMatch[3] === 'list_tools' || toolMatch[3] === 'search_tools')) {
  // Chercher localement dans layer.tools
  const matches = layer.tools.filter(t =>
    t.name.includes(query) || t.description.includes(query)
  );
  result = JSON.stringify(matches);
  // ← Pas d'appel à MCP client ou WebMCP server
}
```

## Tool Aliases — Résolution canonique

### Problème

L'agent veut appeler `search_recipes`, mais le serveur MCP réel expose `find_templates`.

### Solution

Faire correspondre via **4-layer matching** et enregistrer un alias :

```typescript
// Détection
const matches = resolveCanonicalTools([
  { name: 'find_templates', description: 'Search for workflow templates...' }
]);
// → [{ role: 'search_recipes', realToolName: 'find_templates' }]

// Alias map
aliasMap.set('search_recipes', 'find_templates');

// Dispatch
const resolvedName = aliasMap.get('search_recipes') ?? 'search_recipes';
// → 'find_templates'
```

### Implémentation

```typescript
// Dans loop.ts
const resolvedName = localAliasMap.get(name) ?? name;
const toolMatch = resolvedName.match(/^(.+?)_(mcp|webmcp)_(.+)$/);
const [, serverName, protocol, realToolName] = toolMatch;

// Utiliser realToolName, pas name
const mcpResult = await client.callTool(realToolName, block.input);
```

### Alias map parallèle-safe

Chaque appel `runAgentLoop()` crée sa propre alias map locale :

```typescript
const localAliasMap = new Map<string, string>();

const disc = buildDiscoveryToolsWithAliases(options.layers ?? []);
for (const [k, v] of disc.aliasMap) {
  localAliasMap.set(k, v); // ← Copie locale, pas global
}
```

Ceci évite les conditions de course si plusieurs agents tournent en parallèle.

## Flux complet : Exemple

### Requête utilisateur
```
"Affiche un graphique des 5 meilleures ventes par région"
```

### Étape 1 : Discovery
L'agent appelle `recipes_mcp_search_recipes('ventes par région')`

```
MCP Server "recipes" →
[
  { name: 'region_sales_chart', description: 'Chart top regions...' }
]
```

### Étape 2 : Activation
L'agent appelle `recipes_mcp_get_recipe('region_sales_chart')`

```
MCP Server "recipes" →
{
  name: 'region_sales_chart',
  description: 'Chart top regions...',
  recipe: "## Utilisation\n1. Appeler get_sales_by_region()\n2. Afficher avec chart-rich",
  schema: { type: 'object', required: ['limit'], properties: {...} }
}
```

L'agent active le serveur → tous les outils MCP "recipes" deviennent disponibles.

### Étape 3 : Exécution
L'agent appelle `recipes_mcp_get_sales_by_region({ limit: 5, group_by: 'region' })`

```
MCP Server "recipes" →
{
  regions: [
    { name: 'Nord', sales: 450000 },
    { name: 'Sud', sales: 380000 },
    ...
  ]
}
```

### Étape 4 : Affichage
L'agent appelle `autoui_webmcp_widget_display('chart-rich', {
  type: 'bar',
  labels: ['Nord', 'Sud', ...],
  data: [{ label: 'Ventes', values: [450000, 380000, ...] }]
})`

```
WebMCP Server "autoui" →
{
  widget: 'chart-rich',
  data: { /* validé */ },
  id: 'w_xyz789'
}

callbacks.onWidget('chart-rich', data)
→ Render <Chart spec={data} />
```

## Gestion d'erreurs

### Erreur de validation (schéma)

```typescript
// widget_display('stat', { value: 42 }) — manque 'label'
// → validation error

// L'agent doit :
// 1. Lire le message d'erreur
// 2. Retenteren avec les champs manquants
// 3. Ne pas inventer de nouveaux champs

expected_schema: {
  type: 'object',
  required: ['label', 'value'],
  properties: { label: {...}, value: {...} }
}
```

### Erreur MCP (timeout, crash serveur)

```typescript
// client.callTool('search_recipes', {...}) → erreur réseau
// catch (e) {
//   call.error = e.message;
//   toolResults.push({ type: 'tool_result', tool_use_id: id, content: `Error: ${e.message}` });
// }

// Messages suivants incluent l'erreur → l'agent peut:
// - Réessayer avec d'autres paramètres
// - Utiliser un autre outil
// - Abandonner et afficher un message d'erreur au user
```

### Troncature de résultat

```typescript
// Résultat > 10KB → tronqué
const truncated = truncateResult(rawResult, 10000);

// SI JSON avec array
[{...}, {...}, ...] // 5000 items
// → slicé à 50 items, + suffix "... (tronqué, 5000 items total)"

// Sinon texte brut
// → 10000 premiers caractères + "... (tronqué)"
```

L'agent peut appeler `recall()` pour le résultat complet.
