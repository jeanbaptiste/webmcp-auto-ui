Le déploiement de WebMCP Auto-UI utilise un **script centralisé** (`scripts/deploy.sh`) qui gère les différentes apps avec leurs chemins et configurations spécifiques.

## Règle absolue

**TOUJOURS utiliser le script de déploiement, jamais `scp` ou `rsync` manuellement.**

Le script connaît les chemins corrects et les configurations pour chaque app.

## Infra cible

- **Serveur** : `demo.hyperskills.net` (via SSH alias `ssh bot`)
- **Utilisateur** : `webmcp` (non-root)
- **Base** : `/opt/webmcp-demos/`
- **Ports** : 
  - `3000` : flex (Node.js)
  - `3001`–`3010` : réservés aux futures apps Node
  - `80` / `443` : nginx (reverse proxy)

## Déploiement

### Étape 1 : Build localement

```bash
# À la racine du monorepo
pnpm build
```

Ceci construit toutes les apps en parallèle.

### Étape 2 : Déployer via le script

```bash
# Tout déployer
./scripts/deploy.sh

# Ou une app spécifique
./scripts/deploy.sh flex
./scripts/deploy.sh viewer
./scripts/deploy.sh home
```

### Étape 3 : Vérifier sur le serveur

```bash
ssh bot

# Pour flex (Node.js)
ps aux | grep "node index.js"
curl http://localhost:3000

# Pour nginx (apps statiques)
sudo systemctl status nginx
curl http://localhost/home/
```

## Le script de déploiement

### Ce qu'il fait

```bash
#!/bin/bash

# 1. Vérifie que tu as accès SSH
# 2. Nettoie les anciens build sur la cible (pas de pollution JS)
# 3. Copie les artefacts de build UNIQUEMENT (via rsync)
# 4. Redémarre les services (flex via systemd, nginx reload)
# 5. Vérifie la santé (curl http://localhost)
```

### Structure du script

```bash
# Configuration par app
declare -A APPS=(
  [flex]="apps/flex/.svelte-kit:index.js"
  [viewer]="apps/viewer/build:static"
  [home]="apps/home/dist:static"
  [todo]="apps/todo/build:static"
  [showcase]="apps/showcase/build:static"
  [multi-svelte]="apps/multi/build:static"
)

# Pour chaque app : build → clean → rsync → restart
for app in "${!APPS[@]}"; do
  src=$(echo ${APPS[$app]} | cut -d: -f1)
  type=$(echo ${APPS[$app]} | cut -d: -f2)
  
  # Nettoyer avant copie (évite les chunks JS orphelins)
  ssh bot "rm -rf /opt/webmcp-demos/$app/*"
  
  # Copier artefacts
  rsync -avz --delete "$src/" "webmcp@demo.hyperskills.net:/opt/webmcp-demos/$app/"
  
  # Redémarrer si Node.js
  if [ "$type" = "node" ]; then
    ssh bot "systemctl restart webmcp-$app"
  fi
done

# Nginx reload (pour apps statiques)
ssh bot "systemctl reload nginx"
```

## Variables d'environnement (production)

### Sur le serveur

Les fichiers `.env.production` doivent exister **avant déploiement** :

```bash
# /opt/webmcp-demos/flex/.env
NODE_ENV=production
ANTHROPIC_API_KEY=sk-ant-...
LOG_LEVEL=info

# /opt/webmcp-demos/home/.env  (optionnel pour static, mais utile pour build-time)
PUBLIC_BASE_URL=https://demos.hyperskills.net
PUBLIC_GA_ID=G-XXXXXX
```

**NE JAMAIS** déployer les `.env` — ils existent déjà sur le serveur (créés manuellement une fois).

### Build-time variables (home)

`home` est une app **Astro** qui demande `PUBLIC_BASE_URL` à la **build** :

```bash
# Avant build local
export PUBLIC_BASE_URL=https://demos.hyperskills.net

# Puis build
pnpm build
```

Le script **assume** que c'est déjà défini globalement. Si manquant localement :

```bash
# Fallback
PUBLIC_BASE_URL=https://demos.hyperskills.net pnpm build
```

## Networking (nginx)

**nginx** reverse-proxy chaque app :

```nginx
# /etc/nginx/sites-available/default

# Flex (Node.js → port 3000)
location /flex/ {
  proxy_pass http://localhost:3000/;
}

# Apps statiques → servir depuis /opt/webmcp-demos/
location /viewer/ {
  alias /opt/webmcp-demos/viewer/;
  try_files $uri $uri/index.html =404;
}

location /home/ {
  alias /opt/webmcp-demos/home/;
  try_files $uri $uri/index.html =404;
}

location / {
  alias /opt/webmcp-demos/home/;  # Default = home
  try_files $uri $uri/index.html =404;
}
```

## Rollback

Si un déploiement casse quelque chose :

```bash
# Git history
git log --oneline | head -20

# Checkout version précédente
git checkout <commit_avant>

# Rebuild + redeploy
pnpm build
./scripts/deploy.sh
```

**Attention** : Le script fait un `rm -rf` avant copie, donc les anciens fichiers sont perdus. Le rollback doit être **vite**.

## Dépannage

### App affiche du code old après deploy

**Cause** : Chunks JS orphelins après build. Le script `deploy.sh` fait un `rm -rf` pour éviter ça, mais si tu as déployé manuellement :

**Solution** :
```bash
ssh bot
rm -rf /opt/webmcp-demos/{app}/*.js /opt/webmcp-demos/{app}/*.map
systemctl restart nginx  # ou restart webmcp-flex
```

### `.env` manquant après deploy

**Cause** : Les `.env` ne doivent JAMAIS être en git ou déployés via script.

**Solution** : Recréer manuellement sur serveur
```bash
ssh bot
cat > /opt/webmcp-demos/flex/.env << EOF
NODE_ENV=production
ANTHROPIC_API_KEY=sk-ant-...
EOF
systemctl restart webmcp-flex
```

### Nginx retourne 404

**Cause** : Alias wrong ou `try_files` manquant.

**Debug** :
```bash
ssh bot
# Vérifier le contenu déployé
ls -la /opt/webmcp-demos/home/
ls -la /opt/webmcp-demos/viewer/build/

# Tester nginx config
sudo nginx -t

# Reload
sudo systemctl reload nginx

# Vérifier paths dans config
grep -A 5 "location /home/" /etc/nginx/sites-available/default
```

### API rate-limited (Anthropic)

**Symptôme** : Erreurs 429 de Claude.

**Solution** : 
1. Attendre (backoff exponentiel géré par le client)
2. Vérifier la clé API n'est pas divulguée
3. Augmenter `maxTokens` / réduire `maxIterations`

## Maintenance

### Redémarrer tous les services

```bash
ssh bot
sudo systemctl restart webmcp-flex nginx
```

### Nettoyer les anciens logs

```bash
ssh bot
journalctl --vacuum-time=7d  # Garder 7 jours
```

### Mettre à jour les dépendances

```bash
# Localement
pnpm update
pnpm build

# Déployer
./scripts/deploy.sh
```

## Résumé checklist

- ✅ Sempre utiliser `./scripts/deploy.sh`
- ✅ Jamais déployer `.env` (créer manuellement sur serveur)
- ✅ Build `home` avec `PUBLIC_BASE_URL=https://demos.hyperskills.net`
- ✅ Vérifier que build locales réussit avant déployer
- ✅ Après déployer, tester sur le serveur (curl, browser)
- ✅ Monitorer les logs initiaux (5 min après deploy)
- ✅ Rollback rapide si cassure (git checkout + deploy)
