## Overview

Le projet contient plusieurs apps (flex, viewer, home, showcase, etc.) avec des chemins de déploiement différents. **Toujours utiliser le script `scripts/deploy.sh`** — les chemins sont complexes et varient par app.

## Script de déploiement

### Utilisation

```bash
# Déployer toutes les apps
./scripts/deploy.sh

# Déployer une seule app
./scripts/deploy.sh flex
./scripts/deploy.sh viewer
./scripts/deploy.sh home
```

### Pourquoi pas `scp` / `rsync` manuel ?

1. **Chemins différents par app** — chaque app a une destination différente
2. **Nettoyage critique** — `rsync --delete` a supprimé les `.env` de production
3. **Build préalable** — certaines apps requièrent une build avant deploy
4. **Ordre d'opérations** — systemd restart peut interférer avec les copies de fichiers

## Mapping des apps

| App | Type | ExecStart systemd | Destination deploy | Préparation |
|-----|------|-------------------|--------------------|-------------|
| **flex** | Node.js | `node index.js` | `/opt/webmcp-demos/flex/` | `npm run build` |
| **viewer** | Node.js (SvelteKit) | `node build/index.js` | `/opt/webmcp-demos/viewer/build/` | `npm run build` |
| **home** | Static (nginx) | N/A | `/opt/webmcp-demos/home/` | `npm run build` + `PUBLIC_BASE_URL=...` |
| **showcase** | Static (nginx) | N/A | `/opt/webmcp-demos/showcase/` | `npm run build` |
| **multi-svelte** | Static (nginx) | N/A | `/opt/webmcp-demos/multi-svelte/` | `npm run build` |
| **todo** | Static (nginx) | N/A | `/opt/webmcp-demos/todo/` | `npm run build` |

### **Cas spécial : home** — Variable d'environnement requise

L'app home génère des URLs absolues pour les démos. Elle DOIT être buildée avec :

```bash
PUBLIC_BASE_URL=https://demos.hyperskills.net npm run build
```

Sans cette variable, les URLs seront `http://localhost:5173/...` en production.

## Process de déploiement détaillé

### 1. Build locale

```bash
# À la racine du monorepo
npm install
npm run build

# Ou pour une seule app
cd apps/flex
npm run build
```

### 2. Exécuter le script de déploiement

```bash
./scripts/deploy.sh flex
```

Le script :
- ✅ Vérifie que le build existe
- ✅ Nettoie l'ancienne destination (sauf `.env`)
- ✅ Copie les fichiers neufs
- ✅ Redémarre systemd si c'est un service Node.js
- ✅ Affiche le statut final

### 3. Vérifier le déploiement

```bash
# Accéder à l'app
curl https://demos.hyperskills.net/flex

# Vérifier les logs (si Node.js)
ssh bot systemctl status webmcp-demos-flex
ssh bot journalctl -u webmcp-demos-flex -f

# Ou pour une app statique
ssh bot ls -la /opt/webmcp-demos/home/
```

## Règles critiques

### ❌ Ne JAMAIS

1. **Utiliser `rsync --delete` directement** — supprime les `.env` de prod
   ```bash
   # ❌ MAUVAIS
   rsync -av --delete build/ bot:/opt/webmcp-demos/flex/
   ```

2. **Copier `.env` depuis local** — écrase les clés API de prod
   ```bash
   # ❌ MAUVAIS
   scp .env.local bot:/opt/webmcp-demos/flex/.env
   ```

3. **Déployer sans nettoyer les anciens chunks JS**
   ```bash
   # ❌ MAUVAIS (les chunks anciens restent)
   scp -r build/* bot:/opt/webmcp-demos/flex/
   ```

4. **Modifier directement le répertoire de déploiement**
   ```bash
   # ❌ MAUVAIS
   ssh bot "cd /opt/webmcp-demos/flex && npm install"
   ```

### ✅ TOUJOURS

1. **Utiliser `./scripts/deploy.sh`**
   ```bash
   # ✅ BON
   ./scripts/deploy.sh flex
   ```

2. **Builder en local avant de déployer**
   ```bash
   npm run build
   ./scripts/deploy.sh flex
   ```

3. **Vérifier l'accès avant de déployer**
   ```bash
   ssh bot "ls /opt/webmcp-demos/flex/.env"
   # Si "No such file", créer depuis template
   ```

4. **Garder les `.env` et secrets en place**
   ```bash
   # Le script préserve les .env existants
   ```

## Dépannage

### Erreur : "Old app still running"

L'ancien code s'exécute toujours même après déploiement.

**Causes** :
- Chunks JS anciens non nettoyés
- Cache systemd/nginx
- Port encore écouté par ancien process

**Solutions** :
```bash
# Forcer restart systemd
ssh bot systemctl restart webmcp-demos-flex

# Ou (static) recharger nginx
ssh bot systemctl reload nginx

# Vérifier le port
ssh bot lsof -i :3000
```

### Erreur : ".env not found"

L'app ne démarre pas parce que `.env` est manquant.

**Solution** : Créer depuis le serveur
```bash
ssh bot "cat > /opt/webmcp-demos/flex/.env" << EOF
VITE_ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=...
EOF
```

Ou copier un template sécurisé (ne JAMAIS copier depuis `.env.local`).

### Erreur : "Cannot GET /flex"

nginx retourne 404.

**Causes** :
- App statique non buildée
- Chemin nginx incorrect
- Cache vieux

**Solutions** :
```bash
# Vérifier que l'app est builtée
ls -la apps/home/dist/

# Vérifier nginx config
ssh bot "grep -A5 '/home' /etc/nginx/sites-available/demos.hyperskills.net"

# Purger cache
ssh bot "systemctl reload nginx"
```

## Checklist avant déploiement

- [ ] Code commité et poussé
- [ ] Tests locaux passent (`npm run test`)
- [ ] Build local réussit (`npm run build`)
- [ ] Variables d'env correctes
  - [ ] `PUBLIC_BASE_URL=https://demos.hyperskills.net` pour home
  - [ ] Autres apps sans restriction
- [ ] Fichier `.env` existe sur le serveur
- [ ] Espace disque suffisant : `ssh bot df -h /opt`
- [ ] Aucune autre personne en train de déployer

## Script de déploiement complet

```bash
#!/bin/bash
# scripts/deploy.sh — Déploiement sécurisé multi-app

APP="${1:-all}"
HOST="bot"
BASE_PATH="/opt/webmcp-demos"

case "$APP" in
  flex)
    BUILD_DIR="apps/flex/dist"
    DEST_PATH="$BASE_PATH/flex"
    SERVICE="webmcp-demos-flex"
    ;;
  viewer)
    BUILD_DIR="apps/viewer/build"
    DEST_PATH="$BASE_PATH/viewer/build"
    SERVICE="webmcp-demos-viewer"
    ;;
  home)
    BUILD_DIR="apps/home/dist"
    DEST_PATH="$BASE_PATH/home"
    SERVICE=""
    ;;
  *)
    echo "Usage: $0 {flex|viewer|home|showcase|all}"
    exit 1
    ;;
esac

# Vérifier le build
if [ ! -d "$BUILD_DIR" ]; then
  echo "❌ Build non trouvé: $BUILD_DIR"
  exit 1
fi

echo "📦 Déploiement de $APP..."

# Copier (préserver .env et secrets)
ssh "$HOST" "mkdir -p $DEST_PATH && rm -rf $DEST_PATH/* && ([ -f $DEST_PATH/.env ] && mv $DEST_PATH/.env /tmp/.env.bak)"
scp -r "$BUILD_DIR/"* "$HOST:$DEST_PATH/" 2>/dev/null
ssh "$HOST" "[ -f /tmp/.env.bak ] && mv /tmp/.env.bak $DEST_PATH/.env"

# Redémarrer si c'est un service Node.js
if [ -n "$SERVICE" ]; then
  ssh "$HOST" "systemctl restart $SERVICE && sleep 2 && systemctl is-active $SERVICE"
else
  ssh "$HOST" "systemctl reload nginx"
fi

echo "✅ $APP déployé avec succès!"
```
