Démarrez avec WebMCP Auto-UI en 5 minutes.

## Prérequis

- **Node.js** 18.x ou supérieur
- **npm** (le monorepo utilise `package-lock.json`)
- **Claude API key** (optionnel — Gemma in-browser est disponible)

## Installation

### Chemin rapide : Boilerplate

Le moyen le plus simple de démarrer est le **boilerplate** — une app SvelteKit prête à l'emploi avec 3 widgets Tricoteuses :

```bash
npx degit jeanbaptiste/webmcp-auto-ui/apps/boilerplate my-app
cd my-app
npm install
npm run dev
```

L'app démarre sur `http://localhost:5173` avec un canvas, un chat panel et trois widgets fonctionnels.

### Monorepo complet (pour contribuer)

#### 1. Cloner le dépôt

```bash
git clone https://github.com/jeanbaptiste/webmcp-auto-ui.git
cd webmcp-auto-ui
```

#### 2. Installer les dépendances

```bash
npm install
```

#### 3. Configuration des variables d'environnement

Créez un fichier `.env.local` à la racine :

```bash
# API Claude (optionnel, requis pour les agents distants)
ANTHROPIC_API_KEY=sk-ant-...

# URL de serveurs MCP distants (format: protocol://host:port)
MCP_SERVERS=http://localhost:3001,http://localhost:3002
```

#### 4. Lancer en développement

```bash
npm run dev
```

L'app démarre sur `http://localhost:5173`. Vous verrez :

- **Canvas** : Zone de dessin des widgets
- **Chat panel** : Interface agent
- **Settings** : Sélecteur modèle LLM (Claude/Gemma)

## Premier agent simple

### Étape 1 : Choisir un modèle

1. Cliquez sur **Settings**
2. Sélectionnez **Claude (Haiku)** ou **Gemma (in-browser)**
3. Si Claude : vérifiez que `ANTHROPIC_API_KEY` est définie

### Étape 2 : Écrire un prompt

Dans le **Chat panel**, entrez :

```
Affiche un statut avec le label "Visiteurs" et la valeur "1,234"
```

### Étape 3 : Observer l'agent

1. L'agent reçoit le prompt
2. Identifie qu'il faut un widget `stat`
3. Appelle `widget_display('stat', { label: 'Visiteurs', value: '1,234' })`
4. Le canvas affiche le widget

## Architecture de développement

### Monorepo structure

```
webmcp-auto-ui/
├── packages/
│   ├── core/          # Types, polyfill WebMCP, MCP client
│   ├── agent/         # Agent loop, LLM providers, tool layers
│   ├── ui/            # Widgets Svelte (30+), composants UI
│   └── sdk/           # HyperSkills, canvas store
├── apps/
│   ├── boilerplate/   # Point d'entrée nouveaux devs (SvelteKit + Tricoteuses)
│   ├── flex2/         # App SvelteKit principale
│   ├── showcase2/     # Démo showcase
│   ├── todo2/         # Démo todo
│   ├── viewer2/       # Visionneur HyperSkills
│   ├── home/          # Accueil (static)
│   ├── mobile/        # App mobile
│   ├── multi-svelte/  # Multi-widgets Svelte
│   ├── recipes/       # Démo recettes
│   └── wasm/          # Démo Gemma WASM
└── scripts/
    └── deploy.sh      # Script de déploiement
```

### Imports recommandés

```typescript
// Agent loop
import { runAgentLoop } from '@webmcp-auto-ui/agent';

// Providers LLM
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';
import { WasmProvider } from '@webmcp-auto-ui/agent';

// Tool layers
import { buildDiscoveryTools, activateServerTools, resolveCanonicalTools } from '@webmcp-auto-ui/agent';

// Widgets

// Canvas store
import { canvas } from '@webmcp-auto-ui/sdk/canvas';

// MCP Client
import { McpClient } from '@webmcp-auto-ui/core';
```

## Exemples courants

### Créer un provider Claude

```typescript
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({
  proxyUrl: '/api/chat',
  model: 'sonnet',  // résout en claude-sonnet-4-6
});
```

Modèles disponibles : `'haiku'` (claude-haiku-4-5-20251001), `'sonnet'` (claude-sonnet-4-6), `'opus'` (claude-opus-4-6).

### Créer un provider Gemma

```typescript
import { WasmProvider } from '@webmcp-auto-ui/agent';

const provider = new WasmProvider({
  modelId: 'gemma-e4b', // 4B variant
  autoDownload: true,
});

// Attendre le chargement
provider.onStatusChange((status) => {
  console.log('Gemma status:', status);
});
```

### Lancer un agent

```typescript
import { runAgentLoop } from '@webmcp-auto-ui/agent';

const result = await runAgentLoop('Affiche un graphique de ventes', {
  provider,
  layers: [
    // ... tool layers
  ],
  maxIterations: 5,
  callbacks: {
    onToolCall: (call) => console.log('Tool called:', call.name),
    onWidget: (type, data) => console.log('Widget:', type),
  },
});

console.log('Agent finished:', result.text);
```

### Afficher un widget

```svelte
<script>
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';
  
  const widgetData = {
    label: 'Visiteurs',
    value: '1,234',
    trend: 'up',
  };
</script>

<WidgetRenderer 
  type="stat-card" 
  data={widgetData}
  servers={[autoui]}
/>
```

## Connexion à un serveur MCP

### Lancer un serveur MCP local

```bash
# Exemple : serveur MCP météo
npx @example/weather-mcp
```

Doit écouter sur `http://localhost:3000` et exposer un endpoint `/sse` (Server-Sent Events).

### Connecter depuis l'app

```typescript
import { McpClient } from '@webmcp-auto-ui/core';

const client = new McpClient({
  serverUrl: 'http://localhost:3000',
});

await client.initialize();

// Lister les outils
const tools = await client.listTools();
console.log('MCP tools:', tools);
```

## Déploiement local

Pour tester sur un serveur local avant production :

```bash
# Build la production
npm run build

# Lancer le serveur
npm run preview
```

Accessible sur `http://localhost:4173`.

## Dépannage

### L'agent n'appelle pas les outils

- Vérifiez que les `layers` sont configurées correctement
- Vérifiez que `buildDiscoveryTools()` retourne les outils attendus
- Consultez les logs du provider LLM

### Gemma ne charge pas

- Vérifiez la connexion internet (télécharge ~6GB de WASM)
- Vérifiez le navigateur (Chrome/Edge recommandés)
- Consultez la console dev (`F12`)

### Widget ne s'affiche pas

- Vérifiez que le type existe dans le `NATIVE_MAP` de `WidgetRenderer`
- Vérifiez la structure des données vs `inputSchema` du widget
- Consultez les erreurs de validation
