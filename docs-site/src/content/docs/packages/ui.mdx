Le package `@webmcp-auto-ui/ui` fournit une bibliothèque complète de composants Svelte 5 pour afficher les widgets, gérer la UI et orchestrer l'agent LLM.

## Rendu de Widgets

### WidgetRenderer

Composant principal qui résout et affiche un widget (natif, custom serveur, ou fallback).

```svelte
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';
  import { autoui } from '@webmcp-auto-ui/agent';

  const servers = [autoui];
</script>

<WidgetRenderer
  id="block_1"
  type="stat"
  data={{ label: 'Revenue', value: '$42k' }}
  {servers}
  oninteract={(type, action, payload) => console.log(action, payload)}
/>
```

**Props:**

```ts
interface Props {
  id?: string;                                      // Widget ID (auto-généré)
  type: string;                                     // Type de widget
  data: Record<string, unknown>;                    // Données du widget
  servers?: WebMcpServer[];                         // Serveurs WebMCP (pour custom widgets)
  oninteract?: (type: string, action: string, payload: unknown) => void;
}
```

**Actions d'interaction:**

- `itemclick` — Clique sur un item de liste
- `rowclick` — Clique sur une ligne tableau
- `cardclick` — Clique sur une carte
- `imageclick` — Clique sur une image (gallery)
- `slidechange` — Changement de slide (carousel)
- `cellclick` — Clique sur cellule (grid-data)

**Auto-enregistrement WebMCP:**

Le widget s'enregistre automatiquement 3 tools sur `navigator.modelContext`:
- `widget_{id}_get()` — obtenir les données actuelles
- `widget_{id}_update(...)` — mettre à jour les données
- `widget_{id}_remove()` — supprimer le widget

```ts
// L'agent peut appeler ces tools sans configuration supplémentaire
onWidget: (type, data) => {
  // L'agent peut ensuite appeler widget_<id>_update(newData)
}
```

### BlockRenderer

Alias hérité pour `WidgetRenderer`. Utiliser `WidgetRenderer` préférablement.

```svelte
<BlockRenderer type="stat" data={{label: 'Sales', value: '100'}} />
```

## Contrôles Agent

### LLMSelector

Sélecteur de modèle LLM avec support remote (Claude) et WASM (Gemma).

```svelte
<script lang="ts">
  import { LLMSelector } from '@webmcp-auto-ui/ui';
  import { canvas } from '@webmcp-auto-ui/sdk';

  function onSelect(model: string) {
    canvas.llm = model;
  }
</script>

<LLMSelector value={canvas.llm} onchange={onSelect} />
```

**Props:**

```ts
interface Props {
  value?: string;                                   // Modèle sélectionné
  disabled?: boolean;                              // Désactiver
  onchange?: (model: string) => void;             // Callback sélection
}
```

**Modèles supportés:**
- `haiku`, `sonnet`, `opus` — Anthropic Claude
- `gemma-e2b`, `gemma-e4b` — Google Gemma (WASM)

### GemmaLoader

Indicateur de chargement pour Gemma WASM (avant exécution agent).

```svelte
<script lang="ts">
  import { GemmaLoader } from '@webmcp-auto-ui/ui';
  import { WasmProvider } from '@webmcp-auto-ui/agent';

  const provider = new WasmProvider({
    modelSize: '2b',
    quantized: true,
  });
</script>

{#if !provider.ready}
  <GemmaLoader provider={provider} />
{:else}
  <!-- Afficher l'app -->
{/if}
```

**Props:**

```ts
interface Props {
  provider: WasmProvider;
  title?: string;                                   // Titre (défaut: "Gemma Loading")
}
```

Affiche une barre de progression animée + statut de chargement.

### McpStatus

Indicateur de connexion MCP (statut + nombre d'outils).

```svelte
<script lang="ts">
  import { McpStatus } from '@webmcp-auto-ui/ui';
  import { canvas } from '@webmcp-auto-ui/sdk';
</script>

<McpStatus
  connected={canvas.mcpConnected}
  connecting={canvas.mcpConnecting}
  name={canvas.mcpName}
  toolCount={canvas.mcpTools.length}
  onconnect={() => { /* lancer connexion */ }}
/>
```

**Props:**

```ts
interface Props {
  connected: boolean;
  connecting: boolean;
  name?: string;
  toolCount?: number;
  onconnect?: () => void;
}
```

Affiche:
- 🟢 Connecté (+ nom + nb tools)
- 🟡 Connexion en cours
- 🔴 Déconnecté
- 🔴 Erreur

### AgentProgress

Barre de progression et métriques de l'agent (tokens, latence, itérations).

```svelte
<script lang="ts">
  import { AgentProgress } from '@webmcp-auto-ui/ui';
</script>

<AgentProgress
  iteration={2}
  maxIterations={5}
  tokens={{input: 450, output: 120}}
  latencyMs={850}
/>
```

**Props:**

```ts
interface Props {
  iteration?: number;                              // Itération actuelle
  maxIterations?: number;                          // Max itérations
  tokens?: { input: number; output: number };     // Tokens utilisés
  latencyMs?: number;                              // Latence dernière requête
  thinkingTokens?: number;                         // Tokens de réflexion
}
```

Affiche une barre horizontale animée + informations détaillées.

### McpConnector

Interface interactive de connexion MCP (URL input + bouton connexion).

```svelte
<script lang="ts">
  import { McpConnector } from '@webmcp-auto-ui/ui';
  import { canvas } from '@webmcp-auto-ui/sdk';

  async function handleConnect(url: string) {
    // Appeler votre logique de connexion MCP
    canvas.mcpUrl = url;
    canvas.setMcpConnecting(true);
  }
</script>

<McpConnector
  url={canvas.mcpUrl}
  connecting={canvas.mcpConnecting}
  onconnect={handleConnect}
/>
```

**Props:**

```ts
interface Props {
  url?: string;
  connecting?: boolean;
  oninput?: (url: string) => void;
  onconnect?: (url: string) => void;
}
```

## Layouts

### TilingLayout

Layout en tuiles (grille) pour organiser les widgets côte à côte.

```svelte
<script lang="ts">
  import { TilingLayout } from '@webmcp-auto-ui/ui';
</script>

<TilingLayout gap="4" columns={3}>
  <div>Widget 1</div>
  <div>Widget 2</div>
  <div>Widget 3</div>
</TilingLayout>
```

**Props:**

```ts
interface Props {
  gap?: string;                                    // Espace entre items (défaut: '2')
  columns?: number;                                // Nb colonnes (défaut: auto)
  rows?: number;                                   // Nb lignes
}
```

## SafeImage

Composant image robuste avec validation d'URL et fallback.

```svelte

</script>

<SafeImage
  src="https://example.com/image.jpg"
  alt="Description"
  fallbackText="Image"
  hideOnError={false}
/>
```

**Props:**

```ts
interface Props {
  src: string | undefined | null;
  alt?: string;
  fallbackText?: string;        // Texte par défaut
  hideOnError?: boolean;        // Masquer si erreur
}
```

Valide les URL (http://, https://, data:, /) et affiche un placeholder si invalide/cassée.

## FONC Message Bus

Système de messagerie inter-composants basé sur le bus de messages.

```svelte
<script lang="ts">
  import { bus } from '@webmcp-auto-ui/ui';
  
  // S'enregistrer sur le bus
  const unregister = bus.register(
    'my_component',
    'widget',
    ['update', 'interact', '*'],
    (msg) => console.log(msg)
  );

  // Envoyer un message
  bus.broadcast('my_component', 'update', { newValue: 42 });

  // Nettoyer
  onDestroy(() => unregister());
</script>
```

**API bus:**

```ts
bus.register(id, type, channels, handler): () => void
bus.send(from, to, channel, payload): void
bus.broadcast(from, channel, payload): void
bus.subscribe(channels, handler): () => void
bus.link(widgetIds, groupId?): string
bus.unlink(widgetId, groupId): void
bus.getLinks(widgetId): string[]
bus.getGroup(groupId): string[]
bus.hasLinks(widgetId): boolean
```

## Exemples Complets

### Chat avec Agent

```svelte
<script lang="ts">
  import { LLMSelector, AgentProgress, McpStatus } from '@webmcp-auto-ui/ui';
  import { canvas } from '@webmcp-auto-ui/sdk';
  import { runAgentLoop, RemoteLLMProvider } from '@webmcp-auto-ui/agent';

  let input = '';
  let loading = false;

  const provider = new RemoteLLMProvider({
    proxyUrl: '/api/chat',
    model: 'sonnet',  // résout en claude-sonnet-4-6
  });

  async function chat() {
    loading = true;
    try {
      const result = await runAgentLoop(input, {
        provider,
        layers: [],
        callbacks: {
          onToken: (token) => {
            // Streaming en temps réel
          },
        },
      });
      canvas.addMsg('assistant', result.text);
    } finally {
      loading = false;
    }
  }
</script>

<div class="p-4">
  <LLMSelector bind:value={canvas.llm} />
  <McpStatus
    connected={canvas.mcpConnected}
    name={canvas.mcpName}
  />
  <input bind:value={input} />
  <button disabled={loading} onclick={chat}>Envoyer</button>
  {#if loading}
    <AgentProgress iteration={1} maxIterations={5} />
  {/if}
</div>
```
