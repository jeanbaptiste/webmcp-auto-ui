Le package `@webmcp-auto-ui/ui` fournit une bibliothèque complète de composants Svelte 5 pour afficher les widgets, gérer la UI et orchestrer l'agent LLM.

## Theme

### ThemeProvider

Fournit le contexte de theme (dark/light) a tous les composants enfants.

```svelte
<script lang="ts">
  import { ThemeProvider } from '@webmcp-auto-ui/ui';
</script>

<ThemeProvider defaultMode="dark">
  <!-- Tous les composants enfants ont acces au theme -->
  <slot />
</ThemeProvider>
```

**Props:**

```ts
interface Props {
  defaultMode?: ThemeMode;                          // 'light' | 'dark' (defaut: 'light')
  overrides?: Record<string, string>;               // CSS custom properties
  theme?: ThemeJSON;                                // Theme JSON externe
  children: Snippet;
}
```

### getTheme

Fonction contexte Svelte qui retourne l'API de theme depuis n'importe quel composant enfant de `ThemeProvider`.

```svelte
<script lang="ts">
  import { getTheme } from '@webmcp-auto-ui/ui';

  const theme = getTheme();
</script>

<span>Mode actuel : {theme.mode}</span>
<button onclick={theme.toggle}>Basculer le theme</button>
```

**API retournee:**

```ts
interface ThemeAPI {
  readonly mode: ThemeMode;                         // 'light' | 'dark'
  toggle: () => void;                               // Bascule dark/light
  setMode: (m: ThemeMode) => void;                  // Definit le mode
}
```

### Tokens et constantes

```ts
import { DARK_TOKENS, LIGHT_TOKENS, THEME_MAP } from '@webmcp-auto-ui/ui';
import type { ThemeMode, ThemeOverrides, ThemeTokens } from '@webmcp-auto-ui/ui';
```

- `DARK_TOKENS` / `LIGHT_TOKENS` — objets `Record<string, string>` avec les CSS custom properties par mode
- `THEME_MAP` — `{ light: ThemeTokens, dark: ThemeTokens }` pour basculer programmatiquement

## Rendu de Widgets

### WidgetRenderer

Composant principal qui résout et affiche un widget (natif, custom serveur, ou fallback).

```svelte
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';
  import { autoui } from '@webmcp-auto-ui/agent';

  const servers = [autoui];
</script>

<WidgetRenderer
  id="block_1"
  type="stat"
  data={{ label: 'Revenue', value: '$42k' }}
  {servers}
  oninteract={(type, action, payload) => console.log(action, payload)}
/>
```

**Props:**

```ts
interface Props {
  id?: string;                                      // Widget ID (auto-genere)
  type: string;                                     // Type de widget
  data: Record<string, unknown>;                    // Donnees du widget
  servers?: WebMcpServer[];                         // Serveurs WebMCP (pour custom widgets)
  oninteract?: (type: string, action: string, payload: unknown) => void;
}
```

**Actions d'interaction:**

- `itemclick` — Clique sur un item de liste
- `rowclick` — Clique sur une ligne tableau
- `cardclick` — Clique sur une carte
- `imageclick` — Clique sur une image (gallery)
- `slidechange` — Changement de slide (carousel)
- `cellclick` — Clique sur cellule (grid-data)

**Auto-enregistrement WebMCP:**

Le widget s'enregistre automatiquement 3 tools sur `navigator.modelContext`:
- `widget_{id}_get()` — obtenir les donnees actuelles
- `widget_{id}_update(...)` — mettre a jour les donnees
- `widget_{id}_remove()` — supprimer le widget

```ts
// L'agent peut appeler ces tools sans configuration supplementaire
onWidget: (type, data) => {
  // L'agent peut ensuite appeler widget_<id>_update(newData)
}
```

### BlockRenderer

Alias herite pour `WidgetRenderer`. Utiliser `WidgetRenderer` preferablement.

```svelte
<BlockRenderer type="stat" data={{label: 'Sales', value: '100'}} />
```


- `haiku`, `sonnet`, `opus` — Anthropic Claude
- `gemma-e2b`, `gemma-e4b` — Google Gemma (WASM)

### FloatingLayout

Layout flottant avec fenetres deplacables et redimensionnables. Utilise un snippet `children` qui recoit un contexte avec des callbacks de drag/resize.

```svelte
<script lang="ts">
  import { FloatingLayout } from '@webmcp-auto-ui/ui';
  import type { ManagedWindow } from '@webmcp-auto-ui/ui';

  let fl: FloatingLayout;
  let windows = $state<ManagedWindow[]>([]);
</script>

<FloatingLayout bind:this={fl} {windows} defaultWidth={380} defaultHeight={280}>
  {#snippet children(win, _lw, ctx)}
    <div class="bg-surface rounded-lg border border-border">
      <div onmousedown={(e) => ctx.ondragstart(e)}
           ondblclick={() => ctx.ontogglecollapse()}>
        {win.title}
      </div>
      {#if !ctx.collapsed}
        <div>Contenu du widget {win.id}</div>
      {/if}
      <div onmousedown={(e) => ctx.onresizestart(e)}></div>
    </div>
  {/snippet}
</FloatingLayout>
```

**Props:**

```ts
interface Props {
  windows: ManagedWindow[];
  gap?: number;                                    // Defaut: 4
  defaultWidth?: number;                           // Defaut: 400
  defaultHeight?: number;                          // Defaut: 300
  onmove?: (id: string, x: number, y: number) => void;
  onresize?: (id: string, w: number, h: number) => void;
  children: Snippet<[ManagedWindow, LayoutWindow, SnippetCtx]>;
}
```

**Contexte snippet (`SnippetCtx`):**

```ts
interface SnippetCtx {
  ondragstart: (e: MouseEvent) => void;
  ontogglecollapse: () => void;
  onfittocontent: () => void;
  onresizestart: (e: MouseEvent) => void;
  collapsed: boolean;
}
```

**Methodes exposees (via `bind:this`):**

```ts
fl.move(id: string, x: number, y: number): void
fl.resize(id: string, w: number, h: number): void
fl.toggleCollapse(id: string): void
fl.fitToContent(id: string): void
```

### layoutAdapter

Singleton qui connecte les callbacks de layout (move, resize, style) aux outils de l'agent (`move_block`, `resize_block`, `style_block`).

```ts
import { layoutAdapter } from '@webmcp-auto-ui/ui';

// Enregistrer les callbacks (typiquement dans onMount)
layoutAdapter.register({
  move:   (id, x, y) => fl?.move(id, x, y),
  resize: (id, w, h) => fl?.resize(id, w, h),
  style:  (id, styles) => {
    const el = document.querySelector(`[data-block-id="${id}"]`);
    if (el) for (const [k, v] of Object.entries(styles))
      el.style.setProperty(`--color-${k}`, v);
  },
});

// Desenregistrer (dans onDestroy)
layoutAdapter.unregister();
```

**API:**

```ts
layoutAdapter.register(cbs: {
  move?: (id: string, x: number, y: number) => void;
  resize?: (id: string, w: number, h: number) => void;
  style?: (id: string, styles: Record<string, string>) => void;
}): void

layoutAdapter.unregister(): void
layoutAdapter.move(id: string, x: number, y: number): void
layoutAdapter.resize(id: string, w: number, h: number): void
layoutAdapter.style(id: string, styles: Record<string, string>): void
```

## FONC Message Bus

Systeme de messagerie inter-composants base sur le bus de messages.

```svelte
<script lang="ts">
  import { bus } from '@webmcp-auto-ui/ui';
  
  // S'enregistrer sur le bus
  const unregister = bus.register(
    'my_component',
    'widget',
    ['update', 'interact', '*'],
    (msg) => console.log(msg)
  );

  // Envoyer un message
  bus.broadcast('my_component', 'update', { newValue: 42 });

  // Nettoyer
  onDestroy(() => unregister());
</script>
```

**API bus:**

```ts
bus.register(id, type, channels, handler): () => void
bus.send(from, to, channel, payload): void
bus.broadcast(from, channel, payload): void
bus.subscribe(channels, handler): () => void
bus.link(widgetIds, groupId?): string
bus.unlink(widgetId, groupId): void
bus.getLinks(widgetId): string[]
bus.getGroup(groupId): string[]
bus.hasLinks(widgetId): boolean
```

## Exemples Complets

### Chat avec Agent

```svelte
<script lang="ts">
  import { LLMSelector, AgentProgress, McpStatus } from '@webmcp-auto-ui/ui';
  import { canvas } from '@webmcp-auto-ui/sdk';
  import { runAgentLoop, RemoteLLMProvider } from '@webmcp-auto-ui/agent';

  let input = '';
  let loading = false;

  const provider = new RemoteLLMProvider({
    proxyUrl: '/api/chat',
    model: 'sonnet',  // resout en claude-sonnet-4-6
  });

  async function chat() {
    loading = true;
    try {
      const result = await runAgentLoop(input, {
        provider,
        layers: [],
        callbacks: {
          onToken: (token) => {
            // Streaming en temps reel
          },
        },
      });
      canvas.addMsg('assistant', result.text);
    } finally {
      loading = false;
    }
  }
</script>

<div class="p-4">
  <LLMSelector bind:value={canvas.llm} />
  <McpStatus
    connected={canvas.mcpConnected}
    name={canvas.mcpName}
  />
  <input bind:value={input} />
  <button disabled={loading} onclick={chat}>Envoyer</button>
  {#if loading}
    <AgentProgress iteration={1} maxIterations={5} />
  {/if}
</div>
```
