Le package `@webmcp-auto-ui/agent` contient la boucle agent, les providers LLM, la gestion des couches d'outils (tool layers), et l'exécution de recettes.

## API principale : `runAgentLoop`

```ts
export async function runAgentLoop(
  userMessage: string,
  options: AgentLoopOptions
): Promise<AgentResult>
```

Lance la boucle agent : LLM → appels d'outils → LLM jusqu'à `end_turn` ou nombre max d'itérations.

### Options

```ts
export interface AgentLoopOptions {
  client?: McpClient;           // Client MCP (optionnel si tools WebMCP seules)
  provider: LLMProvider;         // Provider LLM (Anthropic, Gemma WASM, local)
  layers?: ToolLayer[];          // Couches d'outils structurées (replaces legacy mcpTools)
  maxIterations?: number;        // Défaut: 5
  maxTokens?: number;            // Limite optionnelle sur réponses
  maxTools?: number;             // Nombre max d'outils en parallèle
  temperature?: number;          // Température LLM
  topK?: number;                 // Top-K sampling
  cacheEnabled?: boolean;        // Utiliser cache d'invite (Claude 3.5+)
  systemPrompt?: string;         // Prompt système custom
  initialMessages?: ChatMessage[]; // Historique de conversation
  callbacks?: AgentCallbacks;    // Événements de streaming
  signal?: AbortSignal;          // Annulation
  truncateResults?: boolean;     // Tronquer résultats outils (défaut: true)
  compressHistory?: boolean;     // Compresser anciens résultats (défaut: true)
  maxResultLength?: number;      // Taille max résultat tronqué (défaut: 10000)
}
```

### Résultat

```ts
export interface AgentResult {
  text: string;                  // Dernier texte produit
  toolCalls: ToolCall[];         // Tous les appels d'outils
  metrics: AgentMetrics;         // Usage tokens, latence, nombre itérations
  stopReason: 'end_turn' | 'max_iterations';
  messages: ChatMessage[];       // Conversation complète
}
```

### Callbacks

```ts
export interface AgentCallbacks {
  onIterationStart?: (iteration: number, maxIterations: number) => void;
  onLLMRequest?: (messages: ChatMessage[], tools: ProviderTool[]) => void;
  onLLMResponse?: (response: LLMResponse, latencyMs: number, tokens?: { input: number; output: number }) => void;
  onToolCall?: (call: ToolCall) => void;
  onWidget?: (type: string, data: Record<string, unknown>) => { id: string } | void;
  onText?: (text: string) => void;
  onToken?: (token: string) => void;
  onDone?: (metrics: AgentMetrics) => void;
  // Canvas mutations
  onClear?.();
  onUpdate?(id: string, data: Record<string, unknown>): void;
  onMove?(id: string, x: number, y: number): void;
  onResize?(id: string, w: number, h: number): void;
  onStyle?(id: string, styles: Record<string, string>): void;
  onUserInteraction?(widgetId: string, widgetType: string, action: string, payload: unknown): void;
}
```

### Exemple

```ts
import { runAgentLoop } from '@webmcp-auto-ui/agent';
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({
  apiKey: process.env.ANTHROPIC_API_KEY,
  model: 'claude-3-5-sonnet-20241022',
});

const result = await runAgentLoop('Montre-moi les recettes disponibles', {
  provider,
  layers: [...],
  callbacks: {
    onText: (text) => console.log(text),
    onToolCall: (call) => console.log('Outil:', call.name),
    onDone: (metrics) => console.log('Tokens:', metrics.totalTokens),
  },
});

console.log(result.text);
```

## System Prompt & Alias

### `buildSystemPrompt` / `buildSystemPromptWithAliases`

```ts
export function buildSystemPrompt(layers: ToolLayer[]): string

export interface SystemPromptResult {
  prompt: string;
  aliasMap: Map<string, string>;  // {serverName}_{protocol}_{canonical} → réel
}
export function buildSystemPromptWithAliases(layers: ToolLayer[]): SystemPromptResult
```

Le prompt système liste :
- **Étape 1** : Recherche de recettes par mot-clé
- **Étape 1b** : Liste des recettes disponibles
- **Étape 1c** : Recherche d'outils
- **Étape 1d** : Liste des outils
- **Étape 2** : Lecture détaillée de la recette sélectionnée
- **Étape 3** : Exécution des instructions / appels outils
- **Étape 4** : Affichage UI (widget_display, canvas, recall)

### Résolution canonique (4 couches)

Les outils MCP sont normalisés en rôles canoniques (search_recipes, list_recipes, get_recipe) via `resolveCanonicalTools` :

```ts
export function resolveCanonicalTools(tools: McpToolDef[]): CanonicalMatch[]

interface CanonicalMatch {
  role: 'search_recipes' | 'list_recipes' | 'get_recipe';
  realToolName: string;
}
```

**Couches de matching :**
1. Nom exact (search_recipes, list_recipes, get_recipe)
2. Décomposition token : (action, ressource) → (search|list|get, recipe|template|...)
3. Mots-clés description : Cherche keywords (recipe, skill, template, workflow)
4. Fallback : Aucun match, lister les outils bruts

### Compression automatique

La boucle agent compresse les anciens résultats d'outils (> 300 chars) en préviews de 200 chars + hint recall(). Active par défaut (option `compressHistory: true`).

## Types

```ts
export interface ChatMessage {
  role: 'user' | 'assistant' | 'system';
  content: string | ContentBlock[];
}

export type ContentBlock =
  | { type: 'text'; text: string }
  | { type: 'tool_use'; id: string; name: string; input: Record<string, unknown> }
  | { type: 'tool_result'; tool_use_id: string; content: string };

export interface ToolCall {
  id: string;
  name: string;
  args: Record<string, unknown>;
  result?: string;
  error?: string;
  elapsed?: number;
  guided?: boolean;  // Précédé d'un outil découverte
}

export interface AgentMetrics {
  totalTokens: number;
  promptTokens: number;
  completionTokens: number;
  totalLatencyMs: number;
  toolCalls: number;
  iterations: number;
  cacheHits: number;
}

export interface LLMResponse {
  content: ContentBlock[];
  stopReason: string;
  usage?: {
    input_tokens: number;
    output_tokens: number;
    cache_read_input_tokens?: number;
  };
  stats?: {
    tokensPerSec: number;
    totalTokens: number;
    latencyMs: number;
  };
}

export interface LLMProvider {
  readonly name: string;
  readonly model: string;
  chat(messages, tools, options?): Promise<LLMResponse>;
}
```

## Legacy / Backward Compatibility

```ts
// Alias deprecated
export const mcpToolsToAnthropic = toProviderTools;
export const toolAliasMap = new Map<string, string>();  // Global singleton (legacy)

export function buildSystemPrompt(layers: ToolLayer[]): string
export function buildDiscoveryTools(layers: ToolLayer[]): ProviderTool[]
// Peuplent toolAliasMap comme side-effect — NON thread-safe pour parallel loops
```

**Préférer** `buildSystemPromptWithAliases()` et `buildDiscoveryToolsWithAliases()` pour les nouvelles implémentations (retourne une Map locale, parallel-safe).
