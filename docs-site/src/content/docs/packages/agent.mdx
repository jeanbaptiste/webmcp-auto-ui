Le package `@webmcp-auto-ui/agent` fournit l'orchestration complète pour exécuter une boucle agent LLM avec support des MCP (Model Context Protocol) et WebMCP, gestion des tools, lazy loading, et exécution de recettes.

## Exports principaux

### runAgentLoop

Exécute une boucle agent LLM → tool call → LLM jusqu'à `end_turn` ou `max_iterations`.

```ts
import { runAgentLoop } from '@webmcp-auto-ui/agent';

const result = await runAgentLoop(userMessage, {
  provider: remoteLLMProvider,
  layers: toolLayers,
  maxIterations: 5,
  callbacks: {
    onToken: (token) => console.log(token),
    onToolCall: (call) => console.log('Tool:', call.name),
  },
});

// result: AgentResult
// - text: réponse finale texte
// - toolCalls: tableau de tous les appels outils
// - metrics: tokens, latence, nombre d'itérations
// - messages: historique complet de la conversation
// - stopReason: 'end_turn' | 'max_iterations'
```

**Interface AgentLoopOptions:**

```ts
interface AgentLoopOptions {
  client?: McpClient;              // Client MCP (optionnel si WebMCP seulement)
  provider: LLMProvider;           // Provider LLM (obligatoire)
  layers?: ToolLayer[];            // Couches de tools structurées
  maxIterations?: number;          // Max itérations (défaut: 5)
  maxTokens?: number;              // Max tokens en output
  maxTools?: number;               // Max tools dans un appel LLM
  temperature?: number;            // Température LLM
  topK?: number;                   // Top K sampling
  cacheEnabled?: boolean;          // Cache de prompts (défaut: true)
  systemPrompt?: string;           // Prompt système custom
  initialMessages?: ChatMessage[]; // Historique précédent
  callbacks?: AgentCallbacks;      // Callbacks streaming
  signal?: AbortSignal;            // Cancellation
  truncateResults?: boolean;       // Tronquer résultats (défaut: true)
  compressHistory?: boolean;       // Compresser anciens résultats (défaut: true)
  maxResultLength?: number;        // Max chars pour résultat (défaut: 10000)
}
```

**Interface AgentResult:**

```ts
interface AgentResult {
  text: string;
  toolCalls: ToolCall[];
  metrics: AgentMetrics;
  stopReason: 'end_turn' | 'max_iterations';
  messages: ChatMessage[];  // Converstion complète
}
```

**Callbacks disponibles:**

```ts
interface AgentCallbacks {
  onIterationStart?: (iteration: number, maxIterations: number) => void;
  onLLMRequest?: (messages: ChatMessage[], tools: ProviderTool[]) => void;
  onLLMResponse?: (response: LLMResponse, latencyMs: number, tokens?: { input: number; output: number }) => void;
  onToolCall?: (call: ToolCall) => void;
  onWidget?: (type: string, data: Record<string, unknown>) => { id: string } | void;
  onClear?: () => void;
  onText?: (text: string) => void;
  onToken?: (token: string) => void;
  onDone?: (metrics: AgentMetrics) => void;
  onUpdate?: (id: string, data: Record<string, unknown>) => void;
  onMove?: (id: string, x: number, y: number) => void;
  onResize?: (id: string, w: number, h: number) => void;
  onStyle?: (id: string, styles: Record<string, string>) => void;
}
```

### Providers LLM

#### RemoteLLMProvider

Provider Anthropic Claude (via API distante).

```ts
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({
  proxyUrl: '/api/chat',
  model: 'sonnet',  // résout en claude-sonnet-4-6
});
```

**Options:**

```ts
interface RemoteLLMProviderOptions {
  proxyUrl: string;
  model?: string;  // 'haiku' | 'sonnet' | 'opus' (défaut: 'haiku')
  apiKey?: string;  // injecté dans le body, lu par le proxy +server.ts
}
```

#### WasmProvider

Provider Gemma 4 LiteRT (in-browser, via WASM sur le thread principal).

```ts
import { WasmProvider } from '@webmcp-auto-ui/agent';

const provider = new WasmProvider({
  model: 'gemma-e2b',    // 'gemma-e2b' ou 'gemma-e4b'
  contextSize: 32768,     // Taille du contexte (défaut: 32768)
  onProgress: (progress, status, loadedBytes, totalBytes) => {
    console.log(`${Math.round(progress * 100)}%`, loadedBytes, totalBytes);
  },
  onStatusChange: (status) => {
    console.log(status);  // 'idle' | 'loading' | 'ready' | 'error'
  },
});

// Initialiser le modèle
await provider.initialize();
```

**Options:**

```ts
interface WasmProviderOptions {
  model: WasmModelId;       // 'gemma-e2b' | 'gemma-e4b'
  contextSize?: number;     // Taille du contexte (défaut: 32768)
  onProgress?: (progress: number, status: string, loaded?: number, total?: number) => void;
  onStatusChange?: (status: 'idle' | 'loading' | 'ready' | 'error') => void;
}
```

Le modèle tourne sur le thread principal via LiteRT (pas de Web Worker). Le format natif `<|tool_call|>` est supporté pour les appels d'outils.

#### LocalLLMProvider

Provider Ollama (LLM local via serveur).

```ts
import { LocalLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new LocalLLMProvider({
  baseUrl: 'http://localhost:11434',
  model: 'mistral',
});
```

#### createProvider

Factory qui instancie le bon provider selon une configuration declarative.

```ts
import { createProvider } from '@webmcp-auto-ui/agent';
import type { LLMConfig } from '@webmcp-auto-ui/agent';

const config: LLMConfig = { type: 'remote', proxyUrl: '/api/chat', model: 'sonnet' };
const provider = createProvider(config);
```

#### Aliases de compatibilite

Les anciens noms `AnthropicProvider` et `GemmaProvider` restent exportes comme aliases :

```ts
// Alias -> classe actuelle
import { AnthropicProvider } from '@webmcp-auto-ui/agent';  // = RemoteLLMProvider
import { GemmaProvider } from '@webmcp-auto-ui/agent';      // = WasmProvider
```

Ces aliases sont deprecies. Preferez `RemoteLLMProvider` et `WasmProvider`.

### Tool Layers

Les **tool layers** structurent les outils MCP/WebMCP en couches pour le lazy loading et l'injection dans le prompt.

```ts
export type ToolLayer = McpLayer | WebMcpLayer;

interface McpLayer {
  protocol: 'mcp';
  serverName: string;
  description?: string;
  tools: McpToolDef[];
  recipes?: McpRecipe[];
}

interface WebMcpLayer {
  protocol: 'webmcp';
  serverName: string;
  description: string;
  tools: WebMcpToolDef[];
}
```

#### buildSystemPromptWithAliases

Génère le prompt système avec alias de tools (pour MCP canonique).

```ts
import { buildSystemPromptWithAliases } from '@webmcp-auto-ui/agent';

const { prompt, aliasMap } = buildSystemPromptWithAliases(layers);

// prompt: texte complet du système
// aliasMap: Map<"alias_name", "real_name"> pour résolution dynamique
```

Le prompt inclut les étapes d'exécution et les pseudos-tools de découverte (`search_tools`, `list_tools`).

#### buildDiscoveryToolsWithAliases

Construit la liste initiale des outils de découverte (avant activation serveur).

```ts
import { buildDiscoveryToolsWithAliases } from '@webmcp-auto-ui/agent';

const { tools, aliasMap } = buildDiscoveryToolsWithAliases(layers);

// tools: ProviderTool[] (discovery + alias)
// aliasMap: Map pour résolution
```

#### activateServerTools

Active tous les outils d'une couche (lazy loading au premier contact).

```ts
import { activateServerTools } from '@webmcp-auto-ui/agent';

const nextTools = activateServerTools(currentTools, layer);
```

#### resolveCanonicalTools

Résout les outils MCP en rôles canoniques (search_recipes, list_recipes, get_recipe) via 4 couches de matching.

```ts
import { resolveCanonicalTools } from '@webmcp-auto-ui/agent';

const matches = resolveCanonicalTools(mcpTools);

// matches: CanonicalMatch[]
// - role: 'search_recipes' | 'list_recipes' | 'get_recipe'
// - realToolName: nom réel du tool MCP
```

Les 4 couches:
1. **Exact match** — recherche `search_recipes`, `list_recipes`, `get_recipe`
2. **Decompose** — tokenise le nom et teste toutes paires (action, resource)
3. **Description keywords** — scanne la description pour recipe/template/workflow
4. **Fallback** — pas de match

### Recipes

#### WEBMCP_RECIPES

Tableau statique de toutes les recettes WebMCP compilees a partir des fichiers `.md` du repo.

```ts
import { WEBMCP_RECIPES } from '@webmcp-auto-ui/agent';

// WEBMCP_RECIPES: Recipe[]
console.log(WEBMCP_RECIPES.length);  // nombre de recettes disponibles
```

#### parseRecipe / parseRecipes

Parsent le frontmatter et le body d'un fichier `.md` en objet `Recipe`.

```ts
import { parseRecipe, parseRecipes } from '@webmcp-auto-ui/agent';

const recipe = parseRecipe(markdownString);
// recipe: Recipe | null

const recipes = parseRecipes(arrayOfMarkdownStrings);
// recipes: Recipe[]
```

#### recipeRegistry

Registre global des recettes. Permet d'enregistrer, filtrer et formater les recettes pour injection dans le prompt.

```ts
import {
  recipeRegistry,
  registerRecipes,
  filterRecipesByServer,
  formatRecipesForPrompt,
  formatMcpRecipesForPrompt,
} from '@webmcp-auto-ui/agent';

// Enregistrer des recettes dans le registre global
registerRecipes(recipes);

// Filtrer par serveur
const filtered = filterRecipesByServer(recipes, 'nasa');

// Formater les recettes WebMCP pour injection dans le system prompt
const promptBlock = formatRecipesForPrompt(recipes);

// Formater les recettes MCP (list_recipes) pour injection dans le system prompt
const mcpPromptBlock = formatMcpRecipesForPrompt(mcpRecipes);
```

### summarizeChat

Resume une conversation agent pour inclusion dans les exports HyperSkills.

```ts
import { summarizeChat } from '@webmcp-auto-ui/agent';
import type { SummarizeOptions, ChatSummaryResult } from '@webmcp-auto-ui/agent';

const result: ChatSummaryResult = await summarizeChat({
  messages: conversationHistory,
  provider: remoteLLMProvider,
  toolsUsed: ['widget_display', 'search_recipes'],
  toolCallCount: 5,
  mcpServers: ['nasa', 'hackernews'],
  skillsReferenced: [],
});

// result.chatSummary: string — resume textuel
// result.provenance: objet de traçabilite (servers, tools, LLM, etc.)
```

### Type Definitions

```ts
interface ToolCall {
  id: string;
  name: string;
  args: Record<string, unknown>;
  result?: string;
  error?: string;
  elapsed?: number;
  guided?: boolean;  // Précédé d'un tool de découverte
}

interface AgentMetrics {
  totalTokens: number;
  promptTokens: number;
  completionTokens: number;
  totalLatencyMs: number;
  toolCalls: number;
  iterations: number;
  cacheHits: number;
}

interface ChatMessage {
  role: 'user' | 'assistant' | 'system';
  content: string | ContentBlock[];
}

type ContentBlock =
  | { type: 'text'; text: string }
  | { type: 'tool_use'; id: string; name: string; input: Record<string, unknown> }
  | { type: 'tool_result'; tool_use_id: string; content: string };

interface LLMResponse {
  content: ContentBlock[];
  stopReason: string;
  usage?: { input_tokens: number; output_tokens: number; cache_read_input_tokens?: number };
}
```

import { autoui, NATIVE_WIDGET_NAMES } from '@webmcp-auto-ui/agent';

// - widget_display(name, params) — afficher un widget

## Token Tracker

Suivre les tokens et la latence en temps réel.

```ts
import { TokenTracker } from '@webmcp-auto-ui/agent';

const tracker = new TokenTracker();

tracker.track('prompt', 100);     // Tokens input
tracker.track('completion', 50);  // Tokens output
tracker.track('latency', 1200);   // Latence en ms

const metrics = tracker.getMetrics();
// {
//   totalTokens: 150,
//   promptTokens: 100,
//   completionTokens: 50,
//   totalLatencyMs: 1200,
//   tokensPerSec: 0.125
// }
```

## Utilitaires

### toProviderTools

Convertit `McpToolDef[]` en `ProviderTool[]` (format Anthropic).

```ts
import { toProviderTools } from '@webmcp-auto-ui/agent';

const providerTools = toProviderTools(mcpTools);
```

### fromMcpTools

Convertit `McpTool[]` (du core) en `McpToolDef[]` (agent).

```ts
import { fromMcpTools } from '@webmcp-auto-ui/agent';

const toolDefs = fromMcpTools(tools);
```

### trimConversationHistory

Réduit l'historique à un budget de tokens en supprimant les messages les plus anciens.

```ts
import { trimConversationHistory } from '@webmcp-auto-ui/agent';

const trimmed = trimConversationHistory(messages, 4096);
```

## Exemple Complet

```ts
import { runAgentLoop, RemoteLLMProvider, autoui } from '@webmcp-auto-ui/agent';
import { McpClient } from '@webmcp-auto-ui/core';

// 1. Créer un provider LLM
const provider = new RemoteLLMProvider({
  proxyUrl: '/api/chat',
  model: 'sonnet',  // résout en claude-sonnet-4-6
});

// 2. Créer un client MCP (si des MCP servers connectés)
const client = new McpClient('http://localhost:3000/mcp');
await client.initialize();

// 3. Construire les tool layers
const layers = [
  // Layer MCP
  {
    protocol: 'mcp' as const,
    serverName: 'recipes',
    tools: await client.listTools(),
  },
  // Layer WebMCP (autoui)
  autoui.layer(),
];

// 4. Lancer la boucle agent
const result = await runAgentLoop('Affiche un graphique des ventes Q1', {
  client,
  provider,
  layers,
  maxIterations: 5,
  callbacks: {
    onToken: (token) => process.stdout.write(token),
    onToolCall: (call) => console.log(`[${call.name}]`),
    onWidget: (type, data) => console.log(`Widget: ${type}`),
  },
});

console.log('Final:', result.text);
console.log('Metrics:', result.metrics);
```
