Le package `@webmcp-auto-ui/core` fournit les fondations: client MCP (streamable HTTP), serveur WebMCP, validation JSON schema, et utilitaires de communication.

## MCP Client

Client JSON-RPC pour le Model Context Protocol via HTTP streamable.

### McpClient

Classe pour communiquer avec un serveur MCP.

```ts
import { McpClient } from '@webmcp-auto-ui/core';

const client = new McpClient('http://localhost:3000/mcp', {
  clientName: 'my-app',
  clientVersion: '1.0.0',
  timeout: 30000,
});

// Initialize
await client.initialize();

// List tools
const tools = await client.listTools();
// McpTool[] avec name, description, inputSchema

// Call a tool
const result = await client.callTool('search_recipes', {
  query: 'desserts',
});
// McpToolResult { content: [...], isError? }
```

**Options:**

```ts
interface McpClientOptions {
  clientName?: string;              // Défaut: 'webmcp-client'
  clientVersion?: string;           // Défaut: '1.0.0'
  timeout?: number;                 // Défaut: 30000 ms
  headers?: Record<string, string>; // Headers custom
  autoReconnect?: boolean;          // Reconnexion auto
  maxReconnectAttempts?: number;    // Défaut: 3
}
```

**Méthodes:**

```ts
client.initialize(): Promise<McpInitializeResult>
client.listTools(): Promise<McpTool[]>
client.callTool(name: string, args: Record<string, unknown>): Promise<McpToolResult>
client.close(): Promise<void>
```

### McpMultiClient

Gérer plusieurs serveurs MCP en parallèle.

```ts
import { McpMultiClient } from '@webmcp-auto-ui/core';

const multi = new McpMultiClient();

await multi.connect('recipes', 'http://localhost:3000/mcp');
await multi.connect('database', 'http://localhost:3001/mcp');

// Lister les serveurs connectés
const servers = multi.listServers();

// Appeler un tool sur un serveur spécifique
const result = await multi.callTool('recipes', 'search_recipes', {
  query: 'pasta'
});

// Fermer tous les serveurs
await multi.closeAll();
```

## WebMCP Server

Créer un serveur WebMCP pour exposer des widgets et des tools.

### createWebMcpServer

Fabrique un serveur WebMCP avec widgets et tools.

```ts
import { createWebMcpServer } from '@webmcp-auto-ui/core';

const server = createWebMcpServer('my-widgets', {
  description: 'Custom UI widgets for my app',
});
```

**Enregistrer un widget:**

```ts
// Via recipe Markdown avec frontmatter YAML
server.registerWidget(`
---
widget: my-chart
description: Graphique personnalisé
schema:
  type: object
  properties:
    title:
      type: string
    data:
      type: array
required:
  - data
---

## Comment utiliser

Affiche un graphique avec les données fournies.
`, renderer);
```

**Renderer vanilla (fonction):**

```ts
const renderer = (container: HTMLElement, data: Record<string, unknown>) => {
  container.innerHTML = `<h1>${data.title}</h1>`;
  // Retourner optionnellement une fonction de cleanup
  return () => {
    // Nettoyage si nécessaire
  };
};
```

**Renderer framework (Svelte, React, etc.):**

```ts
// Passer le composant directement (non-vanilla)
import MyComponent from './MyComponent.svelte';
server.registerWidget(recipeMd, MyComponent);
```

**Ajouter un tool custom:**

```ts
server.addTool({
  name: 'compute',
  description: 'Effectuer un calcul',
  inputSchema: {
    type: 'object',
    properties: {
      operation: { type: 'string', enum: ['add', 'multiply'] },
      a: { type: 'number' },
      b: { type: 'number' },
    },
    required: ['operation', 'a', 'b'],
  },
  execute: async (params) => {
    const { operation, a, b } = params as { operation: string; a: number; b: number };
    const result = operation === 'add' ? a + b : a * b;
    return { result };
  },
});
```

**Lister les widgets:**

```ts
const widgets = server.listWidgets();
// WidgetEntry[] { name, description, inputSchema, recipe, renderer }

const widget = server.getWidget('my-chart');
if (widget) {
  console.log(widget.description);
  console.log(widget.inputSchema);
}
```

**Exporter en tant que layer:**

```ts
const layer = server.layer();
// { protocol: 'webmcp', serverName: 'my-widgets', description: '...', tools: [...] }
```

### parseFrontmatter

Parser le frontmatter YAML d'une recipe Markdown.

```ts
import { parseFrontmatter } from '@webmcp-auto-ui/core';

const markdown = `---
widget: stat
description: Statistique clé
schema:
  type: object
  properties:
    label:
      type: string
    value:
      type: string
---

## Quand utiliser
...
`;

const { frontmatter, body } = parseFrontmatter(markdown);
// frontmatter: { widget: 'stat', description: '...', schema: {...} }
// body: '## Quand utiliser...'
```

### mountWidget

Monter un widget vanilla dans un DOM container.

```ts
import { mountWidget } from '@webmcp-auto-ui/core';

const container = document.getElementById('widget');
const cleanup = mountWidget(container, 'stat', {
  label: 'Revenue',
  value: '$42k'
}, [server]);

// Plus tard...
cleanup?.();
```

## JSON Schema Validation

Valider des données contre un JSON Schema.

### validateJsonSchema

Valider strictement une entrée contre un schéma.

```ts
import { validateJsonSchema } from '@webmcp-auto-ui/core';

const schema = {
  type: 'object',
  properties: {
    name: { type: 'string' },
    age: { type: 'number', minimum: 0 },
  },
  required: ['name'],
};

const data = { name: 'Alice', age: 30 };
const result = validateJsonSchema(data, schema);

if (!result.valid) {
  console.error(result.errors);
  // ValidationError[] { path, message, expected, actual }
}
```

**Interface:**

```ts
interface ValidationResult {
  valid: boolean;
  errors: ValidationError[];
}

interface ValidationError {
  path: string;           // e.g., '$.properties.age'
  message: string;
  expected?: string;      // e.g., 'number'
  actual?: string;        // e.g., 'string'
}
```

## Utilitaires

### sanitizeSchema

Nettoyer un JSON schema pour le rendre compatible Anthropic API.

```ts
import { sanitizeSchema } from '@webmcp-auto-ui/core';

const rawSchema = {
  type: 'object',
  properties: {
    // ...
  },
  // Certains champs peuvent ne pas être compatibles
};

const clean = sanitizeSchema(rawSchema);
```

Supprime:
- Références non résolues (`$ref` sans `$defs`)
- Propriétés additionnelles invalides
- Schémas imbriqués trop profonds

### createToolGroup

Créer un groupe d'outils avec namespace commun.

```ts
import { createToolGroup } from '@webmcp-auto-ui/core';

const group = createToolGroup('database', [
  {
    name: 'query',
    description: 'Execute a SQL query',
    inputSchema: { /* ... */ },
  },
  {
    name: 'insert',
    description: 'Insert a record',
    inputSchema: { /* ... */ },
  },
]);

// group.tools:
// - database_query
// - database_insert
```

## PostMessage Bridge

Communiquer avec des outils via postMessage (iframe/worker).

### listenForAgentCalls

Écouter les appels outils depuis un client via postMessage.

```ts
import { listenForAgentCalls, signalCompletion } from '@webmcp-auto-ui/core';

const stopListening = listenForAgentCalls(async (toolName, args) => {
  // Exécuter le tool
  const result = await executeTool(toolName, args);
  // Signaler le résultat
  signalCompletion(result);
});

// Plus tard...
stopListening();
```

### callToolViaPostMessage

Appeler un tool dans un contexte distant (worker/iframe) via postMessage.

```ts
import { callToolViaPostMessage } from '@webmcp-auto-ui/core';

const result = await callToolViaPostMessage(
  targetWindow,
  'compute',
  { a: 5, b: 3 }
);
```

### isWebMCPEvent

Vérifier si un event est un message WebMCP valide.

```ts
import { isWebMCPEvent } from '@webmcp-auto-ui/core';

window.addEventListener('message', (e) => {
  if (isWebMCPEvent(e.data)) {
    // C'est un événement WebMCP
    console.log(e.data.type); // 'webmcp:call-tool', etc.
  }
});
```

## Exemple Complet: Serveur Custom + Client

```ts
// === Server (backend) ===
import { createWebMcpServer } from '@webmcp-auto-ui/core';

const server = createWebMcpServer('my-api', {
  description: 'API widgets',
});

server.addTool({
  name: 'fetch_data',
  description: 'Récupérer des données',
  inputSchema: {
    type: 'object',
    properties: {
      endpoint: { type: 'string' },
    },
    required: ['endpoint'],
  },
  execute: async (params) => {
    const { endpoint } = params as { endpoint: string };
    const response = await fetch(`/api/${endpoint}`);
    return await response.json();
  },
});

server.registerWidget(`
---
widget: data-display
description: Affichage de données
schema:
  type: object
  properties:
    items:
      type: array
---
`, (container, data) => {
  const items = data.items as any[];
  container.innerHTML = items.map(i => `<div>${i}</div>`).join('');
});

// === Client (agent) ===
import { runAgentLoop } from '@webmcp-auto-ui/agent';

const result = await runAgentLoop('Affiche les users', {
  provider: llmProvider,
  layers: [server.layer()],
  callbacks: {
    onToolCall: (call) => console.log(`Tool: ${call.name}`),
  },
});
```
