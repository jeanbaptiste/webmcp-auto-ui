Le package `@webmcp-auto-ui/sdk` fournit l'état applicatif (canvas store), l'orchestration HyperSkill, et l'enregistrement de skills.

## Canvas Store

Magasin d'état applicatif réactif pour gérer les widgets, le mode, la connexion MCP, et l'historique chat.

### Accès au Store

```ts
// Svelte 5 (runes, reactivity via $state/$derived)
import { canvas } from '@webmcp-auto-ui/sdk/canvas';

// Framework-agnostique (vanilla JS, React, Vue, etc.)
import { canvas } from '@webmcp-auto-ui/sdk/canvas-vanilla';
```

Le chemin `@webmcp-auto-ui/sdk/canvas` exporte le store Svelte 5 avec runes (`$state`/`$derived`). Le chemin `@webmcp-auto-ui/sdk/canvas-vanilla` exporte une version sans dépendance Svelte, utilisable dans n'importe quel environnement JavaScript.

### State Getters/Setters

```ts
// Widgets
canvas.blocks: Widget[];                      // Lecture: liste des widgets
canvas.addWidget(type, data): Widget;         // Ajouter un widget
canvas.removeBlock(id): void;                 // Supprimer par ID
canvas.updateBlock(id, data): void;           // Mettre à jour données
canvas.moveBlock(fromId, toId): void;         // Déplacer widget
canvas.clearBlocks(): void;                   // Vider le canvas
canvas.setBlocks(widgets): void;              // Remplacer tout

// Mode
canvas.mode: 'auto' | 'drag' | 'chat';
canvas.setMode(mode): void;

// LLM
canvas.llm: 'haiku' | 'sonnet' | 'gemma-e2b' | 'gemma-e4b';
canvas.setLlm(model): void;

// MCP
canvas.mcpUrl: string;
canvas.setMcpUrl(url): void;
canvas.mcpConnected: boolean;
canvas.mcpConnecting: boolean;
canvas.mcpName: string;
canvas.mcpTools: McpToolInfo[];
canvas.setMcpConnecting(bool): void;
canvas.setMcpConnected(connected, name?, tools?): void;
canvas.setMcpError(error): void;

// Chat
canvas.messages: ChatMsg[];
canvas.addMsg(role, content, thinking?): ChatMsg;
canvas.updateMsg(id, content, thinking?): void;
canvas.clearMessages(): void;

// Thème
canvas.themeOverrides: Record<string, string>;
canvas.setThemeOverrides(overrides): void;

// Métriques
canvas.blockCount: number;
canvas.isEmpty: boolean;
canvas.generating: boolean;
```

### Exemple Svelte

```svelte
<script lang="ts">
  import { canvas } from '@webmcp-auto-ui/sdk';
</script>

<!-- Binding en Svelte 5 -->
<select bind:value={canvas.llm}>
  <option value="haiku">Haiku</option>
  <option value="sonnet">Sonnet</option>
</select>

<button onclick={() => {
  canvas.addWidget('stat', {
    label: 'Sales',
    value: '1000'
  })
}}>Ajouter widget</button>

<div class="widgets">
  {#each canvas.blocks as widget (widget.id)}
    <WidgetRenderer type={widget.type} data={widget.data} />
  {/each}
</div>
```

### Réactivité

Le store supporte deux modèles de réactivité:

**1. Svelte 5 (runes):**

```ts
// canvas.svelte.ts - avec $state/$derived
import { canvas } from '@webmcp-auto-ui/sdk';

let blocks = $derived(canvas.blocks);
let empty = $derived(canvas.isEmpty);
```

**2. Framework-agnostique (subscribe):**

```ts
// canvas-vanilla.ts - pattern useSyncExternalStore
import { canvas } from '@webmcp-auto-ui/sdk/canvas-vanilla';

const unsubscribe = canvas.subscribe(() => {
  const snapshot = canvas.getSnapshot();
  console.log(snapshot.blocks);
});
```

## HyperSkill: Encoding/Decoding

Le système HyperSkill permet d'encoder/décoder les configurations de skills en URLs courtes et partageables.

### Encode

Sérialiser un skill en param URL (auto-compression si >6KB).

```ts
import { encodeHyperSkill } from '@webmcp-auto-ui/sdk';

const skill = {
  meta: {
    title: 'Mon Skill',
    llm: 'sonnet',
    mcp: 'http://localhost:3000/mcp',
  },
  content: {
    blocks: [
      { type: 'stat', data: { label: 'Revenue', value: '$42k' } },
    ]
  }
};

const url = await encodeHyperSkill(skill, sourceUrl);
// https://example.com?hs=g_qs9K2w... (gzip si >6KB)
```

**Interface HyperSkill:**

```ts
interface HyperSkill {
  meta: HyperSkillMeta;
  content: unknown;  // Données arbitraires (blocks, etc.)
}

interface HyperSkillMeta {
  title?: string;
  description?: string;
  version?: string;
  created?: string;
  mcp?: string;
  mcpName?: string;
  llm?: string;
  tags?: string[];
  theme?: Record<string, string>;
  hash?: string;
  previousHash?: string;
  chatSummary?: string;
  provenance?: {
    mcpServers?: string[];
    toolsUsed?: string[];
    toolCallCount?: number;
    skillsReferenced?: string[];
    llm?: string;
    exportedAt?: string;
  };
}
```

### Decode

Parser une URL HyperSkill en skill object.

```ts
import { decodeHyperSkill } from '@webmcp-auto-ui/sdk';

const skill = await decodeHyperSkill(urlOrParam);
// { meta: {...}, content: {...} }
```

Supporte automatiquement:
- Compression gzip (préfixe `gz.`)
- Compression brotli (préfixe `br.`)
- Base64 plain
- Paramètre `?hs=...`

### Hash & Versioning

Créer une version avec hash pour le contrôle de version.

```ts
import { computeHash, createVersion } from '@webmcp-auto-ui/sdk';

const hash = await computeHash(sourceUrl, skillContent);
// "c7d3e2a1f4..."

const version = await createVersion(skill, sourceUrl, previousHash);
// { hash, previousHash, timestamp, skill }
```

### Diff

Comparer deux versions de skill.

```ts
import { diffSkills } from '@webmcp-auto-ui/sdk';

const changes = await diffSkills(oldSkill, newSkill);
// { added: [...], modified: [...], removed: [...] }
```

## Canvas Store × HyperSkill

Intégration directe du store canvas avec HyperSkill:

```ts
import { canvas } from '@webmcp-auto-ui/sdk';

// Exporter la configuration actuelle
const skill = canvas.buildSkillJSON();
// {
//   version: '1.0',
//   name: 'skill-...',
//   created: '2024-01-15T...',
//   mcp: canvas.mcpUrl,
//   llm: canvas.llm,
//   blocks: canvas.blocks.map(b => ({type, data})),
//   theme: canvas.themeOverrides
// }

// Générer une URL courte HyperSkill
const param = await canvas.buildHyperskillParam();
// "g_qs9K2wZqE..." (compressé)

const url = `https://demos.hyperskills.net?hs=${param}`;

// Charger depuis un param
await canvas.loadFromParam(param);

// Ou depuis une URL complète
await canvas.loadFromUrl(url);
```

## Skills Registry

Gérer une collection de skills en localStorage + API.

### API Registry

```ts
import {
  createSkill,
  updateSkill,
  deleteSkill,
  getSkill,
  listSkills,
  clearSkills,
  loadSkills,
  loadDemoSkills,
  onSkillsChange,
} from '@webmcp-auto-ui/sdk';

// Types
interface Skill {
  id: string;
  name: string;
  description?: string;
  content: any;
  created: number;
  updated: number;
  version?: string;
  tags?: string[];
}

interface SkillBlock {
  type: string;
  data: Record<string, unknown>;
}

// CRUD
const skill = createSkill('my-skill', {
  description: 'My awesome skill',
  content: { blocks: [...] },
});

updateSkill(skill.id, { name: 'New Name' });
deleteSkill(skill.id);
const retrieved = getSkill(skill.id);
const all = listSkills();

// Batch
await loadSkills(skillObjects);
clearSkills();

// Demo skills — charge un jeu de skills prédéfinis pour le développement
await loadDemoSkills();

// Reactivity
onSkillsChange((skills) => {
  console.log('Skills changed:', skills);
});
```

### Exemple: Sauvegarde Auto

```svelte
<script lang="ts">
  import { canvas } from '@webmcp-auto-ui/sdk';
  import { createSkill, updateSkill, listSkills } from '@webmcp-auto-ui/sdk';

  let currentSkillId: string | null = null;

  function autoSave() {
    const skill = canvas.buildSkillJSON();
    if (!currentSkillId) {
      const created = createSkill(`skill_${Date.now()}`, skill);
      currentSkillId = created.id;
    } else {
      updateSkill(currentSkillId, skill);
    }
  }

  function loadAll() {
    return listSkills();
  }

  // Sauvegarder tous les 5s si modifié
  $effect(() => {
    canvas.blocks; // Dependency
    const timer = setTimeout(autoSave, 5000);
    return () => clearTimeout(timer);
  });
</script>

<div>
  <button onclick={autoSave}>Sauvegarder</button>
  <button onclick={() => {
    const skills = loadAll();
    skills.forEach(s => console.log(s.name));
  }}>Lister skills</button>
</div>
```

## Exemple Complet: Application Multi-Skills

```svelte
<script lang="ts">
  import { canvas } from '@webmcp-auto-ui/sdk';
  import { listSkills, createSkill, getSkill } from '@webmcp-auto-ui/sdk';
  import { WidgetRenderer, LLMSelector } from '@webmcp-auto-ui/ui';

  let skills = listSkills();
  let selectedSkillId: string | null = null;

  async function selectSkill(id: string) {
    selectedSkillId = id;
    const skill = getSkill(id);
    if (skill?.content?.blocks) {
      canvas.setBlocks(skill.content.blocks);
    }
    if (skill?.content?.llm) {
      canvas.llm = skill.content.llm;
    }
  }

  function saveAsNew() {
    const skill = canvas.buildSkillJSON();
    const created = createSkill(
      `skill_${Date.now()}`,
      skill
    );
    skills = listSkills();
    selectedSkillId = created.id;
  }
</script>

<div class="flex h-screen">
  <!-- Sidebar: Skills list -->
  <aside class="w-64 p-4 bg-gray-100">
    <h2 class="font-bold mb-4">Skills</h2>
    <ul>
      {#each skills as skill (skill.id)}
        <li class="mb-2">
          <button
            class:selected={selectedSkillId === skill.id}
            onclick={() => selectSkill(skill.id)}
          >
            {skill.name}
          </button>
        </li>
      {/each}
    </ul>
    <button onclick={saveAsNew}>Nouveau skill</button>
  </aside>

  <!-- Main: Canvas + Controls -->
  <main class="flex-1 flex flex-col">
    <div class="p-4 border-b flex gap-4">
      <LLMSelector bind:value={canvas.llm} />
    </div>
    <div class="flex-1 p-4 overflow-auto">
      <div class="grid grid-cols-2 gap-4">
        {#each canvas.blocks as widget (widget.id)}
          <WidgetRenderer
            id={widget.id}
            type={widget.type}
            data={widget.data}
          />
        {/each}
      </div>
    </div>
  </main>
</div>

<style>
  button.selected {
    @apply bg-blue-500 text-white;
  }
</style>
```
