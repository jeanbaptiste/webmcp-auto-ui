## Overview

When the agent calls a tool, the system :

1. **Resolves the tool alias** (recipe → real implementation)
2. **Parses the prefix** to extract `{serverName}_{protocol}_{toolName}`
3. **Routes to the right backend** (MCP client or WebMCP server)
4. **Captures the result** and stores it for possible `recall()`

## Dispatch by protocol

### MCP — Remote calls

```typescript
// Tool: "recipes_mcp_search_recipes"
const [, serverName, protocol, realToolName] = 
  "recipes_mcp_search_recipes".match(/^(.+?)_(mcp|webmcp)_(.+)$/);

// serverName = "recipes"
// protocol = "mcp"
// realToolName = "search_recipes"

// Route to MCP client
const mcpResult = await client.callTool(realToolName, block.input);
const textContent = mcpResult.content?.find(c => c.type === 'text');
const result = textContent?.text ?? JSON.stringify(mcpResult);
```

MCP results can be :
- **text** — String
- **image** — Base64 encoded
- **resource** — External URI
- **error** — Error message

### WebMCP — Local execution

```typescript
// Tool: "autoui_webmcp_widget_display"
const webmcpServer = webmcpServers.get("autoui");
const toolResult = await webmcpServer.executeTool("widget_display", params);

// Structured JSON result
const result = typeof toolResult === 'string' ? toolResult : JSON.stringify(toolResult);
```

WebMCP is **synchronous** and directly local.

## Special tools

### `widget_display()` — Display a widget

```typescript
// Agent call
widget_display('chart', {
  type: 'bar',
  labels: ['Q1', 'Q2', 'Q3', 'Q4'],
  data: [{ label: 'Sales', values: [100, 150, 200, 250] }]
})

// Result
{
  "widget": "chart",
  "data": { /* validated and cleaned params */ },
  "id": "w_abc123"
}

// UI callback
callbacks.onWidget?.('chart', data);
// → Render <Chart spec={data} /> in DOM
```

### `recall()` — Retrieve a compressed result

When results are compressed (> 300 chars), agent can recall them :

```typescript
// Old truncated result
"SELECT * FROM users;... [recall('toolu_abc') for full result, 125000 chars]"

// Recall call
recall('toolu_abc')

// Return: full result from resultBuffer
"SELECT * FROM users; [5000 rows detailed]"
```

**Important** : `recall()` is intercepted **before** the WebMCP server :

```typescript
if (realToolName === 'recall' && resultBuffer.size > 0) {
  const recallId = block.input.id;
  result = resultBuffer.get(recallId) ?? `No result found...`;
  // ← No server call
}
```

### `canvas()` — Manipulate widgets

```typescript
// Available actions
canvas('clear');                              // Clear canvas
canvas('update', { id: 'w_1', data: {...} }); // Update widget
canvas('move', { id: 'w_1', x: 100, y: 200 });
canvas('resize', { id: 'w_1', width: 400, height: 300 });
canvas('style', { id: 'w_1', styles: { color: 'red' } });
```

Each action triggers a corresponding callback :

```typescript
// In AgentCallbacks
callbacks.onClear?.();
callbacks.onUpdate?.(id, data);
callbacks.onMove?.(id, x, y);
callbacks.onResize?.(id, width, height);
callbacks.onStyle?.(id, styles);
```

### `list_tools()` / `search_tools()` — Pseudo discovery tools

These tools do **not require server activation** — it's local metadata :

```typescript
// Call
search_tools('query=chart')

// Intercept BEFORE normal dispatch
if (toolMatch && (toolMatch[3] === 'list_tools' || toolMatch[3] === 'search_tools')) {
  // Search locally in layer.tools
  const matches = layer.tools.filter(t =>
    t.name.includes(query) || t.description.includes(query)
  );
  result = JSON.stringify(matches);
  // ← No call to MCP client or WebMCP server
}
```

## Tool Aliases — Canonical resolution

### Problem

The agent wants to call `search_recipes`, but the real MCP server exposes `find_templates`.

### Solution

Match via **4-layer matching** and register an alias :

```typescript
// Detection
const matches = resolveCanonicalTools([
  { name: 'find_templates', description: 'Search for workflow templates...' }
]);
// → [{ role: 'search_recipes', realToolName: 'find_templates' }]

// Alias map
aliasMap.set('search_recipes', 'find_templates');

// Dispatch
const resolvedName = aliasMap.get('search_recipes') ?? 'search_recipes';
// → 'find_templates'
```

### Implementation

```typescript
// In loop.ts
const resolvedName = localAliasMap.get(name) ?? name;
const toolMatch = resolvedName.match(/^(.+?)_(mcp|webmcp)_(.+)$/);
const [, serverName, protocol, realToolName] = toolMatch;

// Use realToolName, not name
const mcpResult = await client.callTool(realToolName, block.input);
```

### Parallel-safe alias map

Each `runAgentLoop()` call creates its own local alias map :

```typescript
const localAliasMap = new Map<string, string>();

const disc = buildDiscoveryToolsWithAliases(options.layers ?? []);
for (const [k, v] of disc.aliasMap) {
  localAliasMap.set(k, v); // ← Local copy, not global
}
```

This avoids race conditions if multiple agents run in parallel.

## Complete flow: Example

### User request
```
"Show me a chart of top 5 sales by region"
```

### Step 1 : Discovery
Agent calls `recipes_mcp_search_recipes('sales by region')`

```
MCP Server "recipes" →
[
  { name: 'region_sales_chart', description: 'Chart top regions...' }
]
```

### Step 2 : Activation
Agent calls `recipes_mcp_get_recipe('region_sales_chart')`

```
MCP Server "recipes" →
{
  name: 'region_sales_chart',
  description: 'Chart top regions...',
  recipe: "## Usage\n1. Call get_sales_by_region()\n2. Display with chart-rich",
  schema: { type: 'object', required: ['limit'], properties: {...} }
}
```

Agent activates server → all "recipes" MCP tools become available.

### Step 3 : Execution
Agent calls `recipes_mcp_get_sales_by_region({ limit: 5, group_by: 'region' })`

```
MCP Server "recipes" →
{
  regions: [
    { name: 'North', sales: 450000 },
    { name: 'South', sales: 380000 },
    ...
  ]
}
```

### Step 4 : Display
Agent calls `autoui_webmcp_widget_display('chart-rich', {
  type: 'bar',
  labels: ['North', 'South', ...],
  data: [{ label: 'Sales', values: [450000, 380000, ...] }]
})`

```
WebMCP Server "autoui" →
{
  widget: 'chart-rich',
  data: { /* validated */ },
  id: 'w_xyz789'
}

callbacks.onWidget('chart-rich', data)
→ Render <Chart spec={data} />
```

## Error handling

### Validation error (schema)

```typescript
// widget_display('stat', { value: 42 }) — missing 'label'
// → validation error

// Agent must :
// 1. Read error message
// 2. Retry with missing fields
// 3. Not invent new fields

expected_schema: {
  type: 'object',
  required: ['label', 'value'],
  properties: { label: {...}, value: {...} }
}
```

### MCP error (timeout, server crash)

```typescript
// client.callTool('search_recipes', {...}) → network error
// catch (e) {
//   call.error = e.message;
//   toolResults.push({ type: 'tool_result', tool_use_id: id, content: `Error: ${e.message}` });
// }

// Following messages include error → agent can:
// - Retry with other parameters
// - Use another tool
// - Give up and show error message to user
```

### Result truncation

```typescript
// Result > 10KB → truncated
const truncated = truncateResult(rawResult, 10000);

// IF JSON with array
[{...}, {...}, ...] // 5000 items
// → sliced to 50 items, + suffix "... (truncated, 5000 items total)"

// Otherwise plain text
// → first 10000 characters + "... (truncated)"
```

Agent can call `recall()` for full result.
