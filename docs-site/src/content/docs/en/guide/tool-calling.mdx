**Tool calling** is the core of WebMCP Auto-UI. The LLM agent requests tools, and the dispatcher routes them to the correct server (MCP or WebMCP).

## Anatomy of a Tool Call

### 1. Agent generates a tool_use block

```typescript
// LLM response
{
  type: 'tool_use',
  id: 'toolu_1234',
  name: 'weather_mcp_get_forecast',
  input: { location: 'Paris', days: 3 }
}
```

### 2. Dispatcher routes the call

```typescript
// In runAgentLoop, dispatcher.ts processes each block
for (const block of toolBlocks) {
  // Resolve alias
  const resolvedName = aliasMap.get(block.name) ?? block.name;
  
  // Parse prefix: {serverName}_{protocol}_{toolName}
  const match = resolvedName.match(/^(.+?)_(mcp|webmcp)_(.+)$/);
  if (!match) {
    // Error: invalid format
    result = `Error: unknown tool format`;
  } else {
    const [, serverName, protocol, realToolName] = match;
    
    // Routing
    if (protocol === 'mcp') {
      // → MCP client.callTool()
    } else if (protocol === 'webmcp') {
      // → WebMCP server.executeTool()
    }
  }
}
```

### 3. Interceptor (pseudo-tools)

Before normal dispatch, two **pseudo-tools** are intercepted locally:

```typescript
// list_tools, search_tools — no server activation
if (toolMatch && (toolMatch[3] === 'list_tools' || toolMatch[3] === 'search_tools')) {
  const layer = layers.find(l => 
    sanitizeServerName(l.serverName) === serverName && 
    l.protocol === protocol
  );
  
  // Return the list without touching the server
  if (pseudoTool === 'list_tools') {
    result = JSON.stringify(layer.tools);
  } else {
    // search_tools
    const query = String(block.input.query ?? '').toLowerCase();
    const matches = layer.tools.filter(t =>
      t.name.toLowerCase().includes(query) ||
      t.description.toLowerCase().includes(query)
    );
    result = JSON.stringify(matches);
  }
}
```

### 4. Lazy server activation

On **first call** to a server tool:

```typescript
const activateMatch = block.name.match(/^(.+?)_(mcp|webmcp)_(.+)$/);
if (activateMatch) {
  const [, serverName, protocol] = activateMatch;
  const serverKey = `${serverName}_${protocol}`;
  
  if (!activatedServers.has(serverKey)) {
    activatedServers.add(serverKey);
    const layer = layers.find(l => 
      sanitizeServerName(l.serverName) === serverName && 
      l.protocol === protocol
    );
    if (layer) {
      // Add ALL server tools to activeTools
      activeTools = activateServerTools(activeTools, layer);
    }
  }
}
```

## MCP Dispatch

<img src="/diagrams/tool-calling-1.svg" alt="MCP dispatch flow from tool_use block to agent messages" />

### MCP Code

```typescript
if (protocol === 'mcp') {
  if (!client) {
    result = `Error: no MCP client available for tool ${name}`;
  } else {
    const mcpResult = await client.callTool(realToolName, block.input);
    const textContent = mcpResult.content?.find(
      (c) => c.type === 'text'
    ) as { text?: string } | undefined;
    const rawResult = textContent?.text ?? JSON.stringify(mcpResult);
    
    // Truncate if result > 10,000 chars (default MAX_RESULT_LEN)
    result = truncateResults 
      ? truncateResult(rawResult, maxResultLength) 
      : rawResult;
  }
}
```

**Network**: The MCP client establishes an SSE (Server-Sent Events) connection to the remote server and calls `tools/call`.

## WebMCP Dispatch

<img src="/diagrams/tool-calling-2.svg" alt="WebMCP dispatch flow with recall, widget_display, canvas and executeTool branches" />

### WebMCP Code

```typescript
if (protocol === 'webmcp') {
  // ── Special: recall (re-read a compressed result) ──
  if (realToolName === 'recall' && resultBuffer.size > 0) {
    const recallId = (block.input as { id: string }).id;
    result = resultBuffer.get(recallId) 
      ?? `No result found for id '${recallId}'.`;
  } else {
    // ── Normal dispatch to WebMCP server ──
    const webmcpServer = webmcpServers.get(serverName);
    if (!webmcpServer) {
      result = `Error: no WebMCP server "${serverName}" found.`;
    } else {
      const toolResult = await webmcpServer.executeTool(
        realToolName, 
        block.input
      );
      result = typeof toolResult === 'string' 
        ? toolResult 
        : JSON.stringify(toolResult);
      
      // ── Special: widget_display ──
      if (realToolName === 'widget_display') {
        const wr = toolResult as Record<string, unknown>;
        if (wr.widget && wr.data && !wr.error) {
          const widgetResult = callbacks.onWidget?.(
            wr.widget as string, 
            wr.data as Record<string, unknown>
          );
          if (widgetResult?.id) {
            result = JSON.stringify({ ...wr, id: widgetResult.id });
          }
        }
      }
      
      // ── Special: canvas (UI actions) ──
      if (realToolName === 'canvas') {
        const action = block.input.action as string;
        const id = block.input.id as string;
        const actionParams = block.input.params as Record<string, unknown>;
        
        switch (action) {
          case 'clear': callbacks.onClear?.(); break;
          case 'update': callbacks.onUpdate?.(id, actionParams ?? {}); break;
          case 'move': callbacks.onMove?.(
            id, 
            (actionParams?.x ?? block.input.x) as number,
            (actionParams?.y ?? block.input.y) as number
          ); break;
          case 'resize': callbacks.onResize?.(
            id,
            (actionParams?.width ?? block.input.width) as number,
            (actionParams?.height ?? block.input.height) as number
          ); break;
          case 'style': callbacks.onStyle?.(
            id,
            (actionParams?.styles ?? block.input.styles) as Record<string, string>
          ); break;
        }
      }
    }
  }
}
```

### Complete Example

```typescript
// Agent calls
{
  type: 'tool_use',
  id: 'toolu_5678',
  name: 'autoui_webmcp_widget_display',
  input: {
    name: 'stat-card',
    params: {
      label: 'Conversion',
      value: '3.2%',
      unit: '%',
      trend: 'up',
      variant: 'success'
    }
  }
}

// Dispatcher:
// 1. Get schema for 'stat-card' widget
// 2. Validate params vs schema
// 3. Sanitize image URLs (none to remove here)
// 4. Generate ID 'w_1a2b3c'
// 5. Call onWidget('stat-card', {...params})
// 6. Canvas adds widget: { id: 'w_1a2b3c', type: 'stat-card', data: {...} }
// 7. WidgetRenderer mounts StatCard component with props
// 8. UI displays the card
```

## Canvas Tools (UI manipulation)

The `canvas` tool allows the agent to **rearrange widgets**:

```typescript
// Agent calls
{
  type: 'tool_use',
  name: 'autoui_webmcp_canvas',
  input: {
    action: 'move',
    id: 'w_1a2b3c',
    params: { x: 100, y: 50 }
  }
}

// Callbacks in agent
callbacks.onMove?.('w_1a2b3c', 100, 50);
// → Update CSS widget position
```

**Actions**:
- `clear()` — empty canvas
- `update(id, data)` — modify widget data
- `move(id, x, y)` — reposition (CSS transform)
- `resize(id, width, height)` — resize
- `style(id, styles)` — apply CSS styles

## Recall (re-read compressed results)

After ~2 iterations, **old tool results** are compressed to save context:

```typescript
// Before compression
{
  type: 'tool_result',
  tool_use_id: 'toolu_1234',
  content: '{"data": [item1, item2, ... item1000], "total": 1000}'
}

// After compression (mutated in-place)
{
  type: 'tool_result',
  tool_use_id: 'toolu_1234',
  content: '{"data": [item1, item2, ... [recall(\'toolu_1234\') for complete result, 5000 chars]'
}

// Stored in resultBuffer
resultBuffer.set('toolu_1234', '{"data": [item1, item2, ... item1000], "total": 1000}');
```

If the agent calls `recall('toolu_1234')`:

```typescript
// Dispatcher intercepts recall
if (realToolName === 'recall' && resultBuffer.size > 0) {
  const recallId = (block.input as { id: string }).id;
  result = resultBuffer.get(recallId) ?? `No result found...`;
}
```

**Advantage**: Agent can navigate multiple data layers without blocking context.

## Error Handling

### Schema validation error

```typescript
// Invalid widget params
{
  type: 'tool_use',
  name: 'autoui_webmcp_widget_display',
  input: {
    name: 'stat',
    params: { label: 'Sales' }  // ✗ missing value
  }
}

// Response
{
  type: 'tool_result',
  content: JSON.stringify({
    error: 'Validation failed',
    details: [{ path: '/value', message: 'required' }],
    expected_schema: { ... }
  })
}

// Agent receives error + schema → retries with correct fields
```

### Tool not found

```typescript
// MCP
if (!client) {
  result = `Error: no MCP client available for tool ${name}`;
}

// WebMCP
if (!webmcpServer) {
  result = `Error: no WebMCP server "${serverName}" found.`;
}
```

### Tool execution failure

```typescript
try {
  const toolResult = await webmcpServer.executeTool(realToolName, block.input);
  result = ...;
} catch (e) {
  call.error = e instanceof Error ? e.message : String(e);
  toolResults.push({ 
    type: 'tool_result', 
    tool_use_id: block.id, 
    content: `Error: ${call.error}` 
  });
}
```

## Metrics and Monitoring

Each tool call is tracked:

```typescript
const call: ToolCall = {
  id: block.id,
  name: block.name,
  args: block.input,
  result: result,
  error: undefined,
  elapsed: Math.round(performance.now() - t1),
  guided: wasDiscovering,  // true if preceded by discovery tool
};

metrics.toolCalls++;
callbacks.onToolCall?.(call);  // subscriber can log
```

**Useful for**: analytics, debugging, agent optimization.

## Flow Summary

1. **LLM generates** `tool_use` block
2. **Dispatcher detects** format + resolves alias
3. **Pseudo-tools** (list_tools, search_tools) → local response
4. **Lazy activation**: if 1st call, add all server tools
5. **Routing**: MCP → network, WebMCP → local
6. **Post-processing**: compression, sanitize, callbacks
7. **tool_result** returns to agent
8. **Next iteration** of loop
