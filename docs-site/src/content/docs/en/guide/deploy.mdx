WebMCP Auto-UI deployment uses a **centralized script** (`scripts/deploy.sh`) that handles different apps with their specific paths and configurations.

## Golden Rule

**ALWAYS use the deployment script, never `scp` or `rsync` manually.**

The script knows the correct paths and configurations for each app.

## Target Infrastructure

- **Server**: `demo.hyperskills.net` (via SSH alias `ssh bot`)
- **User**: `webmcp` (non-root)
- **Base**: `/opt/webmcp-demos/`
- **Ports**: 
  - `3000`: flex (Node.js)
  - `3001`–`3010`: reserved for future Node apps
  - `80` / `443`: nginx (reverse proxy)

## Deployment

### Step 1: Build locally

```bash
# At monorepo root
pnpm build
```

This builds all apps in parallel.

### Step 2: Deploy via script

```bash
# Deploy everything
./scripts/deploy.sh

# Or a specific app
./scripts/deploy.sh flex
./scripts/deploy.sh viewer
./scripts/deploy.sh home
```

### Step 3: Verify on server

```bash
ssh bot

# For flex (Node.js)
ps aux | grep "node index.js"
curl http://localhost:3000

# For nginx (static apps)
sudo systemctl status nginx
curl http://localhost/home/
```

## The Deployment Script

### What it does

```bash
#!/bin/bash

# 1. Verify SSH access
# 2. Clean old builds on target (no JS pollution)
# 3. Copy build artifacts ONLY (via rsync)
# 4. Restart services (flex via systemd, nginx reload)
# 5. Verify health (curl http://localhost)
```

### Script Structure

```bash
# Configuration per app
declare -A APPS=(
  [flex]="apps/flex/.svelte-kit:index.js"
  [viewer]="apps/viewer/build:static"
  [home]="apps/home/dist:static"
  [todo]="apps/todo/build:static"
  [showcase]="apps/showcase/build:static"
  [multi-svelte]="apps/multi/build:static"
)

# For each app: build → clean → rsync → restart
for app in "${!APPS[@]}"; do
  src=$(echo ${APPS[$app]} | cut -d: -f1)
  type=$(echo ${APPS[$app]} | cut -d: -f2)
  
  # Clean before copy (avoid orphaned JS chunks)
  ssh bot "rm -rf /opt/webmcp-demos/$app/*"
  
  # Copy artifacts
  rsync -avz --delete "$src/" "webmcp@demo.hyperskills.net:/opt/webmcp-demos/$app/"
  
  # Restart if Node.js
  if [ "$type" = "node" ]; then
    ssh bot "systemctl restart webmcp-$app"
  fi
done

# nginx reload (for static apps)
ssh bot "systemctl reload nginx"
```

## Environment Variables (production)

### On Server

`.env.production` files must exist **before deployment**:

```bash
# /opt/webmcp-demos/flex/.env
NODE_ENV=production
ANTHROPIC_API_KEY=sk-ant-...
LOG_LEVEL=info

# /opt/webmcp-demos/home/.env  (optional for static, but useful for build-time)
PUBLIC_BASE_URL=https://demos.hyperskills.net
PUBLIC_GA_ID=G-XXXXXX
```

**NEVER** deploy `.env` files — they already exist on server (created manually once).

### Build-time Variables (home)

`home` is an **Astro** app that requires `PUBLIC_BASE_URL` at **build** time:

```bash
# Before build locally
export PUBLIC_BASE_URL=https://demos.hyperskills.net

# Then build
pnpm build
```

The script **assumes** it's already defined globally. If missing locally:

```bash
# Fallback
PUBLIC_BASE_URL=https://demos.hyperskills.net pnpm build
```

## Networking (nginx)

**nginx** reverse-proxies each app:

```nginx
# /etc/nginx/sites-available/default

# Flex (Node.js → port 3000)
location /flex/ {
  proxy_pass http://localhost:3000/;
}

# Static apps → serve from /opt/webmcp-demos/
location /viewer/ {
  alias /opt/webmcp-demos/viewer/;
  try_files $uri $uri/index.html =404;
}

location /home/ {
  alias /opt/webmcp-demos/home/;
  try_files $uri $uri/index.html =404;
}

location / {
  alias /opt/webmcp-demos/home/;  # Default = home
  try_files $uri $uri/index.html =404;
}
```

## Rollback

If a deployment breaks something:

```bash
# Git history
git log --oneline | head -20

# Checkout previous version
git checkout <commit_before>

# Rebuild + redeploy
pnpm build
./scripts/deploy.sh
```

**Attention**: Script does `rm -rf` before copy, so old files are lost. Rollback must be **fast**.

## Troubleshooting

### App shows old code after deploy

**Cause**: Orphaned JS chunks after build. Script `deploy.sh` does `rm -rf` to prevent this, but if deployed manually:

**Solution**:
```bash
ssh bot
rm -rf /opt/webmcp-demos/{app}/*.js /opt/webmcp-demos/{app}/*.map
systemctl restart nginx  # or restart webmcp-flex
```

### `.env` missing after deploy

**Cause**: `.env` should NEVER be in git or deployed via script.

**Solution**: Recreate manually on server
```bash
ssh bot
cat > /opt/webmcp-demos/flex/.env << EOF
NODE_ENV=production
ANTHROPIC_API_KEY=sk-ant-...
EOF
systemctl restart webmcp-flex
```

### nginx returns 404

**Cause**: Wrong alias or missing `try_files`.

**Debug**:
```bash
ssh bot
# Verify deployed content
ls -la /opt/webmcp-demos/home/
ls -la /opt/webmcp-demos/viewer/build/

# Test nginx config
sudo nginx -t

# Reload
sudo systemctl reload nginx

# Check paths in config
grep -A 5 "location /home/" /etc/nginx/sites-available/default
```

### API rate-limited (Anthropic)

**Symptom**: 429 errors from Claude.

**Solution**: 
1. Wait (exponential backoff handled by client)
2. Verify API key not leaked
3. Increase `maxTokens` / reduce `maxIterations`

## Maintenance

### Restart all services

```bash
ssh bot
sudo systemctl restart webmcp-flex nginx
```

### Clean old logs

```bash
ssh bot
journalctl --vacuum-time=7d  # Keep 7 days
```

### Update dependencies

```bash
# Locally
pnpm update
pnpm build

# Deploy
./scripts/deploy.sh
```

- ✅ Always use `./scripts/deploy.sh`

- ✅ Verify build locally succeeds before deploying
