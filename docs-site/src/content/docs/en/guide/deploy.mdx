## Overview

The project contains several apps (flex, viewer, home, showcase, etc.) with different deployment paths. **Always use the `scripts/deploy.sh` script** — paths are complex and vary by app.

## Deployment script

### Usage

```bash
# Deploy all apps
./scripts/deploy.sh

# Deploy a single app
./scripts/deploy.sh flex
./scripts/deploy.sh viewer
./scripts/deploy.sh home
```

### Why not manual `scp` / `rsync`?

1. **Different paths per app** — each app has a different destination
2. **Critical cleanup** — `rsync --delete` has deleted production `.env`
3. **Pre-build required** — some apps require build before deploy
4. **Operation order** — systemd restart can interfere with file copies

## App mapping

| App | Type | ExecStart systemd | Deploy destination | Preparation |
|-----|------|-------------------|--------------------|-------------|
| **flex** | Node.js | `node index.js` | `/opt/webmcp-demos/flex/` | `npm run build` |
| **viewer** | Node.js (SvelteKit) | `node build/index.js` | `/opt/webmcp-demos/viewer/build/` | `npm run build` |
| **home** | Static (nginx) | N/A | `/opt/webmcp-demos/home/` | `npm run build` + `PUBLIC_BASE_URL=...` |
| **showcase** | Static (nginx) | N/A | `/opt/webmcp-demos/showcase/` | `npm run build` |
| **multi-svelte** | Static (nginx) | N/A | `/opt/webmcp-demos/multi-svelte/` | `npm run build` |
| **todo** | Static (nginx) | N/A | `/opt/webmcp-demos/todo/` | `npm run build` |

### **Special case: home** — Required environment variable

The home app generates absolute URLs for demos. It MUST be built with :

```bash
PUBLIC_BASE_URL=https://demos.hyperskills.net npm run build
```

Without this variable, URLs will be `http://localhost:5173/...` in production.

## Detailed deployment process

### 1. Build locally

```bash
# At monorepo root
npm install
npm run build

# Or for a single app
cd apps/flex
npm run build
```

### 2. Execute deployment script

```bash
./scripts/deploy.sh flex
```

The script :
- ✅ Checks that build exists
- ✅ Cleans old destination (except `.env`)
- ✅ Copies new files
- ✅ Restarts systemd if it's a Node.js service
- ✅ Displays final status

### 3. Verify deployment

```bash
# Access the app
curl https://demos.hyperskills.net/flex

# Check logs (if Node.js)
ssh bot systemctl status webmcp-demos-flex
ssh bot journalctl -u webmcp-demos-flex -f

# Or for a static app
ssh bot ls -la /opt/webmcp-demos/home/
```

## Critical rules

### ❌ NEVER

1. **Use `rsync --delete` directly** — deletes production `.env`
   ```bash
   # ❌ BAD
   rsync -av --delete build/ bot:/opt/webmcp-demos/flex/
   ```

2. **Copy `.env` from local** — overwrites production API keys
   ```bash
   # ❌ BAD
   scp .env.local bot:/opt/webmcp-demos/flex/.env
   ```

3. **Deploy without cleaning old JS chunks**
   ```bash
   # ❌ BAD (old chunks remain)
   scp -r build/* bot:/opt/webmcp-demos/flex/
   ```

4. **Modify deploy directory directly**
   ```bash
   # ❌ BAD
   ssh bot "cd /opt/webmcp-demos/flex && npm install"
   ```

### ✅ ALWAYS

1. **Use `./scripts/deploy.sh`**
   ```bash
   # ✅ GOOD
   ./scripts/deploy.sh flex
   ```

2. **Build locally before deploying**
   ```bash
   npm run build
   ./scripts/deploy.sh flex
   ```

3. **Check access before deploying**
   ```bash
   ssh bot "ls /opt/webmcp-demos/flex/.env"
   # If "No such file", create from template
   ```

4. **Keep `.env` and secrets in place**
   ```bash
   # Script preserves existing .env
   ```

## Troubleshooting

### Error: "Old app still running"

Old code still running even after deployment.

**Causes** :
- Old JS chunks not cleaned
- systemd/nginx cache
- Port still listened to by old process

**Solutions** :
```bash
# Force systemd restart
ssh bot systemctl restart webmcp-demos-flex

# Or (static) reload nginx
ssh bot systemctl reload nginx

# Check port
ssh bot lsof -i :3000
```

### Error: ".env not found"

App won't start because `.env` is missing.

**Solution** : Create from server
```bash
ssh bot "cat > /opt/webmcp-demos/flex/.env" << EOF
VITE_ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=...
EOF
```

Or copy a secure template (NEVER copy from `.env.local`).

### Error: "Cannot GET /flex"

nginx returns 404.

**Causes** :
- Static app not built
- Wrong nginx path
- Old cache

**Solutions** :
```bash
# Check app is built
ls -la apps/home/dist/

# Check nginx config
ssh bot "grep -A5 '/home' /etc/nginx/sites-available/demos.hyperskills.net"

# Purge cache
ssh bot "systemctl reload nginx"
```

## Pre-deployment checklist

- [ ] Code committed and pushed
- [ ] Local tests pass (`npm run test`)
- [ ] Local build succeeds (`npm run build`)
- [ ] Environment variables correct
  - [ ] `PUBLIC_BASE_URL=https://demos.hyperskills.net` for home
  - [ ] Other apps have no restrictions
- [ ] `.env` file exists on server
- [ ] Sufficient disk space: `ssh bot df -h /opt`
- [ ] No one else deploying

## Complete deployment script

```bash
#!/bin/bash
# scripts/deploy.sh — Secure multi-app deployment

APP="${1:-all}"
HOST="bot"
BASE_PATH="/opt/webmcp-demos"

case "$APP" in
  flex)
    BUILD_DIR="apps/flex/dist"
    DEST_PATH="$BASE_PATH/flex"
    SERVICE="webmcp-demos-flex"
    ;;
  viewer)
    BUILD_DIR="apps/viewer/build"
    DEST_PATH="$BASE_PATH/viewer/build"
    SERVICE="webmcp-demos-viewer"
    ;;
  home)
    BUILD_DIR="apps/home/dist"
    DEST_PATH="$BASE_PATH/home"
    SERVICE=""
    ;;
  *)
    echo "Usage: $0 {flex|viewer|home|showcase|all}"
    exit 1
    ;;
esac

# Check build
if [ ! -d "$BUILD_DIR" ]; then
  echo "❌ Build not found: $BUILD_DIR"
  exit 1
fi

echo "📦 Deploying $APP..."

# Copy (preserve .env and secrets)
ssh "$HOST" "mkdir -p $DEST_PATH && rm -rf $DEST_PATH/* && ([ -f $DEST_PATH/.env ] && mv $DEST_PATH/.env /tmp/.env.bak)"
scp -r "$BUILD_DIR/"* "$HOST:$DEST_PATH/" 2>/dev/null
ssh "$HOST" "[ -f /tmp/.env.bak ] && mv /tmp/.env.bak $DEST_PATH/.env"

# Restart if it's a Node.js service
if [ -n "$SERVICE" ]; then
  ssh "$HOST" "systemctl restart $SERVICE && sleep 2 && systemctl is-active $SERVICE"
else
  ssh "$HOST" "systemctl reload nginx"
fi

echo "✅ $APP deployed successfully!"
```
