Get started with WebMCP Auto-UI in 5 minutes.

## Prerequisites

- **Node.js** 18.x or higher
- **npm** (the monorepo uses `package-lock.json`)
- **Claude API key** (optional — Gemma in-browser is available)

## Installation

### Quick start (boilerplate)

The fastest way to get started is with the **boilerplate** app, which includes three pre-configured Tricoteuses widgets:

```bash
npx degit jeanbaptiste/webmcp-auto-ui/apps/boilerplate my-app
cd my-app
npm install
npm run dev
```

### Full monorepo (contributors)

```bash
git clone https://github.com/jeanbaptiste/webmcp-auto-ui.git
cd webmcp-auto-ui
npm install
```

### Configure environment variables

Create a `.env.local` file at the root:

```bash
# Claude API (optional, required for remote agents)
ANTHROPIC_API_KEY=sk-ant-...

# URL of remote MCP servers (format: protocol://host:port)
MCP_SERVERS=http://localhost:3001,http://localhost:3002
```

### Start in development

```bash
npm run dev
```

The app starts on `http://localhost:5173`. You will see:

- **Canvas**: Widget drawing area
- **Chat panel**: Agent interface
- **Settings**: LLM model selector (Claude/Gemma)

## Your First Simple Agent

### Step 1: Choose a model

1. Click **Settings**
2. Select **Claude (Haiku)** or **Gemma (in-browser)**
3. If Claude: verify that `ANTHROPIC_API_KEY` is set

### Step 2: Write a prompt

In the **Chat panel**, enter:

```
Display a stat with label "Visitors" and value "1,234"
```

### Step 3: Watch the agent

1. Agent receives the prompt
2. Identifies it needs a `stat` widget
3. Calls `widget_display('stat', { label: 'Visitors', value: '1,234' })`
4. Canvas displays the widget

## Development Architecture

### Monorepo structure

```
webmcp-auto-ui/
├── packages/
│   ├── core/            # WebMCP types, polyfill, MCP client
│   ├── agent/           # Agent loop, LLM providers, tool layers
│   ├── ui/              # Svelte widgets (30+), UI components
│   └── sdk/             # HyperSkills, canvas store
├── apps/
│   ├── boilerplate/     # Entry point for new devs (SvelteKit + 3 Tricoteuses widgets)
│   ├── flex2/           # Main SvelteKit app
│   ├── showcase2/       # Showcase demo
│   ├── todo2/           # Todo demo
│   ├── viewer2/         # HyperSkills viewer
│   ├── home/            # Homepage (static)
│   ├── mobile/          # Mobile app
│   ├── multi-svelte/    # Multi-widget Svelte
│   ├── recipes/         # Recipes demo
│   └── wasm/            # Gemma WASM demo
└── scripts/
    └── deploy.sh        # Deployment script
```

## Common Examples

### Create a Claude provider

`RemoteLLMProvider` connects through a SvelteKit proxy endpoint, keeping your API key server-side:

```typescript
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({
  proxyUrl: '/api/chat',
  model: 'sonnet', // resolves to claude-sonnet-4-6
});
```

Available model aliases: `'haiku'` (claude-haiku-4-5-20251001), `'sonnet'` (claude-sonnet-4-6), `'opus'` (claude-opus-4-6).

### Create a Gemma provider

```typescript
import { WasmProvider } from '@webmcp-auto-ui/agent';

const provider = new WasmProvider({
  modelId: 'gemma-e4b', // 4B variant
  autoDownload: true,
});

// Wait for loading
provider.onStatusChange((status) => {
  console.log('Gemma status:', status);
});
```

### Run an agent

```typescript
import { runAgentLoop } from '@webmcp-auto-ui/agent';

const result = await runAgentLoop('Display a sales chart', {
  provider,
  layers: [
    // ... tool layers
  ],
  maxIterations: 5,
  callbacks: {
    onToolCall: (call) => console.log('Tool called:', call.name),
    onWidget: (type, data) => console.log('Widget:', type),
  },
});

console.log('Agent finished:', result.text);
```

### Display a widget

```svelte
<script>
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';
  
  const widgetData = {
    label: 'Visitors',
    value: '1,234',
    trend: 'up',
  };
</script>

<WidgetRenderer 
  type="stat-card" 
  data={widgetData}
  servers={[autoui]}
/>
```

## Connect to an MCP Server

### Start a local MCP server

```bash
# Example: weather MCP server
npx @example/weather-mcp
```

Must listen on `http://localhost:3000` and expose an `/sse` endpoint (Server-Sent Events).

### Connect from the app

```typescript
import { McpClient } from '@webmcp-auto-ui/core';

const client = new McpClient({
  serverUrl: 'http://localhost:3000',
});

await client.initialize();

// List tools
const tools = await client.listTools();
console.log('MCP tools:', tools);
```

## Local Deployment

To test on a local server before production:

```bash
# Build for production
npm run build

# Start the server
npm run preview
```

Accessible on `http://localhost:4173`.

## Troubleshooting

### Agent does not call tools

- Verify that `layers` are configured correctly
- Verify that `buildDiscoveryTools()` returns expected tools
- Check LLM provider logs

### Gemma does not load

- Verify internet connection (downloads ~6GB of WASM)
- Verify browser (Chrome/Edge recommended)
- Check dev console (`F12`)

### Widget does not display

- Verify that the type exists in `NATIVE_MAP` of `WidgetRenderer`
- Verify data structure vs widget's `inputSchema`
- Check validation errors
