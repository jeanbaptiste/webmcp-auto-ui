The `@webmcp-auto-ui/agent` package contains the agent loop, LLM providers, tool layer management, and recipe execution.

## Main API: `runAgentLoop`

```ts
export async function runAgentLoop(
  userMessage: string,
  options: AgentLoopOptions
): Promise<AgentResult>
```

Launches the agent loop: LLM → tool calls → LLM until `end_turn` or max iterations.

### Options

```ts
export interface AgentLoopOptions {
  client?: McpClient;           // MCP client (optional if WebMCP tools only)
  provider: LLMProvider;         // LLM provider (Anthropic, Gemma WASM, local)
  layers?: ToolLayer[];          // Structured tool layers (replaces legacy mcpTools)
  maxIterations?: number;        // Default: 5
  maxTokens?: number;            // Optional response limit
  maxTools?: number;             // Max parallel tools
  temperature?: number;          // LLM temperature
  topK?: number;                 // Top-K sampling
  cacheEnabled?: boolean;        // Use prompt cache (Claude 3.5+)
  systemPrompt?: string;         // Custom system prompt
  initialMessages?: ChatMessage[]; // Conversation history
  callbacks?: AgentCallbacks;    // Streaming events
  signal?: AbortSignal;          // Cancellation
  truncateResults?: boolean;     // Truncate tool results (default: true)
  compressHistory?: boolean;     // Compress old results (default: true)
  maxResultLength?: number;      // Max truncated result size (default: 10000)
}
```

### Result

```ts
export interface AgentResult {
  text: string;                  // Last text produced
  toolCalls: ToolCall[];         // All tool calls made
  metrics: AgentMetrics;         // Token usage, latency, iterations
  stopReason: 'end_turn' | 'max_iterations';
  messages: ChatMessage[];       // Full conversation
}
```

### Callbacks

```ts
export interface AgentCallbacks {
  onIterationStart?: (iteration: number, maxIterations: number) => void;
  onLLMRequest?: (messages: ChatMessage[], tools: ProviderTool[]) => void;
  onLLMResponse?: (response: LLMResponse, latencyMs: number, tokens?: { input: number; output: number }) => void;
  onToolCall?: (call: ToolCall) => void;
  onWidget?: (type: string, data: Record<string, unknown>) => { id: string } | void;
  onText?: (text: string) => void;
  onToken?: (token: string) => void;
  onDone?: (metrics: AgentMetrics) => void;
  // Canvas mutations
  onClear?.();
  onUpdate?(id: string, data: Record<string, unknown>): void;
  onMove?(id: string, x: number, y: number): void;
  onResize?(id: string, w: number, h: number): void;
  onStyle?(id: string, styles: Record<string, string>): void;
  onUserInteraction?(widgetId: string, widgetType: string, action: string, payload: unknown): void;
}
```

### Example

```ts
import { runAgentLoop } from '@webmcp-auto-ui/agent';
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({
  apiKey: process.env.ANTHROPIC_API_KEY,
  model: 'claude-3-5-sonnet-20241022',
});

const result = await runAgentLoop('Show me available recipes', {
  provider,
  layers: [...],
  callbacks: {
    onText: (text) => console.log(text),
    onToolCall: (call) => console.log('Tool:', call.name),
    onDone: (metrics) => console.log('Tokens:', metrics.totalTokens),
  },
});

console.log(result.text);
```

## System Prompt & Alias

### `buildSystemPrompt` / `buildSystemPromptWithAliases`

```ts
export function buildSystemPrompt(layers: ToolLayer[]): string

export interface SystemPromptResult {
  prompt: string;
  aliasMap: Map<string, string>;  // {serverName}_{protocol}_{canonical} → real
}
export function buildSystemPromptWithAliases(layers: ToolLayer[]): SystemPromptResult
```

The system prompt lists :
- **Step 1** : Search for recipes by keyword
- **Step 1b** : List available recipes
- **Step 1c** : Search for tools
- **Step 1d** : List tools
- **Step 2** : Read complete recipe
- **Step 3** : Execute instructions / tool calls
- **Step 4** : Display UI (widget_display, canvas, recall)

### Canonical resolution (4 layers)

MCP tools are normalized to canonical roles (search_recipes, list_recipes, get_recipe) via `resolveCanonicalTools` :

```ts
export function resolveCanonicalTools(tools: McpToolDef[]): CanonicalMatch[]

interface CanonicalMatch {
  role: 'search_recipes' | 'list_recipes' | 'get_recipe';
  realToolName: string;
}
```

**Matching layers :**
1. Exact name (search_recipes, list_recipes, get_recipe)
2. Token decomposition : (action, resource) → (search|list|get, recipe|template|...)
3. Description keywords : Search keywords (recipe, skill, template, workflow)
4. Fallback : No match, list raw tools

## Conversation history and compression

### `trimConversationHistory`

Keeps conversation under a token budget by removing old user/assistant pairs from the beginning.

```ts
export function trimConversationHistory(
  history: ChatMessage[],
  maxTokens: number
): ChatMessage[]
```

### Auto compression

The agent loop automatically compresses old tool results (> 300 chars) into 200 char previews + recall() hint. Enabled by default (option `compressHistory: true`).

## Types

```ts
export interface ChatMessage {
  role: 'user' | 'assistant' | 'system';
  content: string | ContentBlock[];
}

export type ContentBlock =
  | { type: 'text'; text: string }
  | { type: 'tool_use'; id: string; name: string; input: Record<string, unknown> }
  | { type: 'tool_result'; tool_use_id: string; content: string };

export interface ToolCall {
  id: string;
  name: string;
  args: Record<string, unknown>;
  result?: string;
  error?: string;
  elapsed?: number;
  guided?: boolean;  // Preceded by a discovery tool
}

export interface AgentMetrics {
  totalTokens: number;
  promptTokens: number;
  completionTokens: number;
  totalLatencyMs: number;
  toolCalls: number;
  iterations: number;
  cacheHits: number;
}

export interface LLMResponse {
  content: ContentBlock[];
  stopReason: string;
  usage?: {
    input_tokens: number;
    output_tokens: number;
    cache_read_input_tokens?: number;
  };
  stats?: {
    tokensPerSec: number;
    totalTokens: number;
    latencyMs: number;
  };
}

export interface LLMProvider {
  readonly name: string;
  readonly model: string;
  chat(messages, tools, options?): Promise<LLMResponse>;
}
```

## Legacy / Backward Compatibility

```ts
// Deprecated alias
export const mcpToolsToAnthropic = toProviderTools;
export const toolAliasMap = new Map<string, string>();  // Global singleton (legacy)

export function buildSystemPrompt(layers: ToolLayer[]): string
export function buildDiscoveryTools(layers: ToolLayer[]): ProviderTool[]
// Populate toolAliasMap as side-effect — NOT thread-safe for parallel loops
```

**Prefer** `buildSystemPromptWithAliases()` and `buildDiscoveryToolsWithAliases()` for new implementations (returns local Map, parallel-safe).
