The `@webmcp-auto-ui/agent` package provides complete orchestration for executing an LLM agent loop with MCP (Model Context Protocol) and WebMCP support, tool management, lazy loading, and recipe execution.

## Main Exports

### runAgentLoop

Executes an LLM agent loop → tool call → LLM until `end_turn` or `max_iterations`.

```ts
import { runAgentLoop } from '@webmcp-auto-ui/agent';

const result = await runAgentLoop(userMessage, {
  provider: remoteLLMProvider,
  layers: toolLayers,
  maxIterations: 5,
  callbacks: {
    onToken: (token) => console.log(token),
    onToolCall: (call) => console.log('Tool:', call.name),
  },
});

// result: AgentResult
// - text: final text response
// - toolCalls: array of all tool calls
// - metrics: tokens, latency, iterations
// - messages: complete conversation history
// - stopReason: 'end_turn' | 'max_iterations'
```

**Interface AgentLoopOptions:**

```ts
interface AgentLoopOptions {
  client?: McpClient;              // MCP client (optional if WebMCP only)
  provider: LLMProvider;           // LLM provider (required)
  layers?: ToolLayer[];            // Structured tool layers
  maxIterations?: number;          // Max iterations (default: 5)
  maxTokens?: number;              // Max output tokens
  maxTools?: number;               // Max tools per LLM call
  temperature?: number;            // LLM temperature
  topK?: number;                   // Top K sampling
  cacheEnabled?: boolean;          // Prompt cache (default: true)
  systemPrompt?: string;           // Custom system prompt
  initialMessages?: ChatMessage[]; // Previous history
  callbacks?: AgentCallbacks;      // Streaming callbacks
  signal?: AbortSignal;            // Cancellation
  truncateResults?: boolean;       // Truncate results (default: true)
  compressHistory?: boolean;       // Compress old results (default: true)
  maxResultLength?: number;        // Max result chars (default: 10000)
}
```

**Interface AgentResult:**

```ts
interface AgentResult {
  text: string;
  toolCalls: ToolCall[];
  metrics: AgentMetrics;
  stopReason: 'end_turn' | 'max_iterations';
  messages: ChatMessage[];  // Complete conversation
}
```

**Available Callbacks:**

```ts
interface AgentCallbacks {
  onIterationStart?: (iteration: number, maxIterations: number) => void;
  onLLMRequest?: (messages: ChatMessage[], tools: ProviderTool[]) => void;
  onLLMResponse?: (response: LLMResponse, latencyMs: number, tokens?: { input: number; output: number }) => void;
  onToolCall?: (call: ToolCall) => void;
  onWidget?: (type: string, data: Record<string, unknown>) => { id: string } | void;
  onClear?: () => void;
  onText?: (text: string) => void;
  onToken?: (token: string) => void;
  onDone?: (metrics: AgentMetrics) => void;
  onUpdate?: (id: string, data: Record<string, unknown>) => void;
  onMove?: (id: string, x: number, y: number) => void;
  onResize?: (id: string, w: number, h: number) => void;
  onStyle?: (id: string, styles: Record<string, string>) => void;
}
```

### LLM Providers

#### RemoteLLMProvider

Anthropic Claude provider (via remote API).

```ts
import { RemoteLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new RemoteLLMProvider({
  proxyUrl: '/api/chat',
  model: 'sonnet',  // resolves to claude-sonnet-4-6
});
```

**Options:**

```ts
interface RemoteLLMProviderOptions {
  proxyUrl: string;
  model?: string;  // 'haiku' | 'sonnet' | 'opus' (default: 'haiku')
  apiKey?: string;  // injected in body, read by proxy +server.ts
}
```

#### WasmProvider

Gemma 2 provider (in-browser, via WASM with Web Workers).

```ts
import { WasmProvider } from '@webmcp-auto-ui/agent';

const provider = new WasmProvider({
  modelSize: '2b',  // '2b' or '7b'
  quantized: true,  // Quantized 4-bit (recommended)
  workerUrl: new URL('@webmcp-auto-ui/agent/gemma-worker', import.meta.url),
});

// Wait for loading
await provider.ready();
```

**Status Events:**

```ts
interface WasmStatus {
  type: 'initializing' | 'loading' | 'ready' | 'error';
  progress?: number;       // 0-1
  message?: string;
}

provider.onStatus((status) => {
  console.log(status.type, status.progress);
});
```

#### LocalLLMProvider

Ollama provider (local LLM via server).

```ts
import { LocalLLMProvider } from '@webmcp-auto-ui/agent';

const provider = new LocalLLMProvider({
  baseUrl: 'http://localhost:11434',
  model: 'mistral',
});
```

### Tool Layers

**Tool layers** structure MCP/WebMCP tools in layers for lazy loading and system prompt injection.

```ts
export type ToolLayer = McpLayer | WebMcpLayer;

interface McpLayer {
  protocol: 'mcp';
  serverName: string;
  description?: string;
  tools: McpToolDef[];
  recipes?: McpRecipe[];
}

interface WebMcpLayer {
  protocol: 'webmcp';
  serverName: string;
  description: string;
  tools: WebMcpToolDef[];
}
```

#### buildSystemPromptWithAliases

Generate system prompt with tool aliases (for MCP canonical matching).

```ts
import { buildSystemPromptWithAliases } from '@webmcp-auto-ui/agent';

const { prompt, aliasMap } = buildSystemPromptWithAliases(layers);

// prompt: complete system text
// aliasMap: Map<"alias_name", "real_name"> for dynamic resolution
```

The prompt includes execution steps and discovery pseudo-tools (`search_tools`, `list_tools`).

#### buildDiscoveryToolsWithAliases

Build the initial list of discovery tools (before server activation).

```ts
import { buildDiscoveryToolsWithAliases } from '@webmcp-auto-ui/agent';

const { tools, aliasMap } = buildDiscoveryToolsWithAliases(layers);

// tools: ProviderTool[] (discovery + alias)
// aliasMap: Map for resolution
```

#### activateServerTools

Activate all tools of a layer (lazy loading on first contact).

```ts
import { activateServerTools } from '@webmcp-auto-ui/agent';

const nextTools = activateServerTools(currentTools, layer);
```

#### resolveCanonicalTools

Resolve MCP tools to canonical roles (search_recipes, list_recipes, get_recipe) via 4 matching layers.

```ts
import { resolveCanonicalTools } from '@webmcp-auto-ui/agent';

const matches = resolveCanonicalTools(mcpTools);

// matches: CanonicalMatch[]
// - role: 'search_recipes' | 'list_recipes' | 'get_recipe'
// - realToolName: real MCP tool name
```

The 4 layers:
1. **Exact match** — search `search_recipes`, `list_recipes`, `get_recipe`
2. **Decompose** — tokenize name and test all pairs (action, resource)
3. **Description keywords** — scan description for recipe/template/workflow
4. **Fallback** — no match

### Type Definitions

```ts
interface ToolCall {
  id: string;
  name: string;
  args: Record<string, unknown>;
  result?: string;
  error?: string;
  elapsed?: number;
  guided?: boolean;  // Preceded by discovery tool
}

interface AgentMetrics {
  totalTokens: number;
  promptTokens: number;
  completionTokens: number;
  totalLatencyMs: number;
  toolCalls: number;
  iterations: number;
  cacheHits: number;
}

interface ChatMessage {
  role: 'user' | 'assistant' | 'system';
  content: string | ContentBlock[];
}

type ContentBlock =
  | { type: 'text'; text: string }
  | { type: 'tool_use'; id: string; name: string; input: Record<string, unknown> }
  | { type: 'tool_result'; tool_use_id: string; content: string };

interface LLMResponse {
  content: ContentBlock[];
  stopReason: string;
  usage?: { input_tokens: number; output_tokens: number; cache_read_input_tokens?: number };
}
```

```ts

const providerTools = toProviderTools(mcpTools);
```

### fromMcpTools

Convert `McpTool[]` (from core) to `McpToolDef[]` (agent).

```ts

const toolDefs = fromMcpTools(tools);
```

### trimConversationHistory

Reduce history to a token budget by removing oldest messages.

```ts
import { trimConversationHistory } from '@webmcp-auto-ui/agent';

const trimmed = trimConversationHistory(messages, 4096);
```

## Complete Example

```ts
import { runAgentLoop, RemoteLLMProvider, autoui } from '@webmcp-auto-ui/agent';
import { McpClient } from '@webmcp-auto-ui/core';

// 1. Create an LLM provider
const provider = new RemoteLLMProvider({
  proxyUrl: '/api/chat',
  model: 'sonnet',  // resolves to claude-sonnet-4-6
});

// 2. Create an MCP client (if connecting to MCP servers)
const client = new McpClient('http://localhost:3000/mcp');
await client.initialize();

// 3. Build tool layers
const layers = [
  // MCP Layer
  {
    protocol: 'mcp' as const,
    serverName: 'recipes',
    tools: await client.listTools(),
  },
  // WebMCP Layer (autoui)
  autoui.layer(),
];

// 4. Run agent loop
const result = await runAgentLoop('Display a Q1 sales chart', {
  client,
  provider,
  layers,
  maxIterations: 5,
  callbacks: {
    onToken: (token) => process.stdout.write(token),
    onToolCall: (call) => console.log(`[${call.name}]`),
    onWidget: (type, data) => console.log(`Widget: ${type}`),
  },
});

console.log('Final:', result.text);
console.log('Metrics:', result.metrics);
```
