The `@webmcp-auto-ui/ui` package contains all Svelte 5 components for building an agent user interface.

## Widget rendering

### `<WidgetRenderer>`

Master component for displaying any widget (native, custom WebMCP, or vanilla).

```svelte
<script>
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';
  import { autoui } from '@webmcp-auto-ui/agent';

  let data = { label: 'KPI', value: '42', trend: 'up' };
</script>

<WidgetRenderer
  id="block_1"
  type="stat-card"
  {data}
  servers={[autoui]}
  oninteract={(type, action, payload) => console.log(action, payload)}
/>
```

```ts
interface Props {
  id?: string;
  type: string;
  data: Record<string, unknown>;
  servers?: WebMcpServer[];
  oninteract?: (type: string, action: string, payload: unknown) => void;
}
```

**Resolution :**
1. Search widget in `servers` (WebMCP custom)
2. Otherwise search in `NATIVE_MAP` (native Svelte widgets)
3. Otherwise show fallback `[type]`

### `<BlockRenderer>` (Alias)

Backward-compat alias for `<WidgetRenderer>`.

## Selectors and indicators

### `<LLMSelector>`

LLM model selector with local (WASM) and remote (Anthropic) options.

```svelte
<script>
  import { LLMSelector } from '@webmcp-auto-ui/ui';

  let selected = 'sonnet';
</script>

<LLMSelector
  bind:value={selected}
  onchange={(model) => console.log('Model:', model)}
/>
```

**Models :**
- `haiku` — Claude 3 Haiku (fast, low-cost)
- `sonnet` — Claude 3.5 Sonnet (balanced)
- `opus` — Claude 3 Opus (powerful)
- `gemma-e2b` — Gemma 2B WASM (in-browser)
- `gemma-e4b` — Gemma 4B WASM (in-browser)

### `<GemmaLoader>`

Loading indicator and status for Gemma WASM.

```svelte
<script>
  import { GemmaLoader, WasmProvider } from '@webmcp-auto-ui/ui';

  let wasmProvider = new WasmProvider({
    modelId: 'gemma-e2b',
    onStatus: (status) => { /* ... */ }
  });
</script>

<GemmaLoader
  provider={wasmProvider}
  showDownload={true}
  onReady={() => console.log('Gemma ready')}
/>
```

Displays :
- `idle` : Not loaded
- `loading` : % progress (32 %, 64 %, 100 %)
- `ready` : ✓ Ready
- `error` : ✗ Error + message

### `<McpStatus>`

MCP connection indicator.

```svelte
<script>
  import { McpStatus } from '@webmcp-auto-ui/ui';

  let connected = false;
  let connecting = false;
  let serverName = '';
  let toolCount = 0;
</script>

<McpStatus
  {connected}
  {connecting}
  {serverName}
  {toolCount}
  onconnect={() => { /* ... */ }}
/>
```

States :
- `● no MCP` (gray)
- `● connecting…` (orange)
- `● serverName · N tools` (green)
- `● error: ...` (red)

### `<AgentProgress>`

Agent iteration indicator + tokens.

```svelte
<script>
  import { AgentProgress } from '@webmcp-auto-ui/ui';

  let iteration = 1;
  let maxIterations = 5;
  let totalTokens = 1234;
  let currentTool = 'search_recipes';
</script>

<AgentProgress
  {iteration}
  {maxIterations}
  {totalTokens}
  {currentTool}
  oncancel={() => console.log('Stopped')}
/>
```

Displays progress bar + live token count.

### `<McpConnector>`

MCP connection dialog (URL input + button).

```svelte
<script>
  import { McpConnector } from '@webmcp-auto-ui/ui';

  let mcpUrl = 'http://localhost:3000/mcp';
</script>

<McpConnector
  bind:url={mcpUrl}

```

## Chat and history

### `<ChatPanel>`

Chat history display with user/assistant bubbles.

```svelte
<script>
  import { ChatPanel } from '@webmcp-auto-ui/ui';
  import type { ChatFeedItem } from '@webmcp-auto-ui/ui';

  let feed: ChatFeedItem[] = [];
</script>

<ChatPanel {feed} />
```

**Types :**

```ts
export interface ChatBubble {
  role: 'user' | 'assistant';
  text: string;
  thinking?: boolean;
  timestamp?: number;
}

export interface ChatBlock {
  type: 'widget' | 'tool-call' | 'error';
  widgetType?: string;
  toolName?: string;
  data?: Record<string, unknown>;
  error?: string;
}

export interface ChatFeedItem {
  id: string;
  bubble?: ChatBubble;
  blocks?: ChatBlock[];
}
```

### `<EphemeralBubble>`

Temporary feedback bubble (notification).

```svelte
<script>
  import { EphemeralBubble } from '@webmcp-auto-ui/ui';
</script>

<EphemeralBubble
  text="Tool called: search_recipes"
  duration={3000}

```

Types : `info`, `success`, `warning`, `error`.
