WebMCP Auto-UI is an automatic user interface generation platform driven by AI agents. It combines the **Model Context Protocol (MCP)** for data integration, an **LLM agent engine** for orchestration, and a **declarative widget system** for client-side rendering.

## Overview

## Key Features

- **Autonomous agents**: Native agentic loop with support for Claude (API) and Gemma (in-browser WASM)
- **Multi-protocol tool calling**: Transparent dispatch to MCP (network protocol) and WebMCP (local)
- **30+ native widgets**: Stat, chart, timeline, map, gallery, hemicycle, etc.
- **Lazy loading**: Load tools on demand to optimize LLM context
- **Skill serialization**: Export/import via HyperSkills with gzip compression
- **Reactive canvas**: Svelte 5 rendering with FONC message bus for inter-component communication

## Key Concepts

### 1. Tool Layers (tool stacks)

Tools are organized in **structured layers**:
- **MCP Layers**: Remote MCP servers (data, external APIs)
- **WebMCP Layers**: Local servers (widgets, UI actions, storage)

Each tool receives a canonical prefix: `{serverName}_{protocol}_{toolName}`

### 2. Agent Loop (agentic loop)

The agent executes a loop until `end_turn` or `max_iterations`:

```
1. LLM generates tool_use blocks
2. Dispatcher routes to MCP or WebMCP
3. Compressed results return to agent
4. Back to step 1
```

### 3. Widget Discovery (lazy discovery)

On startup, only **discovery tools** are available:
- `search_recipes()` / `list_recipes()` / `get_recipe()`
- `search_tools()` / `list_tools()`

When the agent calls a real tool for the 1st time, the server **activates** and all its tools become available.

### 4. HyperSkills (serialization)

Canvases are exported as **skills** JSON, compressed into short URLs via gzip/Brotli:
- `encodeHyperSkill()`: canvas → shareable URL
- `decodeHyperSkill()`: URL → restored canvas

## Packages

| Package | Responsibility |
|---------|-----------------|
| `@webmcp-auto-ui/core` | WebMCP types, polyfill, MCP client, validation |
| `@webmcp-auto-ui/agent` | Agent loop, LLM providers, tool layers, recipes |
| `@webmcp-auto-ui/ui` | Svelte components (30+ widgets), canvas store |
| `@webmcp-auto-ui/sdk` | HyperSkills, skills registry, canvas store |

## Tech Stack

- **Frontend framework**: Svelte 5 (runes)
- **Styling**: UnoCSS (dark/light themes)
- **LLM**: Anthropic Claude (API) + Google Gemma 2 (WASM)
- **Worker**: Web Worker for Gemma (non-blocking UI)
- **Schema**: JSON Schema (agent-side validation)
- **Communication**: postMessage bridge (cross-document WebMCP)

## Use Cases

- **Data explorers**: Agent navigates database, displays results in charts/tables
- **Creative assistants**: LLM generates content, auto-arranges UI in grid/carousel
- **Smart dashboards**: Agent collects KPIs, builds reactive dashboard
- **Interactive configurators**: Agent guides users via generated forms
- **Skill sharing**: Export canvas → short URL → share with collaborators
