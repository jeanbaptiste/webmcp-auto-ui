## MVC + LLM + HyperSkills Architecture

```
User prompt --> Agent Loop --> LLM (Claude/Gemma/Ollama)
                  |                    |
                  |              tool_use blocks
                  |                    |
         +--------+--------+          |
         |                  |          v
    McpLayer (DATA)    UILayer (UI)
    MCP Servers        component()
         |                  |
    Raw data           Rendered blocks
         |                  |
         +--------+---------+
                  |
            BlockRenderer
                  |
            Canvas / UI
```

<CardGrid stagger>
  <Card title="LLM Agent" icon="rocket">
    Iterative agent loop with 4 LLM providers: Claude (remote), Gemma LiteRT (WASM in-browser), Ollama and Llamafile (local). Smart mode (`component()` single tool) or explicit mode (31 `render_*` tools).
  </Card>

  <Card title="34+ UI Components" icon="puzzle">

  </Card>

  <Card title="MCP Servers" icon="setting">
    MCP Streamable HTTP client with simultaneous multi-server support. Tool discovery, server recipes, and automatic routing via `McpMultiClient`.
  </Card>

  <Card title="HyperSkills" icon="document">
    Skill encode/decode into URLs (`?hs=`), CRUD registry, SHA-256 versioning, reactive canvas store (Svelte 5 runes + vanilla).
  </Card>
</CardGrid>

## Packages

| Package | Description |
|---------|-------------|
