WebMCP Auto-UI provides 30+ ready-to-use widgets: charts, tables, maps, galleries, timelines, and more. This tutorial shows how to use them in a Svelte app.

## Available widgets

### Simple widgets (direct data)

| Widget | Usage | Example |
|--------|-------|---------|
| `stat` | Simple KPI | `{label: "Total", value: "42", trend: "up"}` |
| `alert` | Notification | `{title: "Attention", level: "warn", message: "..."}` |
| `code` | Code block | `{lang: "python", content: "print('hello')"}` |
| `text` | Paragraph | `{content: "Free text..."}` |
| `list` | Enumeration | `{items: ["A", "B", "C"]}` |
| `kv` | Key-value pairs | `{rows: [["Name", "Dupont"], ["Age", "42"]]}` |
| `tags` | Badges | `{tags: [{text: "JS", active: true}]}` |
| `actions` | Buttons | `{buttons: [{label: "OK", primary: true}]}` |
| `chart` | Simple bars | `{bars: [["Jan", 10], ["Feb", 20]]}` |

### Rich widgets (complex specifications)

| Widget | Usage | Use case |
|--------|-------|----------|
| `stat-card` | Enhanced KPI | Dashboards with delta, trend, color |
| `data-table` | Interactive table | Tabular data with sorting |
| `timeline` | Chronology | Dated events with statuses |
| `profile` | Person card | Bio, fields, statistics |
| `trombinoscope` | Portrait grid | Teams, assemblies |
| `cards` | Card grid | Results, folders, gallery |
| `json-viewer` | JSON tree | Complex structure inspection |
| `chart-rich` | Multi-series | Bar, line, area, pie, donut |
| `sankey` | Flow diagram | Votes, journeys, transfers |
| `map` | Leaflet map | Geographic markers |
| `hemicycle` | SVG hemicycle | Parliamentary composition |
| `grid-data` | Heatmap grid | Comparison, highlighting |
| `d3` | D3 visualizations | Treemap, hex-heatmap, force graph |
| `gallery` | Image gallery | Lightbox and navigation |
| `carousel` | Slideshow | Slides with auto-play |
| `log` | Log stream | System events |
| `js-sandbox` | JavaScript code | Interactive prototypes |

## Example 1: Dashboard with simple widgets

```svelte
<!-- routes/dashboard/+page.svelte -->
<script lang="ts">
  import { BlockRenderer } from '@webmcp-auto-ui/ui';

  let blocks = [
    {
      id: 'stat-1',
      type: 'stat',
      data: {
        label: 'Active users',
        value: '2,847',
        trend: 'up',
        trendDir: 'up',
      },
    },
    {
      id: 'alert-1',
      type: 'alert',
      data: {
        title: 'Scheduled maintenance',
        message: 'Friday 22:00-23:00 UTC',
        level: 'info',
      },
    },
    {
      id: 'list-1',
      type: 'list',
      data: {
        title: 'Priority tasks',
        items: [
          'Migrate PostgreSQL database',
          'Enable 2FA for all',
          'Audit API permissions',
        ],
      },
    },
  ];
</script>

<div class="grid grid-cols-1 md:grid-cols-3 gap-4 p-6">
  {#each blocks as block (block.id)}
    <BlockRenderer type={block.type} data={block.data} />
  {/each}
</div>
```

## Example 2: Data table with sorting

```svelte
<!-- routes/users/+page.svelte -->
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';

  const tableData = {
    title: 'Users',
    columns: [
      { key: 'name', label: 'Name', align: 'left' },
      { key: 'email', label: 'Email', align: 'left' },
      { key: 'role', label: 'Role', align: 'center' },
      { key: 'joined', label: 'Joined', align: 'right' },
    ],
    rows: [
      { name: 'Alice Dupont', email: 'alice@example.com', role: 'Admin', joined: '2023-01-15' },
      { name: 'Bob Martin', email: 'bob@example.com', role: 'User', joined: '2023-06-20' },
      { name: 'Claire Lemaître', email: 'claire@example.com', role: 'Editor', joined: '2024-02-10' },
    ],
  };

  function handleRowClick(row: Record<string, unknown>) {
    console.log('Row clicked:', row);
    // Navigate to user profile
  }
</script>

<WidgetRenderer
  type="data-table"
  data={tableData}
  oninteract={(type, action, payload) => {
    if (action === 'rowclick') {
      handleRowClick(payload as Record<string, unknown>);
    }
  }}
/>
```

## Example 5: Image gallery with URL validation

WebMCP Auto-UI automatically validates image URLs. Hallucinated URLs are filtered:

```svelte
<script lang="ts">
  import { WidgetRenderer } from '@webmcp-auto-ui/ui';

  const gallery = {
    title: 'Conference photos',
    images: [
      {
        src: 'https://images.example.com/conf-01.jpg', // ✅ Valid URL (https)
        alt: 'Keynote speaker',
        caption: 'Main speech',
      },
      {
        src: 'https://images.example.com/conf-02.jpg',
        alt: 'Networking',
        caption: 'Coffee break',
      },
      // src: 'hallucinatedimage.jpg' — auto-filtered ❌
    ],
    columns: 3,
  };
</script>

<WidgetRenderer type="gallery" data={gallery} />
```

## Integration with agent loop

Use `widget_display()` from the agent to add widgets dynamically:

```typescript
// In an MCP recipe or WebMCP tool
const result = await webmcpServer.executeTool('widget_display', {
  name: 'data-table',
  params: {
    title: 'Search results',
    columns: [
      { key: 'title', label: 'Title' },
      { key: 'relevance', label: 'Relevance' },
    ],
    rows: searchResults.map((r, i) => ({
      title: r.title,
      relevance: `${Math.round(r.score * 100)}%`,
    })),
  },
});
```

## Interaction callbacks

Rich widgets support callbacks to capture user interactions:

```svelte
<WidgetRenderer
  type="cards"
  data={cardsData}
  oninteract={(type, action, payload) => {
    if (action === 'cardclick') {
      // User clicked a card
      const card = payload as Record<string, unknown>;
      console.log('Card clicked:', card);
    }
  }}
/>
```

**Available actions by widget:**

| Widget | Actions |
|--------|---------|
| `data-table` | `rowclick` |
| `timeline` | `eventclick` |
| `trombinoscope` | `personclick` |
| `hemicycle` | `groupclick` |
| `cards` | `cardclick` |
| `gallery` | `imageclick` |
| `carousel` | `slidechange` |
| `list` | `itemclick` |

## Key points

✅ **SafeImage filters hallucinated URLs** — no manual validation needed

✅ **BlockRenderer for simple widgets** — lighter weight when no interaction

✅ **WidgetRenderer when callbacks are needed** — to handle user clicks

✅ **Full TypeScript types** — IDE suggests available fields

❌ **Don't pass invalid image URLs** — they will be filtered

❌ **Don't forget required schema fields** — widgets must validate input
