This tutorial shows how to create and register a custom widget in the WebMCP Auto-UI component system, then use it via `BlockRenderer`.

## Overview

A custom widget is composed of:
1. **A Svelte component** that displays the data
2. **A registration** in the canvas registry
3. **A TypeScript type** for validation

The system supports two rendering modes:
- **Svelte**: via `<BlockRenderer>` (recommended for web apps)
- **Vanilla**: via `mountWidget()` for framework-free integration

## Step 1: Create the component

Create a simple Svelte component, for example `CustomProgressBar.svelte`:

```svelte
<script lang="ts">
  interface Props {
    data: {
      label: string;
      current: number;
      max: number;
      color?: string;
    };
  }

  let { data }: Props = $props();
  const percentage = $derived((data.current / data.max) * 100);
</script>

<div class="card">
  <h3>{data.label}</h3>
  <div class="progress-container">
    <div
      class="progress-bar"
      style="width: {percentage}%; background-color: {data.color ?? '#3b82f6'}"
    />
  </div>
  <p class="text-sm">{data.current} / {data.max}</p>
</div>

<style>
  .card {
    padding: 1rem;
    border: 1px solid #e5e7eb;
    border-radius: 0.5rem;
  }

  .progress-container {
    height: 8px;
    background: #f3f4f6;
    border-radius: 4px;
    overflow: hidden;
    margin: 0.5rem 0;
  }

  .progress-bar {
    height: 100%;
    transition: width 0.3s ease;
  }
</style>
```

## Step 2: Register the widget in the registry

Edit your app's registry file (e.g. `src/lib/widgets.ts`):

```typescript
import CustomProgressBar from './widgets/CustomProgressBar.svelte';

export const CUSTOM_WIDGETS: Record<string, Component> = {
  'progress-bar': CustomProgressBar,
};
```

## Step 3: Use BlockRenderer

In your parent component, import `BlockRenderer` and pass the widget:

```svelte
<script lang="ts">
  import { BlockRenderer } from '@webmcp-auto-ui/ui';
  import { CUSTOM_WIDGETS } from './lib/widgets';
  import { canvas } from '@webmcp-auto-ui/sdk/canvas';

  // Add a widget to the canvas
  function addProgress() {
    canvas.addWidget('progress-bar', {
      label: 'Download',
      current: 45,
      max: 100,
      color: '#10b981',
    });
  }
</script>

<button onclick={addProgress}>Add progress bar</button>

<div>
  {#each canvas.blocks as block (block.id)}
    <BlockRenderer 
      id={block.id}
      type={block.type}
      data={block.data}
      customWidgets={CUSTOM_WIDGETS}
    />
  {/each}
</div>
```

## Step 4: Type the widget

Add the type to the `WidgetType` union definition:

```typescript
// In your types.ts or types.d.ts
declare global {
  namespace WebMCP {
    type WidgetType = 
      | 'stat' | 'chart' | 'kv' | 'list'
      | 'progress-bar';  // <-- your custom widget
  }
}
```

Or in the SDK's `canvas.ts` if you are modifying the `WidgetType` type:

```typescript
export type WidgetType =
  | 'stat' | 'kv' | 'list' | 'chart' | 'alert' | 'code' | 'text' | 'actions' | 'tags'
  | 'stat-card' | 'data-table' | 'timeline' | 'profile' | 'trombinoscope' | 'json-viewer'
  | 'hemicycle' | 'chart-rich' | 'cards' | 'grid-data' | 'sankey' | 'map' | 'log'
  | 'gallery' | 'carousel' | 'd3' | 'js-sandbox'
  | 'progress-bar'  // <-- custom type addition
  | (string & {});  // fallback for plugin widgets
```

## Step 5: Add interactions

If your widget needs to communicate with the agent via the FONC bus, use callbacks:

```svelte
<script lang="ts">
  import { bus } from '@webmcp-auto-ui/ui';

  interface Props {
    id?: string;
    data: {
      label: string;
      current: number;
      max: number;
      onUpdate?: (value: number) => void;
    };
  }

  let { id, data }: Props = $props();

  function handleClick() {
    // Emit a bus event to the canvas
    if (id) {
      bus.broadcast(id, 'progress-complete', { label: data.label });
    }
    data.onUpdate?.(data.max);
  }
</script>

<div class="progress-bar">
  <progress value={data.current} max={data.max}></progress>
  <button onclick={handleClick}>Done</button>
</div>
```

## Step 6: Integrate with the WebMCP bus (optional)

To allow the agent to update the widget:

```svelte
<script lang="ts">
  import { onMount } from 'svelte';

  onMount(() => {
    const mc = (navigator as any).modelContext;
    if (!mc) return;

    mc.registerTool({
      name: `progress_${id}_update`,
      description: `Update progress bar widget`,
      inputSchema: {
        type: 'object',
        properties: {
          current: { type: 'number' },
        },
        required: ['current'],
      },
      execute: (args: Record<string, any>) => {
        data.current = args.current;
        return {
          content: [{ type: 'text', text: `Progress updated to ${args.current}` }],
        };
      },
    });
  });
</script>
```

## Full example: widget with persistent state

```svelte
<script lang="ts">
  import { canvas } from '@webmcp-auto-ui/sdk/canvas';

  interface Props {
    id: string;
    data: {
      items: string[];
      completed: number;
    };
    oninteract?: (type: string, action: string, payload: unknown) => void;
  }

  let { id, data, oninteract }: Props = $props();
  let items = $state(data.items);
  let completed = $state(data.completed);

  function toggleItem(index: number) {
    completed = completed === index ? -1 : index;
    // Update the canvas
    canvas.updateBlock(id, { completed });
    // Emit an interaction
    oninteract?.('checklist', 'toggle', { index, completed });
  }
</script>

<div class="checklist">
  <h3>Checklist ({completed + 1}/{items.length})</h3>
  {#each items as item, i (i)}
    <label>
      <input
        type="checkbox"
        checked={completed === i}
        onchange={() => toggleItem(i)}
      />
      <span class:done={completed === i}>{item}</span>
    </label>
  {/each}
</div>

<style>
  .checklist { padding: 1rem; }
  label { display: flex; gap: 0.5rem; margin: 0.5rem 0; }
  .done { text-decoration: line-through; color: #6b7280; }
</style>
```

## Integration with the WebMCP system

If you want to register your widget in a **custom WebMCP server** (for example an app that exposes its own widgets via MCP):

```typescript
import { createWebMcpServer } from '@webmcp-auto-ui/core';

const myServer = createWebMcpServer('my-widgets', {
  description: 'Custom widgets for my app',
});

// Register the widget with its recipe
myServer.registerWidget(`---
widget: progress-bar
description: Interactive progress bar
schema:
  type: object
  required:
    - label
    - current
    - max
  properties:
    label:
      type: string
    current:
      type: number
    max:
      type: number
    color:
      type: string
---

## Usage

Call widget_display('progress-bar', {label: "Download", current: 45, max: 100})
`, undefined); // renderer = undefined -- will be resolved by BlockRenderer

export { myServer };
```
