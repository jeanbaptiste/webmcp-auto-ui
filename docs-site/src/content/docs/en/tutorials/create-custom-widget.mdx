A **widget** in WebMCP Auto-UI is a reusable component that displays data on the canvas. This tutorial shows how to create a custom widget, register it in the system, and use it in your application.

## Widget Architecture

WebMCP Auto-UI supports two rendering modes:

- **Svelte** (default): via `<WidgetRenderer>` and `<BlockRenderer>` in `@webmcp-auto-ui/ui`
- **Vanilla**: via `mountWidget()` in `@webmcp-auto-ui/core` (no framework dependency)

Native widgets are defined in the WebMCP `autoui` server (`packages/agent/src/autoui-server.ts`), which exposes approximately 30 widgets (stat, chart, gallery, carousel, etc.).

## Example: Creating an "Enhanced Stat Card" Widget

Suppose you want a widget that displays a title, main value, trend, and mini progress bar.

### Step 1: Define the JSON Schema

The schema defines the parameters expected by the widget. It must follow the JSON Schema format:

```typescript
const widgetSchema = {
  type: 'object',
  required: ['title', 'value'],
  properties: {
    title: {
      type: 'string',
      description: 'Widget title'
    },
    value: {
      type: 'number',
      description: 'Main displayed value'
    },
    max: {
      type: 'number',
      description: 'Maximum bar value (default: 100)',
      default: 100
    },
    unit: {
      type: 'string',
      description: 'Unit of measurement (e.g., "%", "ms")'
    },
    color: {
      type: 'string',
      description: 'Hexadecimal color (e.g., "#3b82f6")',
      default: '#3b82f6'
    },
    trend: {
      type: 'string',
      enum: ['up', 'down', 'flat'],
      description: 'Trend direction'
    }
  }
};
```

### Step 2: Create the Svelte Component

Create a file `StatProgress.svelte`:

```svelte
<script lang="ts">
  interface Props {
    data: {
      title: string;
      value: number;
      max?: number;
      unit?: string;
      color?: string;
      trend?: 'up' | 'down' | 'flat';
    };
  }

  let { data }: Props = $props();

  const percentage = $derived(Math.min(100, (data.value / (data.max ?? 100)) * 100));
  
  const trendIcon = $derived.by(() => {
    switch (data.trend) {
      case 'up': return '↗';
      case 'down': return '↘';
      default: return '→';
    }
  });

  const trendColor = $derived.by(() => {
    switch (data.trend) {
      case 'up': return 'text-green-500';
      case 'down': return 'text-red-500';
      default: return 'text-gray-500';
    }
  });
</script>

<div class="p-4 rounded-lg border border-gray-200 bg-white shadow-sm">
  <div class="flex items-center justify-between mb-2">
    <h3 class="font-semibold text-gray-900">{data.title}</h3>
    <span class={`text-lg font-bold ${trendColor}`}>
      {trendIcon}
    </span>
  </div>

  <div class="flex items-baseline gap-2 mb-3">
    <span class="text-3xl font-bold" style="color: {data.color ?? '#3b82f6'}">
      {data.value}
    </span>
    {#if data.unit}
      <span class="text-sm text-gray-600">{data.unit}</span>
    {/if}
  </div>

  <div class="w-full bg-gray-100 rounded-full h-2">
    <div
      class="h-2 rounded-full transition-all"
      style="width: {percentage}%; background-color: {data.color ?? '#3b82f6'}"
    />
  </div>

  <div class="text-xs text-gray-500 mt-2 text-right">
    {percentage.toFixed(0)}% of {data.max ?? 100}
  </div>
</div>

<style>
  :global {
    /* Optional: global styles if needed */
  }
</style>
```

### Step 3: Register the Widget

#### Option A: Via a WebMCP Server

If you're creating a custom WebMCP server, register the widget with a frontmatter recipe:

```typescript
import { createWebMcpServer } from '@webmcp-auto-ui/core';
import StatProgress from './StatProgress.svelte';

const myServer = createWebMcpServer('my-widgets', {
  description: 'My custom widgets'
});

const recipe = `---
widget: stat-progress
description: Enhanced stat card with progress bar and trend indicator.
schema:
  type: object
  required:
    - title
    - value
  properties:
    title:
      type: string
    value:
      type: number
    max:
      type: number
      default: 100
    unit:
      type: string
    color:
      type: string
      default: "#3b82f6"
    trend:
      type: string
      enum: [up, down, flat]
---

## When to use

For displaying a main metric with progress bar and trend indicator.

## How

Call widget_display('stat-progress', {title: "Uptime", value: 99.8, max: 100, unit: "%", trend: "up", color: "#10b981"})
`;

myServer.registerWidget(recipe, StatProgress);
```

#### Option B: In an App (Canvas Store)

If you're using the canvas store, add the widget to the types list and render it via `<WidgetRenderer>`:

```svelte
<script lang="ts">
  import { canvas } from '@webmcp-auto-ui/sdk/canvas';
  import WidgetRenderer from '@webmcp-auto-ui/ui/widgets/WidgetRenderer.svelte';
  import StatProgress from './StatProgress.svelte';

  // Create a WebMCP layer to register the widget
  function createCustomWidgets() {
    const server = createWebMcpServer('custom', {
      description: 'App custom widgets'
    });

    const recipe = `---
widget: stat-progress
description: Stat with progress bar
schema:
  type: object
  required: [title, value]
  properties:
    title: { type: string }
    value: { type: number }
    max: { type: number, default: 100 }
    unit: { type: string }
    color: { type: string, default: "#3b82f6" }
    trend: { type: string, enum: [up, down, flat] }
---

Enhanced stat card.`;

    server.registerWidget(recipe, StatProgress);
    return server;
  }

  const servers = [createCustomWidgets()];
</script>

<div class="space-y-4">
  {#each canvas.blocks as block (block.id)}
    <WidgetRenderer 
      id={block.id}
      type={block.type}
      data={block.data}
      {servers}
      oninteract={(type, action, payload) => {
        console.log(`Widget ${type} triggered action ${action}:`, payload);
      }}
    />
  {/each}
</div>
```

### Step 4: Use the Widget in the Agent

Once registered, the agent can call the widget via `widget_display()`:

```typescript
import { runAgentLoop } from '@webmcp-auto-ui/agent';

// The agent automatically calls widget_display() with the right parameters
const result = await runAgentLoop(
  'Display system performance statistics',
  {
    provider: llmProvider,
    layers: [
      myServer.layer() // Include the WebMCP layer
    ],
    callbacks: {
      onWidget: (type, data) => {
        if (type === 'stat-progress') {
          canvas.addWidget('stat-progress', data);
        }
      }
    }
  }
);
```

## Advanced: Adding Interactions

Widgets can emit events and react to user interactions:

```svelte
<script lang="ts">
  interface Props {
    data: Record<string, unknown>;
    oninteract?: (action: string, payload: unknown) => void;
  }

  let { data, oninteract }: Props = $props();

  function handleClick() {
    oninteract?.('expand', { widgetId: data.id });
  }
</script>

<div class="cursor-pointer hover:shadow-md transition" onclick={handleClick}>
  <!-- Widget content -->
</div>
```

Then in the parent:

```svelte
<WidgetRenderer
  id={block.id}
  type={block.type}
  data={block.data}
  oninteract={(type, action, payload) => {
    if (action === 'expand') {
      // Open a modal, scroll to widget, etc.
      console.log('Widget expand:', payload);
    }
  }}
/>
```

## Summary

| Step | Action |
|------|--------|
| 1 | Define the JSON schema of parameters |
| 2 | Create the Svelte component (receives `data` as props) |
| 3 | Register via `server.registerWidget(recipe, Component)` |
| 4 | Render via `<WidgetRenderer>` or agent via `widget_display()` |
| 5 | (Optional) Add interactions with `oninteract` |

For more details, see the native widgets in `packages/agent/src/autoui-server.ts` and Svelte components in `packages/ui/src/widgets/`.
