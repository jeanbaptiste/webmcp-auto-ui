WebMCP Auto-UI est une plateforme de génération automatique d'interfaces utilisateur pilotées par des agents IA. Elle combine le **Model Context Protocol (MCP)** pour l'intégration de données, un **moteur d'agents LLM** pour l'orchestration, et un **système de widgets déclaratifs** pour le rendu côté client.

## Caractéristiques principales

- **Agents autonomes** : Boucle agentic native avec support pour Claude (API) et Gemma (in-browser WASM)
- **Tool calling multi-protocoles** : Dispatch transparent vers MCP (protocole réseau) et WebMCP (local)
- **30+ widgets natifs** : Stat, chart, timeline, carte, galerie, hemicycle parlementaire, etc.
- **Lazy loading** : Charge les outils à la demande pour optimiser le contexte LLM
- **Skill sérialisation** : Export/import via HyperSkills avec compression gzip
- **Canvas réactif** : Rendu Svelte 5 avec message bus FONC pour l'inter-composants

## Concepts clés

### 1. Tool Layers (couches d'outils)

Les outils sont organisés en **couches structurées** :
- **MCP Layers** : Serveurs MCP distants (données, APIs externes)
- **WebMCP Layers** : Serveurs locaux (widgets, actions UI, stockage)

Chaque outil reçoit un préfixe canonique : `{serverName}_{protocol}_{toolName}`

### 2. Agent Loop (boucle agentic)

L'agent exécute une boucle jusqu'à `end_turn` ou `max_iterations` :

```
1. LLM génère tool_use blocks
2. Dispatcher route vers MCP ou WebMCP
3. Résultats compressés reviennent à l'agent
4. Retour à l'étape 1
```

### 3. Widget Discovery (découverte lazy)

Au démarrage, seuls les outils de **discovery** sont disponibles :
- `search_recipes()` / `list_recipes()` / `get_recipe()`
- `search_tools()` / `list_tools()`

Quand l'agent appelle un outil réel pour la 1ère fois, le serveur s'**active** et tous ses outils deviennent disponibles.

### 4. HyperSkills (sérialisation)

Les canvas sont exportés en tant que **skills** JSON, compressés en URLs courtes via gzip/Brotli :
- `encodeHyperSkill()` : canvas → URL shareable
- `decodeHyperSkill()` : URL → canvas restauré

## Packages

| Package | Responsabilité |
|---------|-----------------|
| `@webmcp-auto-ui/core` | Types WebMCP, polyfill, MCP client, validation |
| `@webmcp-auto-ui/agent` | Boucle agent, providers LLM, tool layers, recipes |
| `@webmcp-auto-ui/ui` | Composants Svelte (30+ widgets), theme, message bus |
| `@webmcp-auto-ui/sdk` | HyperSkills, skills registry, canvas store |

## Stack technique

- **Framework frontend** : Svelte 5 (runes)
- **Styling** : UnoCSS (dark/light themes)
- **LLM** : Anthropic Claude (API) + Google Gemma 4 (LiteRT)
- **Worker** : Web Worker pour Gemma (sans blocage UI)
- **Schéma** : JSON Schema (validation côté agent)
- **Communication** : postMessage bridge (cross-document WebMCP)

## Cas d'usage

- **Explorateurs de données** : agent navigue une base de données, affiche résultats en charts/tables
- **Assistants créatifs** : LLM génère contenus, arrange UI automatiquement en grille/carousel
- **Dashboards intelligents** : agent collecte KPIs, construit tableau de bord réactif
- **Configurateurs interactifs** : agent guident utilisateur via des formulaires générés
- **Skill sharing** : exporte canvas → URL courte → partage avec collaborateurs
