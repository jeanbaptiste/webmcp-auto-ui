Creer des widgets custom avec WebMCP Auto-UI

Creating Custom Widgets with WebMCP Auto-UI

WebMCP Auto-UI permet de creer des composants UI que l'IA sait utiliser automatiquement. Chaque widget est accompagne d'une recette — un fichier YAML qui decrit son schema de donnees et son contexte d'utilisation. L'agent IA lit ces recettes, comprend quand afficher chaque widget et le remplit avec les bonnes donnees.

Ce guide montre comment creer 3 widgets pour afficher des donnees parlementaires francaises, inspires du projet Tricoteuses (open data Assemblee nationale).

WebMCP Auto-UI lets you create UI components that AI agents can use automatically. Each widget comes with a recipe — a YAML file describing its data schema and usage context. The AI agent reads these recipes, understands when to display each widget, and fills it with the right data.

This guide walks through creating 3 widgets for displaying French parliamentary data, inspired by the Tricoteuses project (National Assembly open data).

Fonctionnement

How it works

Le flux complet : un serveur MCP expose des donnees, l'agent cherche une recette adaptee, lit le schema, appelle les outils MCP, puis affiche le widget.

The full flow: an MCP server exposes data, the agent searches for a matching recipe, reads its schema, calls MCP tools, then displays the widget.

Chaque widget se compose de 3 parties : le composant Svelte (rendu), la recette YAML (contrat de donnees), et l'appel registerWidget() (enregistrement).

Each widget has 3 parts: the Svelte component (rendering), the YAML recipe (data contract), and the registerWidget() call (registration).

Widget 1 : Fiche Depute

Widget 1: Deputy Card

Ce widget affiche la fiche complete d'un depute : photo, groupe parlementaire, circonscription et statistiques de vote.

This widget displays a deputy's full card: photo, parliamentary group, constituency, and voting statistics.

Recette YAML

YAML Recipe

widget: fiche-depute
description: Fiche d'un depute de l'Assemblee nationale. Photo, groupe, circonscription, stats de vote.
schema:
  type: object
  required: [nom, prenom, groupe]
  properties:
    nom:            { type: string }
    prenom:         { type: string }
    photo:          { type: string }
    groupe:         { type: string }
    groupeCouleur:  { type: string }
    circonscription: { type: string }
    mandatDebut:    { type: string }
    mandatFin:      { type: string }
    actif:          { type: boolean }
    participation:  { type: number }
    votePour:       { type: number }
    voteContre:     { type: number }
    voteAbstention: { type: number }

Composant Svelte

Svelte Component

<!-- FicheDepute.svelte -->
<script>
  export let data;
  $: total = (data.votePour || 0) + (data.voteContre || 0) + (data.voteAbstention || 0);
  $: pct = (v) => total ? Math.round(v / total * 100) : 0;
</script>

<div class="fiche">
  {#if data.photo}
    <img src={data.photo} alt="{data.prenom} {data.nom}" class="photo" />
  {:else}
    <div class="photo placeholder">{data.prenom?.[0]}{data.nom?.[0]}</div>
  {/if}
  <h3>{data.prenom} {data.nom}</h3>
  <span class="badge" style="background:{data.groupeCouleur || '#6366f1'}30">
    {data.groupe}
  </span>
  {#if data.circonscription}
    <p class="circ">{data.circonscription}</p>
  {/if}
  {#if total}
    <div class="bar">
      <div style="width:{pct(data.votePour)}%;background:#22c55e"></div>
      <div style="width:{pct(data.voteContre)}%;background:#ef4444"></div>
      <div style="width:{pct(data.voteAbstention)}%;background:#f59e0b"></div>
    </div>
  {/if}
</div>

Enregistrement

Registration

import { registerWidget } from '@webmcp-auto-ui/core';
import FicheDepute from './FicheDepute.svelte';

registerWidget('fiche-depute', FicheDepute);

Widget 2 : Resultat de Scrutin

Widget 2: Vote Result

Affiche le resultat d'un vote a l'Assemblee nationale avec une barre proportionnelle pour/contre/abstentions.

Displays the result of a National Assembly vote with a proportional for/against/abstention bar.

Recette YAML

YAML Recipe

widget: resultat-scrutin
description: Resultat d'un scrutin de l'Assemblee nationale.
schema:
  type: object
  required: [titre, pour, contre]
  properties:
    titre:       { type: string }
    numero:      { type: number }
    date:        { type: string }
    pour:        { type: number }
    contre:      { type: number }
    abstentions: { type: number }
    votants:     { type: number }
    adopte:      { type: boolean }

Composant Svelte

Svelte Component

<!-- ResultatScrutin.svelte -->
<script>
  export let data;
  $: total = (data.pour||0) + (data.contre||0) + (data.abstentions||0);
  $: w = (v) => total ? (v/total*100).toFixed(1) : 0;
</script>

<div class="scrutin">
  <h3>{data.titre}</h3>
  {#if data.numero}<p class="sub">Scrutin n.{data.numero}</p>{/if}
  <div class="bar">
    <div class="for" style="width:{w(data.pour)}%"></div>
    <div class="against" style="width:{w(data.contre)}%"></div>
    <div class="abs" style="width:{w(data.abstentions)}%"></div>
  </div>
  <div class="result" class:adopted={data.adopte}>
    {data.adopte ? 'ADOPTE' : 'REJETE'}
  </div>
</div>
import { registerWidget } from '@webmcp-auto-ui/core';
import ResultatScrutin from './ResultatScrutin.svelte';

registerWidget('resultat-scrutin', ResultatScrutin);

Widget 3 : Amendement

Widget 3: Amendment

Affiche un amendement parlementaire avec son auteur, son article cible et son sort.

Displays a parliamentary amendment with its author, target article, and outcome.

Recette YAML

YAML Recipe

widget: amendement
description: Amendement parlementaire.
schema:
  type: object
  required: [numero, article, auteur, sort]
  properties:
    numero:       { type: string }
    article:      { type: string }
    auteur:       { type: string }
    groupe:       { type: string }
    exposeMotifs: { type: string }
    sort:         { type: string, enum: [adopte, rejete, retire, non_soutenu] }
    date:         { type: string }

Composant Svelte

Svelte Component

<!-- Amendement.svelte -->
<script>
  export let data;
  const colors = {
    adopte: '#22c55e', rejete: '#ef4444',
    retire: '#f59e0b', non_soutenu: '#94a3b8'
  };
  $: color = colors[data.sort] || '#94a3b8';
  $: label = data.sort?.replace('_', ' ').toUpperCase();
</script>

<div class="amendement">
  <h3>Amendement {data.numero}</h3>
  <p class="article">Article {data.article}</p>
  <p>{data.auteur} {#if data.groupe}<span class="grp">({data.groupe})</span>{/if}</p>
  {#if data.exposeMotifs}<blockquote>{data.exposeMotifs}</blockquote>{/if}
  <span class="badge" style="background:{color}30;color:{color}">{label}</span>
</div>
import { registerWidget } from '@webmcp-auto-ui/core';
import Amendement from './Amendement.svelte';

registerWidget('amendement', Amendement);

Cycle de vie de l'agent

Agent Lifecycle

Voici le cycle complet, de la question utilisateur jusqu'au rendu du widget.

Here is the full cycle, from user question to widget rendering.

L'agent ne connait pas a l'avance les widgets disponibles. Il decouvre les recettes via l'outil buildDiscoveryTools, puis active les outils du serveur MCP avec activateServerTools. Le rendu est declenche par widget_display qui passe les donnees au composant Svelte enregistre.

The agent does not know available widgets in advance. It discovers recipes via buildDiscoveryTools, then activates MCP server tools with activateServerTools. Rendering is triggered by widget_display which passes data to the registered Svelte component.

Bonnes pratiques

Best Practices

Descriptions claires. La description de la recette est ce que le LLM lit pour decider quel widget utiliser. Soyez precis sur le contexte d'usage.
Clear descriptions. The recipe description is what the LLM reads to decide which widget to use. Be specific about the usage context.
Schemas stricts. Declarez les champs required pour que l'agent sache quelles donnees sont indispensables. Utilisez les champs optionnels pour l'enrichissement (photo, couleur du groupe, etc.).
Strict schemas. Declare required fields so the agent knows which data is mandatory. Use optional fields for enrichment (photo, group color, etc.).
Enums pour les choix fermes. Utilisez enum pour les valeurs a choix restreint (comme sort dans l'amendement). Cela guide l'agent vers des valeurs valides.
Enums for closed choices. Use enum for constrained values (like sort in the amendment widget). This guides the agent toward valid values.
Test sans serveur MCP. Vous pouvez tester un widget en passant des donnees statiques au composant Svelte directement, sans avoir besoin d'un serveur MCP fonctionnel.
Test without MCP server. You can test a widget by passing static data to the Svelte component directly, without needing a running MCP server. 
// Quick test — pass static data
import FicheDepute from './FicheDepute.svelte';

new FicheDepute({
  target: document.getElementById('test'),
  props: {
    data: {
      nom: 'Dupont', prenom: 'Jean',
      groupe: 'RE', circonscription: 'Paris 5e',
      votePour: 72, voteContre: 18, voteAbstention: 10
    }
  }
});
